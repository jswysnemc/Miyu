### []{#_converting_a_text_file_with_iconv}11.1.1. 用 iconv 命令来转换文本文件 {#用-iconv-命令来转换文本文件 .title}

</div>

</div>
:::::

::: {.tip style="margin-left: 0.5in; margin-right: 0.5in;"}
  ----------------------------- ------------------------------------------------------------------------------------------------------
   ![\[提示\]](images/tip.png)  提示
                                `iconv`{.literal}(1) 是 `libc6`{.literal} 软件包的一部分并且它可以在类 Unix 的系统上转换字符的编码。
  ----------------------------- ------------------------------------------------------------------------------------------------------

你能够通过如下的命令用 `iconv`{.literal}(1) 来转换文本文件的编码。

``` screen
$ iconv -f encoding1 -t encoding2 input.txt >output.txt
```

编码值是大小写不敏感的，且会在匹配时忽略"`-`{.literal}"和"`_`{.literal}"。可以使用"`iconv -l`{.literal}"命令检查支持的编码。

对于多字符的字符集，比如说，用于日文的 [JIS X 0213](https://zh.wikipedia.org/wiki/JIS_X_0213){.ulink}，或用于差不多所有语言的 [Universal Character Set (UCS, Unicode, ISO-10646-1)](https://zh.wikipedia.org/wiki/Universal_Character_Set){.ulink} , 有多种编码方案来序列化它们的字节数据。

::: itemizedlist
-   日文的 [EUC](https://zh.wikipedia.org/wiki/Extended_Unix_Code){.ulink} 和 [ISO/IEC 2022 (也被称为 JIS X 0202)](https://zh.wikipedia.org/wiki/ISO/IEC_2022){.ulink}

-   Unicode 的 [UTF-8](https://zh.wikipedia.org/wiki/UTF-8){.ulink}、[UTF-16/UCS-2](https://zh.wikipedia.org/wiki/UTF-16/UCS-2){.ulink} 和 [UTF-32/UCS-4](https://zh.wikipedia.org/wiki/UTF-32/UCS-4){.ulink} 编码
:::

对于以上这些，字符集和字符编码之间有着明显的区别。

对某些计算机厂家而言，[code page](https://zh.wikipedia.org/wiki/Code_page){.ulink} 是作为字符编码表的同义词来使用。

### []{#_converting_file_names_with_iconv}11.1.3. 使用 iconv 转换文件名 {#使用-iconv-转换文件名 .title}

</div>

</div>
:::::

这里是一个示例脚步，在同一目录下，将在老的操作系统系统下创建的文件名编码，转换为现代 UTF-8.

``` screen
#!/bin/sh
ENCDN=iso-8859-1
for x in *;
 do
 mv "$x" "$(echo "$x" | iconv -f $ENCDN -t utf-8)"
done
```

\"`$ENCDN`{.literal}\" 变量定义了在老的操作系统下，文件名使用的原始编码，见 [表 11.2 "编码值和用法的列表"](ch11.zh-cn.html#list-of-encoding-values "表 11.2. 编码值和用法的列表"){.xref}.

对于更加复杂的情况，请使用适当的编码作为 `mount`{.literal}(8) 的选项(参见 [第 8.1.3 节 "文件名编码"](ch08.zh-cn.html#_filename_encoding "8.1.3. 文件名编码"){.xref}) 来挂载包含有这样文件名的文件系统（比如说，磁盘上的一个分区），使用 \"`cp -a`{.literal}\" 命令来拷贝它的整个内容到另外一个使用 UTF-8 挂载的文件系统上。

### []{#_editors_with_auto_conversion}11.1.6. 带有自动转换功能的编辑器 {#带有自动转换功能的编辑器 .title}

</div>

</div>
:::::

像 `vim`{.literal} 这样的现代智能编辑器软件是相当聪明的并且能够处理任何编码系统以及任何文件格式。你应该在支持 UTF-8 编码的控制台上并在 UTF-8 环境下使用这些编辑器来获得最好的兼容性。

以 latin1（iso-8859-1）编码存储的旧西欧语言的 Unix 文本文件，"`u-file.txt`{.literal}"，能通过如下所示的用 `vim`{.literal} 轻易的编辑。

``` screen
$ vim u-file.txt
```

这是可能的因为 `vim`{.literal} 的文件编码自动检测机制先假定文件是 UTF-8 编码，如果失败了，则假定它是 latin1 编码。

以 latin2（iso-8859-2） 编码存储的旧波兰语的 Unix 文本文件，"`pu-file.txt`{.literal}"，能通过如下所示的用 `vim`{.literal} 编辑。

``` screen
$ vim '+e ++enc=latin2 pu-file.txt'
```

以 eucJP 编码存储的旧日语的 Unix 文本文件，\"`ju-file.txt`{.literal}\"，能通过如下所示的用 `vim`{.literal} 编辑。

``` screen
$ vim '+e ++enc=eucJP ju-file.txt'
```

以所谓的 shift-JIS 编码 (更确切的说法是：CP932) 存储的旧日语 MS-Windows 文本文件，\"`jw-file.txt`{.literal}\"，能通过如下所示的用 `vim`{.literal} 编辑。

``` screen
$ vim '+e ++enc=CP932 ++ff=dos jw-file.txt'
```

当一个文件用 vim 打开的时候带有 \"`++enc`{.literal}\" 和 \"`++ff`{.literal}\" 选项，在 Vim 命令行输入 \"`:w`{.literal}\" 命令会以原格式存储文件并且会覆盖原文件。你也可以在 Vim 命令行指定存储文件名及其格式，例如，\"`:w ++enc=utf8 new.txt`{.literal}\"。

请查阅 `vim`{.literal} 在线帮助中的 mbyte.txt，\"多字节文本支持\"和[表 11.2 "编码值和用法的列表"](ch11.zh-cn.html#list-of-encoding-values "表 11.2. 编码值和用法的列表"){.xref}来获得 \"`++enc`{.literal}\" 使用的本地值的信息。

`emacs`{.literal} 家族的程序能够实现同样的功能。

### []{#_basic_hints_for_xml}11.2.1. XML 的基本提示 {#xml-的基本提示 .title}

</div>

</div>
:::::

XML 文本看起来有些像 [HTML](https://zh.wikipedia.org/wiki/HTML){.ulink}.它能够使我们管理一个文档的多个格式。一个简单的 XML 系统是 `docbook-xsl`{.literal} 软件包，在这里使用。

每一个 XML 文件使用下面的标准 XML 声明开始。

``` screen
<?xml version="1.0" encoding="UTF-8"?>
```

XML 元素的基本语法是按下面的方式标记。

``` screen
<name attribute="value">content</name>
```

内容为空的 XML 元素，使用下面的短格式标记。

``` screen
<name attribute="value" />
```

上面列子中的 \"`attribute="value"`{.literal}\" 是可选的。

XML 里面的注释部分，是按下面的方式标记。

``` screen
<!-- comment -->
```

不同于增加标记，XML 至少要求使用预定义实体里的内容来转化下列字符。

  `&apos;`{.literal}   `'`{.literal} : 撇号
  `&lt;`{.literal}     `<`{.literal} : 小于号
  `&gt;`{.literal}     `>`{.literal} : 大于号
  `&amp;`{.literal}    `&`{.literal} :＆号

有许多在线资源存在。

::: itemizedlist

### []{#_printing_with_cups}11.4.4. 用 CUPS 打印 {#用-cups-打印 .title}

</div>

</div>
:::::

[Unix 通用打印系统（CUPS）](https://zh.wikipedia.org/wiki/Common_Unix_Printing_System){.ulink} 中的 `lp`{.literal}(1) 和 `lpr`{.literal}(1) 命令都提供了自定义打印数据的选项。

你可以使用下列命令中的一个来打印 3 份有装订页码的文件。

``` screen
$ lp -n 3 -o Collate=True filename
```

``` screen
$ lpr -#3 -o Collate=True filename
```

你能够通过 \"`-o number-up=2`{.literal}\", \"`-o page-set=even`{.literal}\", \"`-o page-set=odd`{.literal}\", \"`-o scaling=200`{.literal}\", \"`-o natural-scaling=200`{.literal}\" 等等打印机选项来进一步定制打印机操作，详细的文档参见[命令行打印和选项](http://localhost:631/help/options.html){.ulink}。

### []{#_mail_data_basics}11.5.1. 邮件数据基础 {#邮件数据基础 .title}

</div>

</div>
:::::

邮件 ([SMTP](https://zh.wikipedia.org/wiki/Simple_Mail_Transfer_Protocol){.ulink}) 数据需要被限制为 7 位数据序列。二进制数据和 8 位文本数据使用 [Multipurpose Internet Mail Extensions (MIME) 互联网多用途邮件扩展](https://zh.wikipedia.org/wiki/MIME){.ulink} 和选择的字符集编码到 7 位格式。(参见 [表 11.2 "编码值和用法的列表"](ch11.zh-cn.html#list-of-encoding-values "表 11.2. 编码值和用法的列表"){.xref})。

标准的邮件存储格式是 mbox，它是依据 [RFC2822 (由 RFC822 更新)](https://datatracker.ietf.org/doc/rfc2822/){.ulink} 来的格式. 参见 `mbox`{.literal}(5) (由 `mutt`{.literal} 软件包提供).

对于欧洲语言,由于没有什么 8 位字符，\"`Content-Transfer-Encoding: quoted-printable`{.literal}\" 加 ISO-8859-1 字符集通常被用于邮件。如果欧洲文本是被编码为 UTF-8,由于几乎全是 7 位数据，使用 \"`Content-Transfer-Encoding: quoted-printable`{.literal}\" 也是合适的。

对于日语，传统的 \"`Content-Type: text/plain; charset=ISO-2022-JP`{.literal}\" 通常被用于邮件来保持文本在 7 位。但是老的微软系统会在没有声明的情况下使用 Shift-JIS 来发送邮件。如果日语文本是用 UTF-8 编码, 由于含有许多 8 位数据，使用 [Base64](https://zh.wikipedia.org/wiki/Base64){.ulink} 是合适的。其它亚洲语言也是类似情形。

::: {.note style="margin-left: 0.5in; margin-right: 0.5in;"}
  ------------------------------ ----------------------------------------------------------------------------------------------------------------------------------
   ![\[注意\]](images/note.png)  注意
                                 如果你的非 Unix 邮件数据可以通过一个具备和 IMAP4 服务通讯的非 Debian 客户端访问，你可以通过运行你的 IMAP4 服务来将邮件数据移出。
  ------------------------------ ----------------------------------------------------------------------------------------------------------------------------------
:::

::: {.note style="margin-left: 0.5in; margin-right: 0.5in;"}
  ------------------------------ ---------------------------------------------------------------------------------------------------------------------------------------
   ![\[注意\]](images/note.png)  注意
                                 如果你使用其它邮件存储格式，第一步把它们移动到 mbox 格式比较好。像 `mutt`{.literal}(1) 这样多功能的客户端程序可以便捷的完成这类操作。
  ------------------------------ ---------------------------------------------------------------------------------------------------------------------------------------
:::

你可以使用 `procmail`{.literal}(1) 和 `formail`{.literal}(1) 把邮箱内容分开成每一封邮件.

每一封邮件能够使用来自 `mpack`{.literal} 软件包的 `munpack`{.literal}(1) 命令（或其它特异的工具）来获得 MIME 编码内容。

## []{#_graphic_data_tools}11.6. 图形数据工具 {#图形数据工具 .title}

</div>

</div>
:::::

实际上的数码相机的图像是[可交换的图像文件格式](https://zh.wikipedia.org/wiki/Exchangeable_image_file_format){.ulink}(EXIF)，这种格式是在 [JPEG](https://zh.wikipedia.org/wiki/JPEG){.ulink} 图像文件格式上添加一些元数据标签。它能够保存诸如日期、时间和相机设置的信息。

[Lempel-Ziv-Welch (LZW)无损数据压缩](https://zh.wikipedia.org/wiki/Lempel-Ziv-Welch){.ulink}专利已经过期了。使用 LZW 压缩方式的 [图形交互格式（GIF）](https://zh.wikipedia.org/wiki/Graphics_Interchange_Format){.ulink}工具现在可以在 Debian 系统上自由使用了。
