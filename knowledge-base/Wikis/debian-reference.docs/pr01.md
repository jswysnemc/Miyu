## []{#_what_is_debian}2. 什么是 Debian {#什么是-debian .title}

</div>

</div>
:::::

[Debian 项目](https://www.debian.org){.ulink}是一个由个人组成的团体，该团体的成员均把创建一个自由操作系统作为共同事业。Debian 的发布具有下文所列的特征。

::: itemizedlist
-   承诺软件自由：[Debian 社群契约和 Debian 自由软件指导方针（DFSG）](https://www.debian.org/social_contract){.ulink}

-   基于因特网上无酬劳的志愿者的工作发布：[https://www.debian.org](https://www.debian.org){.ulink}

-   大量预编译的高质量软件包

-   专注于稳定性和安全性，同时易于获取安全更新

-   在 `testing`{.literal} 版仓库中注重软件包最新版本的平滑升级

-   支持大量硬件架构
:::

Debian 系统中的自由软件来自[GNU](https://zh.wikipedia.org/wiki/GNU){.ulink}, [Linux](https://zh.wikipedia.org/wiki/Linux){.ulink}, [BSD](https://zh.wikipedia.org/wiki/Berkeley_Software_Distribution){.ulink}, [Ghostscript](https://zh.wikipedia.org/wiki/Ghostscript){.ulink}, [Common Unix Printing System](https://zh.wikipedia.org/wiki/Common_Unix_Printing_System){.ulink} , [Samba](https://zh.wikipedia.org/wiki/Samba_(software)){.ulink}, [GNOME](https://zh.wikipedia.org/wiki/GNOME){.ulink}, [KDE](https://zh.wikipedia.org/wiki/KDE){.ulink}, [Mozilla](https://zh.wikipedia.org/wiki/Mozilla){.ulink}, [LibreOffice](https://zh.wikipedia.org/wiki/LibreOffice){.ulink}, [Vim](https://zh.wikipedia.org/wiki/Vim_(text_editor)){.ulink}, [TeX](https://zh.wikipedia.org/wiki/TeX){.ulink}, [LaTeX](https://zh.wikipedia.org/wiki/LaTeX){.ulink}, [DocBook](https://zh.wikipedia.org/wiki/DocBook){.ulink}, [Perl](https://zh.wikipedia.org/wiki/Perl){.ulink}, [Python](https://zh.wikipedia.org/wiki/Python_(programming_language)){.ulink}, [Tcl](https://zh.wikipedia.org/wiki/Tcl){.ulink}, [Java](https://zh.wikipedia.org/wiki/Java_(programming_language)){.ulink}, [Ruby](https://zh.wikipedia.org/wiki/Ruby_(programming_language)){.ulink}, [PHP](https://zh.wikipedia.org/wiki/PHP){.ulink}, [Berkeley DB](https://zh.wikipedia.org/wiki/Berkeley_DB){.ulink}, [MariaDB](https://zh.wikipedia.org/wiki/MariaDB){.ulink}, [PostgreSQL](https://zh.wikipedia.org/wiki/PostgreSQL){.ulink}, ### []{#_conventions}3.3. 排版约定 {#排版约定 .title}

</div>

</div>
:::::

本文通过如下使用 `bash`{.literal}(1) shell 命令例子的简要方式来提供信息。

``` screen
# command-in-root-account
$ command-in-user-account
```

这些 shell 提示符区分了所使用的帐户。为了可读性，在本手册中 shell 提示符相关的环境变量被设置为"`PS1='\$'`{.literal}"和"`PS2=' '`{.literal}。这与实际安装的系统所使用的 shell 提示符很有可能会不同。

所有的命令示例都是运行在英语语言环境"`LANG=en_US.UTF8`{.literal}"下的。请不要期望命令示例中像 [*`command-in-root-account`{.literal}*]{.emphasis} 和 [*`command-in-user-account`{.literal}*]{.emphasis} 这样的占位符会被翻译。这么做是为了保持所有翻译示例都是最新的。

::: {.note style="margin-left: 0.5in; margin-right: 0.5in;"}
  ------------------------------ -------------------------------------------------------------------------------------------
   ![\[注意\]](images/note.png)  注意
                                 参见在 `bash`{.literal}(1) 中对环境变量 "`$PS1`{.literal}" 和 "`$PS2`{.literal}" 的解释。
  ------------------------------ -------------------------------------------------------------------------------------------
:::

要求系统管理员执行的[**操作**]{.strong}，须用祈使句描述，如"在 shell 中输入命令字符串后，键入 Enter 键。"

这些[**描述**]{.strong}列或类似信息在表格有一个[**名词短语**]{.strong}，后面会紧跟[软件包短描述](https://www.debian.org/doc/manuals/developers-reference/best-pkging-practices#bpp-desc-basics){.ulink}，这些短语会省略掉前面的\"a\"和\"the\"。它们也可以包含一个不定式短语作[**名词短语**]{.strong}，在联机帮助的短命令描述约定后面不带 \"to\"。有些人可能觉得这看起来有点可笑，这里故意保留这种风格是为了让文档看起来尽可能的简单。这些[**名词短语**]{.strong}在短命令描述约定里并不会采用首字母大写的方式。

::: {.note style="margin-left: 0.5in; margin-right: 0.5in;"}
  ------------------------------ ----------------------------------------------------------
   ![\[注意\]](images/note.png)  注意
                                 无论专有名词和命令名位于何处，保持其英文字母大小写不变。
  ------------------------------ ----------------------------------------------------------
:::

在文本段落中引用的[**命令片断**]{.strong}由双引号括起来的打印机字体进行标记，就像\"`aptitude safe-upgrade`{.literal}\"。

在文本段落中引用的来自配置文件的[**文本数据**]{.strong}由双引号括起来的打印机字体进行标记，就像\"`deb-src`{.literal}\"。

[**命令**]{.strong}和置于其后的圆括号内的手册页章节数（可选），由打字机字体进行标记，就像 `bash`{.literal}(1)。我们鼓励您这样通过输入以下命令来获得信息。

``` screen
$ man 1 bash
```

[**manpage**]{.strong} 会在打字机字体后面括号中显示 manpage 页章节号，如 `sources.list`{.literal}(5)。建议你通过键入以下命令来获取帮助信息。

``` screen
$ man 5 sources.list
```

[**info page**]{.strong} 页是由双引号之间的打字机字体来标注，如 `info make`{.literal}。建议你通过键入以下的命令来获取帮助信息。

``` screen
$ info make
```

[**文件名**]{.strong}将由双引号括起来的打印机字体进行标记，就像 \"`/etc/passwd`{.literal}\"。对于配置文件，你可以输入下列的命令来获取它的信息。

``` screen
$ sensible-pager "/etc/passwd"
```

[**目录名**]{.strong}将由双引号括起来的打印机字体进行标记，就像 \"`/etc/apt/`{.literal}\"。你可以输入下列的命令来浏览目录的内容。

``` screen
$ mc "/etc/apt/"
```

[**软件包名称**]{.strong}将由打印机字体进行标记，就像 `vim`{.literal}。你可以输入下列的命令来获取它的信息。

``` screen
$ dpkg -L vim
$ apt-cache show vim
$ aptitude show vim
```

一个[**文档**]{.strong}可能通过文件名来指示它的位置，文件名将由双引号括起来的打印机字体进行标记，就像\"`/usr/share/doc/base-passwd/users-and-groups.txt.gz`{.literal}\"和\"`/usr/share/doc/base-passwd/users-and-groups.html`{.literal} \"，或通过它的 [URL](https://zh.wikipedia.org/wiki/Uniform_Resource_Locator){.ulink}，就像 [https://www.debian.org](https://www.debian.org){.ulink}。你可以通过输入下列命令来阅读文档。

``` screen
$ zcat "/usr/share/doc/base-passwd/users-and-groups.txt.gz" | sensible-pager
$ sensible-browser "/usr/share/doc/base-passwd/users-and-groups.html"
$ sensible-browser "https://www.debian.org"
```

[**环境变量**]{.strong}将由双引号括起来的打印机字体进行标记，并带有 \"`$`{.literal}\" 前缀，就像 \"`$TERM`{.literal}\"。你可以输入下列命令来获取它的当前值。

``` screen
$ echo "$TERM"
```

### []{#_the_popcon}3.4. popcon 流行度 {#popcon-流行度 .title}

</div>

</div>
:::::

[popcon](https://popcon.debian.org/){.ulink} 数据被用来客观地衡量每个包的流行度。它的下载时间为 2026-03-07 15:18:04 UTC，包含了超过 215675 个二进制软件包和 27 个架构的全部 280230 份提交。

::: {.note style="margin-left: 0.5in; margin-right: 0.5in;"}
  ------------------------------ ------------------------------------------------------------------------------------------------------------------------------------
   ![\[注意\]](images/note.png)  注意
                                 请注意 `amd64`{.literal} `不稳定（unstable）`{.literal} 版的软件仓库中只包含当前 74874 软件包。popcon 数据包含许多旧系统安装报告。
  ------------------------------ ------------------------------------------------------------------------------------------------------------------------------------
:::

以"V:"开头表示"votes"的 popcon 数值计算方式为"1000 \* （当前运行在 PC 上的包的 popcon 提交）/ （总的 popcon 提交）"。

以"I:"开头表示"安装数"的 popcon 数值计算方式为"1000 \* （当前安装在 PC 上的包的 popcon 提交）/ （总的 popcon 提交）"。

::: {.note style="margin-left: 0.5in; margin-right: 0.5in;"}
  ------------------------------ -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
   ![\[注意\]](images/note.png)  注意
                                 流行度评比 popcon 数据不应视为对包的重要性的绝对度量。有许多因素可以影响统计数据。例如，参与流行度评比的某些系统可能有像"`/usr/bin`{.literal}"的目录，挂载的时候带"`noatime`{.literal}"选项以提升系统性能，这样的系统有效的禁用了"投票（vote）"功能。
  ------------------------------ -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

### []{#_the_package_size}3.5. 软件包大小 {#软件包大小 .title}

</div>

</div>
:::::

软件包的大小数据同样表明了对每个包的客观衡量。它基于"`apt-cache show`{.literal}"或"`aptitude show`{.literal}"命令（目前在`amd64`{.literal} 架构的`不稳定`{.literal}版）报告的"`安装大小：`{.literal}"。报告的大小单位是 KiB（[Kibibyte](https://zh.wikipedia.org/wiki/Kibibyte){.ulink}=表示 1024 Bytes 的单位）。

::: {.note style="margin-left: 0.5in; margin-right: 0.5in;"}
  ------------------------------ --------------------------------------------------------------------------------------------------------------------------------------------------------------
   ![\[注意\]](images/note.png)  注意
                                 包大小是一个小数值的包可能显示了这个在"`不稳定`{.literal}"版的包是一个虚拟包，它包含关于依赖关系的重要内容，会安装其他的包。虚拟包使能平稳过度或分割一个包。
  ------------------------------ --------------------------------------------------------------------------------------------------------------------------------------------------------------
:::

::: {.note style="margin-left: 0.5in; margin-right: 0.5in;"}
  ------------------------------ --------------------------------------------------------------------------------------------------------------------------------
   ![\[注意\]](images/note.png)  注意
                                 包大小后面跟着 \"(\*)\" 表明这个软件包在`不稳定`{.literal}版本中是缺失的同时使用了`实验性`{.literal}版本中的软件包大小来替代。
  ------------------------------ --------------------------------------------------------------------------------------------------------------------------------

## []{#_reminders_for_new_users}4. 一些对新使用者的提醒 {#一些对新使用者的提醒 .title}

</div>

</div>
:::::

这里给出对新用户的一些提醒信息：

::: itemizedlist
-   备份你的数据

    ::: itemizedlist
    -   参见 [第 10.2 节 "备份和恢复"](ch10.zh-cn.html#_backup_and_recovery "10.2. 备份和恢复"){.xref}。
    :::

-   妥善保存你的密码和安全信息

-   [KISS（保持简单而傻瓜式）](https://zh.wikipedia.org/wiki/KISS原则){.ulink}

    ::: itemizedlist
    -   不要在系统中过度设计（overengineering）
    :::

-   阅读你的日志文件

    ::: itemizedlist
    -   [**第一条**]{.strong} 错误信息才是最重要的
    :::

-   [RTFM（阅读手册与指导）](https://zh.wikipedia.org/wiki/RTFM){.ulink}

-   在问问题前，先在互联网上搜索

-   当不是必须要使用 root 的时候，就不要使用 root

-   不要胡乱折腾软件包管理系统

-   不要输入任何你不理解的命令

-   （在完全地检查过安全问题之前）不要随意修改文件权限

-   在[**测试**]{.strong}过你所做的修改之前不要关闭 root shell

-   总是保留一个替代的启动媒介([U 盘](https://zh.wikipedia.org/wiki/USB_flash_drive){.ulink}、[CD-ROM](https://en.wikipedia.org/wiki/CD-ROM){.ulink}, ...)
