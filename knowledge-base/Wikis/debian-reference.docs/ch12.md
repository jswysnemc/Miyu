## []{#_the_shell_script}12.1. Shell 脚本 {#shell-脚本 .title}

</div>

</div>
:::::

[Shell 脚本](https://zh.wikipedia.org/wiki/Shell_script){.ulink} 是指包含有下面格式的可执行的文本文件。

``` screen
#!/bin/sh
 ... command lines
```

第一行指明了读取并执行这个文件的 shell 解释器。

读懂 shell 脚本的[**最好**]{.strong} 办法是先理解类 Unix 系统是如何工作的。这里有一些 shell 编程的提示。看看"Shell 错误"（[https://www.greenend.org.uk/rjk/2001/04/shell.html](https://www.greenend.org.uk/rjk/2001/04/shell.html){.ulink}），可以从错误中学习。

不像 shell 交互模式（参见[第 1.5 节 "简单 shell 命令"](ch01.zh-cn.html#_the_simple_shell_command "1.5. 简单 shell 命令"){.xref} 和 [第 1.6 节 "类 Unix 的文本处理"](ch01.zh-cn.html#_unix_like_text_processing "1.6. 类 Unix 的文本处理"){.xref}），shell 脚本会频繁使用参数、条件和循环等。

### []{#_posix_shell_compatibility}12.1.1. POSIX shell 兼容性 {#posix-shell-兼容性 .title}

</div>

</div>
:::::

系统中的许多脚本都可以通过任意 [POSIX](https://zh.wikipedia.org/wiki/POSIX){.ulink} shell（参见 [表 1.13 "shell 程序列表"](ch01.zh-cn.html#list-of-shell-programs "表 1.13. shell 程序列表"){.xref}）来执行。

::: itemizedlist
-   默认的非交互 POSIX shell \"`/usr/bin/sh`{.literal}\" 是一个指向到 `/usr/bin/dash`{.literal} 的符号链接，并被许多系统程序使用。

-   默认的交互式 POSIX shell 是 `/usr/bin/bash`{.literal}。
:::

避免编写具有 [**bashisms**]{.strong}（bash 化）或者 [**zshisms**]{.strong}（zsh 化）语法的 shell 脚本，确保脚本在所有 POSIX shell 之间具有可移植性。你可以使用 `checkbashisms`{.literal}(1) 对其进行检查。

:::: table
[]{#listoftypicalbashisms}

**表 12.1. 典型 bashism 语法列表**

::: table-contents
  好的：POSIX                                   应该避免的：bashism
  --------------------------------------------- ----------------------------------------------
  `if [ "$foo" = "$bar" ] ; then …`{.literal}   `if [ "$foo" == "$bar" ] ; then …`{.literal}
  `diff -u file.c.orig file.c`{.literal}        `diff -u file.c{.orig,}`{.literal}
  `mkdir /foobar /foobaz`{.literal}             `mkdir /foo{bar,baz}`{.literal}
  `funcname() { … }`{.literal}                  `function funcname() { … }`{.literal}
  八进制格式：\"`\377`{.literal}\"              十六进制格式：\"`\xff`{.literal}\"
:::
::::

\

使用 \"`echo`{.literal}\" 命令的时候需要注意以下几个方面，因为根据内置 shell 和外部命令的不同，它的实现也有差别。

::: itemizedlist
-   避免使用除"`-n`{.literal}"以外的任何命令行选项。

-   避免在字符串中使用转义序列，因为根据 shell 不同，计算后的结果也不一样。
:::

::: {.note style="margin-left: 0.5in; margin-right: 0.5in;"}
  ------------------------------ ------------------------------------------------------------------------------
   ![\[注意\]](images/note.png)  注意
                                 尽管"`-n`{.literal}"选项并[**不是**]{.strong} POSIX 语法，但它已被广泛接受。
  ------------------------------ ------------------------------------------------------------------------------
:::

::: {.tip style="margin-left: 0.5in; margin-right: 0.5in;"}
  ----------------------------- ------------------------------------------------------------------------------------------------------
   ![\[提示\]](images/tip.png)  提示
                                如果你想要在输出字符串中嵌入转义序列，用 \"`printf`{.literal}\" 命令替代 \"`echo`{.literal}\" 命令。
  ----------------------------- ------------------------------------------------------------------------------------------------------

### []{#_shell_conditionals}12.1.3. Shell 条件语句 {#shell-条件语句 .title}

</div>

</div>
:::::

每个命令都会返回 [**退出状态**]{.strong}，这可以被条件语句使用。

::: itemizedlist
-   成功：0 (\"True\")

-   失败：非 0 (\"False\")
:::

::: {.note style="margin-left: 0.5in; margin-right: 0.5in;"}
  ------------------------------ --------------------------------------------------------------------------------------------
   ![\[注意\]](images/note.png)  注意
                                 \"0\" 在 shell 条件语句中的意思是 \"True\"，然而 \"0\" 在 C 条件语句中的含义为 \"False\"。
  ------------------------------ --------------------------------------------------------------------------------------------
:::

::: {.note style="margin-left: 0.5in; margin-right: 0.5in;"}
  ------------------------------ ----------------------------------------------------------------------------------------------------------------
   ![\[注意\]](images/note.png)  注意
                                 \"`[`{.literal}\" 跟 `test`{.literal} 命令是等价的，它评估到 \"`]`{.literal}\" 之间的参数来作为一个条件表达式.
  ------------------------------ ----------------------------------------------------------------------------------------------------------------
:::

如下所示是需要记忆的基础 [**条件语法**]{.strong}。

::: itemizedlist
-   \"[*`command`{.literal}*]{.emphasis}` && `{.literal}[*`if_success_run_this_command_too`{.literal}*]{.emphasis}` || true`{.literal}\"

-   \"[*`command`{.literal}*]{.emphasis}` || `{.literal}[*`if_not_success_run_this_command_too`{.literal}*]{.emphasis}` || true`{.literal}\"

-   如下所示是多行脚本片段
:::

``` screen
if [ conditional_expression ]; then
 if_success_run_this_command
else
 if_not_success_run_this_command
fi
```

这里末尾的"`|| true`{.literal}"是需要的，它可以保证这个 shell 脚本在不小心使用了"`-e`{.literal}"选项而被调用时不会在该行意外地退出。

[]{#listoffilecompartionalexpression}

**表 12.5. 在条件表达式中进行文件比较**

::: table-contents
  表达式                                                                              返回逻辑真所需的条件
  ----------------------------------------------------------------------------------- --------------------------------------------------------------------------------------
  `-e `{.literal}[*`file`{.literal}*]{.emphasis}                                      [*file*]{.emphasis} 存在
  `-d `{.literal}[*`file`{.literal}*]{.emphasis}                                      [*file*]{.emphasis} 存在并且是一个目录
  `-f `{.literal}[*`file`{.literal}*]{.emphasis}                                      [*file*]{.emphasis} 存在并且是一个普通文件
  `-w `{.literal}[*`file`{.literal}*]{.emphasis}                                      [*file*]{.emphasis} 存在并且可写
  `-x `{.literal}[*`file`{.literal}*]{.emphasis}                                      [*file*]{.emphasis} 存在并且可执行
  [*`file1`{.literal}*]{.emphasis}` -nt `{.literal}[*`file2`{.literal}*]{.emphasis}   [*file1*]{.emphasis} 是否比 [*file2*]{.emphasis} 新
  [*`file1`{.literal}*]{.emphasis}` -ot `{.literal}[*`file2`{.literal}*]{.emphasis}   [*file1*]{.emphasis} 是否比 [*file2*]{.emphasis} 旧
  [*`file1`{.literal}*]{.emphasis}` -ef `{.literal}[*`file2`{.literal}*]{.emphasis}   [*file1*]{.emphasis} 和 [*file2*]{.emphasis} 位于相同的设备上并且有相同的 inode 编号

### []{#_the_shell_command_line_processing_sequence}12.1.6. shell 命令行的处理顺序 {#shell-命令行的处理顺序 .title}

</div>

</div>
:::::

shell 大致以下列的顺序来处理一个脚本。

::: itemizedlist
-   shell 读取一行。

-   如果该行包含有`"…"`{.literal} 或 `'…'`{.literal}，shell 对该行各部分进行分组作为 [**一个标识（one token）**]{.strong} （译注：one token 是指 shell 识别的一个结构单元）.

-   shell 通过下列方式将行中的其它部分分隔进 [**标识（tokens）**]{.strong}。

    ::: itemizedlist
    -   空白字符：[*`空格`{.literal}*]{.emphasis}` `{.literal}[*`tab`{.literal}*]{.emphasis}` `{.literal}[*`换行符`{.literal}*]{.emphasis}

    -   元字符： `< > | ; & ( )`{.literal}
    :::

-   shell 会检查每一个不位于 `"…"`{.literal} 或 `'...'`{.literal} 的 token 中的 [**保留字**]{.strong} 来调整它的行为。

    ::: itemizedlist
    -   [**保留字**]{.strong}：`if then elif else fi for in while unless do done case esac`{.literal}
    :::

-   shell 展开不位于 `"…"`{.literal} 或 `'...'`{.literal} 中的 [**别名**]{.strong}。

-   shell 展开不位于 `"…"`{.literal} 或 `'...'`{.literal} 中的 [**波浪线**]{.strong}。

    ::: itemizedlist
    -   \"`~`{.literal}\" → 当前用户的家目录

    -   \"`~`{.literal}[*`user`{.literal}*]{.emphasis}\" → [*`user`{.literal}*]{.emphasis} 的家目录
    :::

-   shell 将不位于 `'...'`{.literal} 中的 [**变量**]{.strong} 展开为它的值。

    ::: itemizedlist
    -   [**变量**]{.strong}：\"`$PARAMETER`{.literal}\" 或 \"`${PARAMETER}`{.literal}\"
    :::

-   shell 展开不位于 `'...'`{.literal} 中的 [**命令替换**]{.strong}。

    ::: itemizedlist
    -   \"`$( command )`{.literal}\" → \"`command`{.literal}\" 的输出

    -   \"`` ` command ` ``{.literal}\" → \"`command`{.literal}\" 的输出
    :::

-   shell 将不位于 `"…"`{.literal} 或 `'...'`{.literal} 中的 [**glob 路径**]{.strong} 展开为匹配的文件名。

    ::: itemizedlist
    -   `*`{.literal} → 任何字符

    -   `?`{.literal} → 一个字符

    -   `[…]`{.literal} → 任何位于 \"`…`{.literal}\" 中的字符
    :::

-   shell 从下列几方面查找 [**命令**]{.strong} 并执行。

    ::: itemizedlist
    -   [**函数**]{.strong} 定义

    -   [**内建**]{.strong}命令

    -   "`$PATH`{.literal}" 中的[**可执行文件**]{.strong}
    :::

-   shell 前往下一行，并按照这个顺序从头再次进行处理。
:::

双引号中的单引号是没有效果的。

在 shell 中执行 "`set -x`{.literal}" 或使用 "`-x`{.literal}" 选项启动 shell 可以让 shell 显示出所有执行的命令。这对调试来说是非常方便的。

### []{#_perl_short_script_madness}12.2.4. Perl 短脚本的疯狂 {#perl-短脚本的疯狂 .title}

</div>

</div>
:::::

为了处理数据，`sh`{.literal} 需要生成子进程运行 `cut`{.literal}、`grep`{.literal}、 `sed`{.literal} 等，是慢的。从另外一个方面，`perl`{.literal} 有内部处理数据能力，是快的。所以 Debian 上的许多系统维护脚本使用 `perl`{.literal}。

让我们考虑下面一行 AWK 脚本片段和它在 Perl 中的等价物。

``` screen
awk '($2=="1957") { print $3 }' |
```

这等价于下列的任意一行。

``` screen
perl -ne '@f=split; if ($f[1] eq "1957") { print "$f[2]\n"}' |
```

``` screen
perl -ne 'if ((@f=split)[1] eq "1957") { print "$f[2]\n"}' |
```

``` screen
perl -ne '@f=split; print $f[2] if ( $f[1]==1957 )' |
```

``` screen
perl -lane 'print $F[2] if $F[1] eq "1957"' |
```

``` screen
perl -lane 'print$F[2]if$F[1]eq+1957' |
```

最后一个简直就是个迷。它用上了下面列出的这些 Perl 的特性。

::: itemizedlist
-   空格为可选项。

-   存在从数字到字符串的自动转换。

-   通过命令行选项： `perlrun`{.literal}(1) 的 Perl 执行技巧

-   Perl 特异变量：`perlvar`{.literal}(1)
:::

灵活性是 Perl 的强项。与此同时，这允许我们创建令人困惑和繁乱的代码。所以请小心。

### []{#_simple_c_program_gcc}12.3.2. 简单的 C 程序（gcc） {#简单的-c-程序gcc .title}

</div>

</div>
:::::

一个简单的例子 "`example.c`{.literal}" 可以通过如下方式和 "`libm`{.literal}" 库一起编译为可执行程序 "`run_example`{.literal}"。

``` screen
$ cat > example.c << EOF
#include <stdio.h>
#include <math.h>
#include <string.h>

int main(int argc, char **argv, char **envp){
        double x;
        char y[11];
        x=sqrt(argc+7.5);
        strncpy(y, argv[0], 10); /* prevent buffer overflow */
        y[10] = '\0'; /* fill to make sure string ends with '\0' */
        printf("%5i, %5.3f, %10s, %10s\n", argc, x, y, argv[1]);
        return 0;
}
EOF
$ gcc -Wall -g -o run_example example.c -lm
$ ./run_example
        1, 2.915, ./run_exam,     (null)
$ ./run_example 1234567890qwerty
        2, 3.082, ./run_exam, 1234567890qwerty
```

为了使用 `sqrt`{.literal}(3)，必须使用 "`-lm`{.literal}" 链接来自 `libc6`{.literal} 软件包的库 "`/usr/lib/libm.so`{.literal}"。实际的库文件位于 "`/lib/`{.literal}"，文件名为 "`libm.so.6`{.literal}"，它是指向 "`libm-2.7.so`{.literal}" 的一个链接。

请看一下输出文本的最后一段。即使指定了 "`%10s`{.literal}"，它依旧超出了 10 个字符。

使用没有边界检查的指针内存操作函数，比如 `sprintf`{.literal}(3) 和 `strcpy`{.literal}(3), 是不建议使用，是为防止缓存溢出泄露而导致上面的溢出问题。请使用 `snprintf`{.literal}(3) 和 `strncpy`{.literal}(3) 来替代.

你需要提供你自己的的 \"`main()`{.literal}\" 和 \"`yyerror()`{.literal}\".通常，Flex 创建的 \"`main()`{.literal}\" 调用 \"`yyparse()`{.literal}\"，它又调用了 \"`yylex()`{.literal}\".

这里是一个创建简单终端计算程序的例子。

让我们创建 `example.y`{.literal}:

``` screen
/* calculator source for bison */
%{
#include <stdio.h>
extern int yylex(void);
extern int yyerror(char *);
%}

/* declare tokens */
%token NUMBER
%token OP_ADD OP_SUB OP_MUL OP_RGT OP_LFT OP_EQU

%%
calc:
 | calc exp OP_EQU    { printf("Y: RESULT = %d\n", $2); }
 ;

exp: factor
 | exp OP_ADD factor  { $$ = $1 + $3; }
 | exp OP_SUB factor  { $$ = $1 - $3; }
 ;

factor: term
 | factor OP_MUL term { $$ = $1 * $3; }
 ;

term: NUMBER
 | OP_LFT exp OP_RGT  { $$ = $2; }
  ;
%%

int main(int argc, char **argv)
{
  yyparse();
}

int yyerror(char *s)
{
  fprintf(stderr, "error: '%s'\n", s);
}
```

让我们创建 `example.l`{.literal}:

``` screen
/* calculator source for flex */
%{
#include "example.tab.h"
%}

%%
[0-9]+ { printf("L: NUMBER = %s\n", yytext); yylval = atoi(yytext); return NUMBER; }
"+"    { printf("L: OP_ADD\n"); return OP_ADD; }
"-"    { printf("L: OP_SUB\n"); return OP_SUB; }
"*"    { printf("L: OP_MUL\n"); return OP_MUL; }
"("    { printf("L: OP_LFT\n"); return OP_LFT; }
")"    { printf("L: OP_RGT\n"); return OP_RGT; }
"="    { printf("L: OP_EQU\n"); return OP_EQU; }
"exit" { printf("L: exit\n");   return YYEOF; } /* YYEOF = 0 */
.      { /* ignore all other */ }
%%
```

L: NUMBER = 1
L: OP_RGT

</div>

Debian 系统在默认情况下，所有安装的二进制程序会被 stripped，因此大部分调试符号（debugging symbols）在通常的软件包里面会被移除。为了使用 `gdb`{.literal}(1) 调试 Debian 软件包， `*-dbgsym`{.literal} 软件包需要被安装。（例如，安装 `coreutils-dbgsym`{.literal}，用于调试`coreutils`{.literal}）源代码软件包和普通的二进制软件包一起自动生成 `*-dbgsym`{.literal} 软件包。那些调试软件包将被独立放在 [debian-debug](http://deb.debian.org/debian-debug/){.ulink} 档案库。更多信息请参阅 [Debian Wiki 文档](https://wiki.debian.org/DebugPackage){.ulink} 。

``` screen
$ mkdir /path/new ; cd /path/new
$ sudo apt-get update
$ sudo apt-get dist-upgrade
$ sudo apt-get install fakeroot devscripts build-essential
$ apt-get source package_name
$ cd package_name*
$ sudo apt-get build-dep ./
```

按需修改 bug。

``` screen
$ dch -i
```

如下所示编译并安装带有调试符号的软件包。

``` screen
$ export DEB_BUILD_OPTIONS="nostrip noopt"
$ debuild
$ cd ..
$ sudo debi package_name*.changes
```

你需要检查软件包的构建脚本并确保编译二进制的时候使用了 \"`CFLAGS=-g -Wall`{.literal}\" 选项。

### []{#_obtaining_backtrace}12.5.3. 获得栈帧 {#获得栈帧 .title}

</div>

</div>
:::::

当你碰到程序崩溃的时候，报告 bug 时附上栈帧信息是个不错的注意。

使用如下方案之一，可以通过 `gdb`{.literal}(1) 取得栈帧信息：

::: itemizedlist
-   在 GDB 中崩溃的方案：

    ::: itemizedlist
    -   从 GDB 运行程序。

    -   崩溃程序。

    -   在 GDB 提示符输入 \"`bt`{.literal}\"。
    :::

-   先奔溃的方案：

    ::: itemizedlist
    -   更新"[**/etc/security/limits.conf**]{.strong}"文件，包括下面内容：

        ``` screen
        * soft core unlimited
        ```

    -   shell 提示符下输入 \"`ulimit -c unlimited`{.literal}\"。

    -   从这个 shell 提示符运行程序。

    -   崩溃的程序产生一个 [core dump](https://zh.wikipedia.org/wiki/Core_dump){.ulink} 文件。

    -   加载 [core dump](https://zh.wikipedia.org/wiki/Core_dump){.ulink} 文件到 GDB，用 \"`gdb gdb ./program_binary core`{.literal}\" 。

    -   在 GDB 提示符输入 \"`bt`{.literal}\"。
    :::
:::

对于无限循环或者键盘冻结的情况，你可以通过按 `Ctrl-\`{.literal} 或 `Ctrl-C`{.literal} 或者执行 "`kill -ABRT `{.literal}[*`PID`{.literal}*]{.emphasis}" 强制奔溃程序。(参见 [第 9.4.12 节 "杀死一个进程"](ch09.zh-cn.html#_killing_a_process "9.4.12. 杀死一个进程"){.xref})

### []{#_make}12.6.1. make {#make .title}

</div>

</div>
:::::

[Make](https://zh.wikipedia.org/wiki/Make_(software)){.ulink} 是一个维护程序组的工具。一旦执行 `make`{.literal}(1)，`make`{.literal} 会读取规则文件 `Makefile`{.literal}，自从上次目标文件被修改后，如果目标文件依赖的相关文件发生了改变，那么就会更新目标文件，或者目标文件不存在，那么这些文件更新可能会同时发生。

规则文件的语法如下所示。

``` screen
target: [ prerequisites ... ]
 [TAB]  command1
 [TAB]  -command2 # ignore errors
 [TAB]  @command3 # suppress echoing
```

这里面的 \"`[TAB]`{.literal}\" 是一个 TAB 代码。每一行在进行变量替换以后会被 shell 解释。在行末使用 \"`\`{.literal}\" 来继续此脚本。使用 \"`$$`{.literal}\" 输入 \"`$`{.literal}\" 来获得 shell 脚本中的环境变量值。

目标跟相关文件也可以通过隐式规则给出，例如，如下所示。

``` screen
%.o: %.c header.h
```

在这里，目标包含了 \"`%`{.literal}\" 字符 (只是它们中确切的某一个）。\"`%`{.literal}\" 字符能够匹配实际的目标文件中任意一个非空的子串。相关文件同样使用 \"`%`{.literal}\" 来表明它们是怎样与目标文件建立联系的。

::: table-contents
  自动变量         值
  ---------------- ----------------------------------------------------------
  `$@`{.literal}   当前目标
  `$<`{.literal}   首个相关文件
  `$?`{.literal}   所有较新的相关文件
  `$^`{.literal}   所有相关文件
  `$*`{.literal}   目标模式中，\$\* 指代匹配符 \"`%`{.literal}\" 匹配的部分

### []{#_meson}12.6.3. Meson {#meson .title}

</div>

</div>
:::::

软件构建系统也在演进：

::: itemizedlist
-   [Autotools](https://en.wikipedia.org/wiki/GNU_Autotools){.ulink} 位于 [Make](https://zh.wikipedia.org/wiki/Make_(software)){.ulink} 之上，从 90 年代开始，便是可移植构建架构的事实标准。它是相当慢的。

-   [CMake](https://zh.wikipedia.org/wiki/Cmake){.ulink} 在 2000 年初始发布，显著地改善了速度，但是它源于建立在本质上慢的 [Make](https://zh.wikipedia.org/wiki/Make_(software)){.ulink} 之上。 (目前 [Ninja](https://zh.wikipedia.org/wiki/Ninja_(build_system)){.ulink} 能够作为它的后端。)

-   [Ninja](https://zh.wikipedia.org/wiki/Ninja_(build_system)){.ulink} 在 2012 年初始发布，是为了取代 Make，进一步改善构建速度，在设计上，它的输入文件由上层的构建系统来生成。

-   [Meson](https://zh.wikipedia.org/wiki/Meson_(software)){.ulink} 在 2013 年初始发布，是新的和流行的，并且是快速的和上层的构建系统，它使用 [Ninja](https://zh.wikipedia.org/wiki/Ninja_(build_system)){.ulink} 作为它的后端。
:::

参见在 \"[The Meson Build system](https://mesonbuild.com/){.ulink}\" 和 \"[The Ninja build system](https://ninja-build.org/manual.html){.ulink}\"里发现的文档。

## []{#_web}12.7. Web {#web .title}

</div>

</div>
:::::

基本的动态交互网页可由如下方法制作。

::: itemizedlist
-   呈现给浏览器用户的是 [HTML](https://zh.wikipedia.org/wiki/HTML){.ulink} 形式。

-   填充并点击表单条目将会从浏览器向 web 服务器发送带有编码参数的下列 [URL](https://zh.wikipedia.org/wiki/Uniform_Resource_Locator){.ulink} 字符串之一。

    ::: itemizedlist
    -   \"`https://www.foo.dom/cgi-bin/program.pl?VAR1=VAL1&VAR2=VAL2&VAR3=VAL3`{.literal}\"

    -   \"`https://www.foo.dom/cgi-bin/program.py?VAR1=VAL1&VAR2=VAL2&VAR3=VAL3`{.literal}\"

    -   \"`https://www.foo.dom/program.php?VAR1=VAL1&VAR2=VAL2&VAR3=VAL3`{.literal}\"
    :::

-   在 URL 里面 \"`%nn`{.literal}\" 是使用一个 16 进制字符的 `nn`{.literal} 值代替。

-   环境变量设置为: \"`QUERY_STRING="VAR1=VAL1 VAR2=VAL2 VAR3=VAL3"`{.literal}\".

-   Web服务器上的[CGI](https://zh.wikipedia.org/wiki/通用网关接口){.ulink}程序 (任何一个 \"`program.*`{.literal}\")在执行时，都会使用\"`$QUERY_STRING`{.literal}\"环境变量.

-   CGI 程序的 `stdout`{.literal}发送到浏览器，作为交互式的动态 web 页面展示。
:::

出于安全考虑，最好不要自己从头编写解析 CGI 参数的手艺. 在 Per l和 Python 中有现有的模块可以使用. [PHP](https://zh.wikipedia.org/wiki/PHP){.ulink} 中包含这些功能. 当需要客户端数据存储时, 可使用[HTTP cookies](https://zh.wikipedia.org/wiki/HTTP_cookie){.ulink} . 当需要处理客户端数据时, 通常使用 [Javascript](https://zh.wikipedia.org/wiki/JavaScript){.ulink}.

更多信息，参见 [通用网关接口](https://zh.wikipedia.org/wiki/通用网关接口){.ulink}, [Apache 软件基金会](https://zh.wikipedia.org/wiki/Apache软件基金会){.ulink}, 和 [JavaScript](https://zh.wikipedia.org/wiki/JavaScript){.ulink}.

直接在浏览器地址中输入 [https://www.google.com/search?hl=en&ie=UTF-8&q=CGI+tutorial](https://www.google.com/search?hl=en&ie=UTF-8&q=CGI+tutorial){.ulink} 就可以在 Google 上搜索 "CGI tutorial"。这是在 Google 服务器上查看 CGI 脚本运行的好方法。
