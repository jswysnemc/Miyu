### []{#_email_basics}6.2.1. 电子邮件基础 {#电子邮件基础 .title}

</div>

</div>
:::::

[电子邮件](https://zh.wikipedia.org/wiki/Email){.ulink} 由三个部分组成，消息的信封，邮件头及邮件正文。

::: itemizedlist
-   [SMTP](https://zh.wikipedia.org/wiki/Simple_Mail_Transfer_Protocol){.ulink} 用电子邮件信封上的 \"To\" 和 \"From\" 信息来投递邮件。(信封上的 \"From\" 信息也被叫做[退回地址](https://zh.wikipedia.org/wiki/Bounce_address){.ulink}, 例如 From\_ 等等）。

-   电子邮件头的\"To\" 和 \"From\" 信息，显示在 [电子邮件客户端](https://zh.wikipedia.org/wiki/Email_client){.ulink}上. (在大部分情况下，这些信息是跟电子邮件信封一致，但并不全是这样。）

-   覆盖邮件头和正文数据的电子邮件消息格式被 [多用途互联网邮件扩展 (MIME)](https://zh.wikipedia.org/wiki/MIME){.ulink} 扩展，从纯文本的 ASCII 到其它字符编码，包括作为附件的音频、视频、图像和应用程序。
:::

功能全面的基于 [电子邮件客户端](https://zh.wikipedia.org/wiki/Email_client){.ulink}的 GUI 程序使用基于 GUI 的直观的配置，提供下列所有功能。

::: itemizedlist
-   为了处理正文数据类型及其编码，它创建和使用[多用途互联网邮件扩展 (MIME)](https://zh.wikipedia.org/wiki/MIME){.ulink}来解释邮件标头和邮件正文。

-   它使用旧的 [基础访问认证](https://en.wikipedia.org/wiki/Basic_access_authentication){.ulink} 或现代的 [OAuth 2.0](https://zh.wikipedia.org/wiki/OAuth){.ulink}向 ISP（互联网服务提供商）的 SMTP 和 IMAP 服务器认证它自己。 (对于 [OAuth 2.0](https://zh.wikipedia.org/wiki/OAuth){.ulink},通过桌面环境设置来设置它，例如，\"Settings\" -\> \"Online Accounts\".)

-   它发送消息到 ISP 的智能主机的 SMTP 服务监听的消息递交端口（587）。

-   从 TLS/IMAP4 端口（993）接收存储在 ISP 的服务器上的消息。

-   它能够通过他们的属性过滤邮件。

-   它能够提供额外的功能：联系人、日历、任务、备忘录。
:::

### []{#_modern_mail_service_limitation}6.2.2. 现代邮件服务限制 {#现代邮件服务限制 .title}

</div>

</div>
:::::

现代邮件服务器有一些限制来最小化暴露滥用（不希望和未被要求的电子邮件）问题。

::: itemizedlist
-   在消费者级的网络上运行 SMTP 服务器来直接可靠的发送邮件到远端主机是不现实的。

-   一个邮件能够被任何主机静悄悄的拒绝，即使路由到了目的地，除非它尽可能看起来是经过认证的。

-   期望单个智能主机可靠的发送不相关的源邮件地址到远程主机，这是不现实的。
:::

这是因为：

::: itemizedlist
-   从消费者级网络提供的主机连接到互联网的 SMTP 端口（25）已经被封锁了。

-   从互联网的 SMTP 端口（25）连接到消费者级网络提供的主机已经被封锁了。

-   从消费者级网络提供的主机发出到互联网的消息，只能够通过消息递交端口（587）发送。

-   像[域名密钥识别邮件 (DKIM)](https://zh.wikipedia.org/wiki/DomainKeys_Identified_Mail){.ulink}、[发信者策略框架 (SPF)](https://zh.wikipedia.org/wiki/Sender_Policy_Framework){.ulink} 和 [基于域名的消息认证、报告和反应（DMARC）](https://zh.wikipedia.org/wiki/DMARC){.ulink} 这样的[反垃圾邮件技术](https://zh.wikipedia.org/wiki/反垃圾邮件技术){.ulink}广泛用于[电子邮件过滤](https://zh.wikipedia.org/wiki/Email_filtering){.ulink}。

-   [域名密钥识别邮件](https://zh.wikipedia.org/wiki/DomainKeys_Identified_Mail){.ulink}服务可能会用于你的通过 smarthost 的电子邮件发送。

-   智能主机可以在邮件头重写源电子邮件地址为你的邮件账户，来阻止电子邮件欺诈。
:::

### []{#_historic_mail_service_expectation}6.2.3. 历史邮件服务端期望 {#历史邮件服务端期望 .title}

</div>

</div>
:::::

一些在 Debian 上的程序，它们默认期望访问 `/usr/sbin/sendmail`{.literal} 命令来发送邮件，或者从一个个性化设置的 UNIX 系统邮件服务器来发送邮件，实现历史的功能：

::: itemizedlist
-   邮件是由纯文本文件创建。

-   邮件是由 `/usr/sbin/sendmail`{.literal} 命令处理。

-   对于目的地址为同一主机，`/usr/sbin/sendmail`{.literal} 命令进行邮件的本地分发，将邮件附在 `/var/mail/$username`{.literal} 文件后。

    ::: itemizedlist
    -   期望这个特征的命令： `apt-listchanges`{.literal}, `cron`{.literal}, `at`{.literal}, \...
    :::

-   对于目的地址在远程主机，`/usr/sbin/sendmail`{.literal} 命令远程传输邮件到目的主机，使用 SMTP 发现 DNS MX 记录。

    ::: itemizedlist
    -   期望这个特征的命令：`popcon`{.literal}, `reportbug`{.literal}, `bts`{.literal}, \...
    :::
:::

### []{#_mail_transport_agent_mta}6.2.4. 邮件传输代理 (MTA) {#邮件传输代理-mta .title}

</div>

</div>
:::::

在 Debian 12 Bookworm 后，在没有 [mail transfer agent (MTA)](https://zh.wikipedia.org/wiki/Message_transfer_agent){.ulink} 程序的情况下，Debian 移动工作站可以基于 [电子邮件客户端](https://zh.wikipedia.org/wiki/Email_client){.ulink}，配置为全功能的 GUI （图像用户界面）。

以往的 Debian 会安装某个 MTA 程序来支持期望 `/usr/sbin/sendmail`{.literal} 命令的程序。移动工作站上这样的 MTA 必须和 [第 6.2.2 节 "现代邮件服务限制"](ch06.zh-cn.html#_modern_mail_service_limitation "6.2.2. 现代邮件服务限制"){.xref} [第 6.2.3 节 "历史邮件服务端期望"](ch06.zh-cn.html#_historic_mail_service_expectation "6.2.3. 历史邮件服务端期望"){.xref} 协同工作。

对于移动工作站，典型的 MTA 选择是 `exim4-daemon-light`{.literal} 或 `postfix`{.literal}，并选择类似这样的安装选项："Mail sent by smarthost; received via SMTP or fetchmail"。这些是轻量 MTA 和 \"`/etc/aliases`{.literal}\" 匹配。

::: {.tip style="margin-left: 0.5in; margin-right: 0.5in;"}
  ----------------------------- ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
   ![\[提示\]](images/tip.png)  提示
                                配置 `exim4`{.literal} 来发送互联网邮件，多个源电子邮件地址使用多个相应的智能主机，这是不寻常的。如果一些程序需要这样的能力，使用 `msmtp`{.literal} 来设置他们，它比较容易来设置多个源电子邮件地址。然后给主 MTA 仅仅保留单个电子邮件地址。
  ----------------------------- ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

#### []{#_the_configuration_of_exim4}6.2.4.1. exim4 的配置 {#exim4-的配置 .title}

</div>

</div>
:::::

对于那些通过 smarthost 的网络邮件，你应该按如下所示的 (重新) 配置 `exim4-*`{.literal} 软件包。

``` screen
$ sudo systemctl stop exim4
$ sudo dpkg-reconfigure exim4-config
```

配置 \"General type of mail configuration\" 时，选择 \"mail sent by smarthost; received via SMTP or fetchmail\"。

设置 \"System mail name:\" 为默认的 FQDN (参见[第 5.1.1 节 "主机名解析"](ch05.zh-cn.html#_the_hostname_resolution "5.1.1. 主机名解析"){.xref})。

设置 \"IP-addresses to listen on for incoming SMTP connections:\" 为默认的 \"127.0.0.1; ::1\"。

\"Other destinations for which mail is accepted:\" 选项留空。

\"Machines to relay mail for:\" 选项留空。

设置 \"IP address or host name of the outgoing smarthost:\" 为 \"smtp.hostname.dom:587\"。

设置 \"Hide local mail name in outgoing mail?\" 选项为 \"No\"。(或者像[第 6.2.4.3 节 "邮件地址配置"](ch06.zh-cn.html#_the_mail_address_configuration "6.2.4.3. 邮件地址配置"){.xref}描述的那样使用 `/etc/email-addresses`{.literal}\" 代替)

选择如下所示的其中一个来回答 \"Keep number of DNS-queries minimal (Dial-on-Demand)?\"。

::: itemizedlist
-   \"No\" 如果启动的时候，系统就连上了互联网。

-   \"Yes\" 如果启动的时候，系统[**没有**]{.strong}连上互联网。
:::

设置 \"Delivery method for local mail:\" 选项为 \"mbox format in /var/mail/\"。

\"Split configuration into small files?:\" 选项设为 \"Yes\"。

通过修改 \"`/etc/exim4/passwd.client`{.literal}\" 文件，来创建用于 smarthost 的密码条目。

``` screen
$ sudo vim /etc/exim4/passwd.client
 ...
$ cat /etc/exim4/passwd.client
^smtp.*\.hostname\.dom:username@hostname.dom:password
```

配置 `exim4`{.literal}(8)，在 \"`/etc/default/exim4`{.literal}\" 文件中写入 \"`QUEUERUNNER='queueonly'`{.literal}\"，\"`QUEUERUNNER='nodaemon'`{.literal}\" 等等，来最小化系统资源使用。（可选的）

通过如下所示的启动 `exim4`{.literal}。

``` screen
$ sudo systemctl start exim4
```

\"`/etc/exim4/passwd.client`{.literal}\" 文件中的主机名不应该是别名，你应该按如下所示的检查真正的主机名。

``` screen
$ host smtp.hostname.dom
smtp.hostname.dom is an alias for smtp99.hostname.dom.
smtp99.hostname.dom has address 123.234.123.89
```

我在 \"`/etc/exim4/passwd.client`{.literal}\" 文件中使用正则表达式来绕过别名问题。即使 ISP 更改了别名所指向的主机名，SMTP AUTH 还是可能工作的。

你能够通过如下所示的手动更新 `exim4`{.literal} 配置:

::: itemizedlist
-   更新 \"`/etc/exim4/`{.literal}\" 目录下的 `exim4`{.literal} 配置文件。

    ::: itemizedlist
    -   创建 \"`/etc/exim4/exim4.conf.localmacros`{.literal}\" 来设置宏命令和修改 \"`/etc/exim4/exim4.conf.template`{.literal}\" 文件。(没有分割的配置）

    -   在 "`/etc/exim4/exim4.conf.d`{.literal}\" 子目录中创建新文件或编辑已存在的文件。(分割的配置）
    :::

-   运行 \"`systemctl reload exim4`{.literal}\".
:::

::: {.caution style="margin-left: 0.5in; margin-right: 0.5in;"}
  --------------------------------- --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
   ![\[小心\]](images/caution.png)  小心
                                    如果 debconf 询问 \"Keep number of DNS-queries minimal (Dial-on-Demand)?\" 这个问题时，选择 了 \"No\" (默认值)，那么启动 `exim4`{.literal} 会花很长时间并且系统在启动的时候[**不会**]{.strong}连接到互联网。
  --------------------------------- --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
:::

请阅读 \"`/usr/share/doc/exim4-base/README.Debian.gz`{.literal}\" 官方指导和 `update-exim4.conf`{.literal}(8)。

::: {.warning style="margin-left: 0.5in; margin-right: 0.5in;"}
  --------------------------------- -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
   ![\[警告\]](images/warning.png)  警告
                                    从所有的实践考虑，使用带 [STARTTLS](https://zh.wikipedia.org/wiki/STARTTLS){.ulink} 的 [SMTP](https://zh.wikipedia.org/wiki/Simple_Mail_Transfer_Protocol){.ulink} 端口 587，或者 [SMTPS](https://zh.wikipedia.org/wiki/SMTPS){.ulink} (SSL 之上的 SMTP ) 端口 465, 代替纯 SMTP 端口 25。

### []{#_connecting_without_remote_passwords}6.3.3. 免密码远程连接 {#免密码远程连接 .title}

</div>

</div>
:::::

使用\"`PubkeyAuthentication`{.literal}\" (SSH-2 协议)，人们可以避免记住远程系统的密码.

### []{#_setting_up_ssh_agent}6.3.5. 建立 ssh 代理 {#建立-ssh-代理 .title}

</div>

</div>
:::::

用密码来保护你的 SSH 认证私钥是安全的。如果密码没有设置，使用 \"`ssh-keygen -p`{.literal}\" 来设置。

把你的公钥 (比如：\"`~/.ssh/id_rsa.pub`{.literal}\") 放到远程主机的\"`~/.ssh/authorized_keys`{.literal}\"，这个远程主机使用上面描述的基于密码的连接方式。

``` screen
$ ssh-agent bash
$ ssh-add ~/.ssh/id_rsa
Enter passphrase for /home/username/.ssh/id_rsa:
Identity added: /home/username/.ssh/id_rsa (/home/username/.ssh/id_rsa)
```

从这里执行接下来的命令，就不再需要密码。

``` screen
$ scp foo username@remote.host:foo
```

按 \^D 来终结 ssh 代理会话。

对于 X 服务端，通常的 Debian 启动脚本会作为父进程执行 `ssh-agent`{.literal}。所以你只需要执行一次 `ssh-add`{.literal}。进一步的信息，请阅读 `ssh-agent`{.literal}(1) 和 `ssh-add`{.literal}(1).

### []{#_troubleshooting_ssh}6.3.9. SSH 故障排查 {#ssh-故障排查 .title}

</div>

</div>
:::::

如果你遇到问题，检查配置文件的权限并用 \"`-v`{.literal}\" 选项运行 `ssh`{.literal}。

如果你是 root 账户，并有使用防火墙，使用 \"`-p`{.literal}\" 选项; 这可以避免使用1 --- 1023 之间的服务端口.

如果 `ssh`{.literal} 连接到远程站点突然停止工作，这也许是系统管理员胡乱操作的结果，可能是在系统维护时改变了 \"`host_key`{.literal}\". 在确认这个情况后，并且没有人试图用聪明的黑客技术来篡改远程主机，你可以在本机 \"`~/.ssh/known_hosts`{.literal}\" 里删除 \"`host_key`{.literal}\" 条目来重新获得连接。

## []{#_the_print_server_and_utilities}6.4. 打印服务和工具 {#打印服务和工具 .title}

</div>

</div>
:::::

在老的类 Unix 系统中，BSD [Line printer daemon(lpd) 行打印机后台守护](https://zh.wikipedia.org/wiki/Line_Printer_Daemon_protocol){.ulink} 曾经是标准。传统的自由软件的标准打印输出格式是 [PostScript (PS)](https://zh.wikipedia.org/wiki/PostScript){.ulink}。为了能够打印到非 PostScript 打印机，需要将一些过滤器系统和 [Ghostscript](https://zh.wikipedia.org/wiki/Ghostscript){.ulink} 一道使用。参见 [第 11.4.1 节 "Ghostscript"](ch11.zh-cn.html#_ghostscript "11.4.1. Ghostscript"){.xref}。

## []{#_the_diagnosis_of_the_system_daemons}6.7. 系统后台守护进程（daemon）诊断 {#系统后台守护进程daemon诊断 .title}

</div>

</div>
:::::

`telnet`{.literal} 程序能够手工连接到系统后台守护进程（daemon），并进行诊断。

测试纯 [POP3](https://zh.wikipedia.org/wiki/Post_Office_Protocol){.ulink} 服务，尝试用下面的操作

``` screen
$ telnet mail.ispname.net pop3
```

部分 ISP 提供 [TLS](https://zh.wikipedia.org/wiki/Transport_Layer_Security){.ulink}/SSL 加密的[POP3](https://zh.wikipedia.org/wiki/Post_Office_Protocol){.ulink} 服务，为了测试它，你需要用到 `telnet-ssl`{.literal} 包里支持 TLS/SSL 的 `telnet`{.literal} 客户端，或 `openssl`{.literal} 软件包。

``` screen
$ telnet -z ssl pop.gmail.com 995
```

``` screen
$ openssl s_client -connect pop.gmail.com:995
```

下面的 [RFCs](https://www.ietf.org/standards/rfcs/){.ulink} 提供每一个系统后台守护进程（daemon）所需要的知识。
