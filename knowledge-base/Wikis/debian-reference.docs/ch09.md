### []{#_recording_the_shell_activities_cleanly}9.1.1. 清晰的记录 shell 活动 {#清晰的记录-shell-活动 .title}

</div>

</div>
:::::

简单地使用 `script`{.literal}(1) （参见 [第 1.4.9 节 "记录 shell 活动"](ch01.zh-cn.html#_recording_the_shell_activities "1.4.9. 记录 shell 活动"){.xref}）记录 shell 活动会产生一个有控制字符的文件。这些控制字符可以按下面的方式，使用 `col`{.literal}(1) 去掉。

``` screen
$ script
Script started, file is typescript
```

做些操作......按 `Ctrl-D`{.literal} 退出 `script`{.literal}.

``` screen
$ col -bx < typescript > cleanedfile
$ vim cleanedfile
```

有替代的方式来记录 shell 活动：

::: itemizedlist
-   使用 `tee`{.literal} (在 initramfs 的启动过程中可用)：

    ``` screen
    $ sh -i 2>&1 | tee typescript
    ```

-   使用 `gnome-terminal`{.literal} 增加行缓冲，用滚动条查看。

-   使用 `screen`{.literal} 和 \"`^A H`{.literal}\" (参见 [第 9.1.2 节 "screen 程序"](ch09.zh-cn.html#_the_screen_program "9.1.2. screen 程序"){.xref})来进行控制台记录。

-   使用 `vim`{.literal} 输入 \"`:terminal`{.literal}\" 进入终端模式。使用 \"`Ctrl-W N`{.literal}\" 从终端模式退出到普通模式。使用 \"`:w typescript`{.literal}\" 将缓存写到一个文件。

-   使用 `emacs`{.literal} 和 \"`M-x shell`{.literal}\", \"`M-x eshell`{.literal}\", 或 \"`M-x term`{.literal}\" 来进入记录控制台。使用 \"`C-x C-w`{.literal}\" 将缓存写到文件。
:::

### []{#_the_screen_program}9.1.2. screen 程序 {#screen-程序 .title}

</div>

</div>
:::::

`screen`{.literal}(1) 不但允许一个终端窗口运行多个进程，还允许[**远程 shell 进程支持中断的连接**]{.strong}.这里是一个典型的 `screen`{.literal}(1) 使用场景.

::: orderedlist
1.  登录到一个远程机器。

2.  在单个控制台上启动 `screen`{.literal}。

3.  使用 `^A c`{.literal} (\"Control-A\" 接着 \"c\")在 `screen`{.literal} 中创建的窗口执行多个程序.

4.  按 `^A n`{.literal} (\"Control-A\" 接着\"n\")来在多个 `screen`{.literal} 窗口间转换.

5.  突然，你需要离开你的终端，但你不想丢掉正在做的工作，而这些工作需要连接来保持。

6.  你可以通过任何方式[**分离**]{.strong} `screen`{.literal} 会话。

    ::: itemizedlist
    -   残忍地拔掉你的网络连接

    -   输入 `^A d`{.literal} (\"Control-A\" 接着 \"d\") 并手工从远程连接退出登录

    -   输入 `^A DD`{.literal} (\"Control-A\" 接着 \"DD\") 分离 `screen`{.literal} 并退出登录
    :::

7.  你重新登录到同一个远处主机（即使从不同的终端）。

8.  使用 \"`screen -r`{.literal}\" 启动 `screen`{.literal}.

9.  `screen`{.literal} 魔术般的[**重新附上**]{.strong}先前所有的 `screen`{.literal} 窗口和所有在活动运行的程序.
:::

::: {.tip style="margin-left: 0.5in; margin-right: 0.5in;"}
  ----------------------------- ------------------------------------------------------------------------------------------------------------------------------------------------------
   ![\[提示\]](images/tip.png)  提示
                                对于拨号或者按包计费的网络连接，你可以通过 `screen`{.literal} 节省连接费用，应为你可以在断开连接时让一个进程继续运行，当你稍后再次连接时重新附上它。
  ----------------------------- ------------------------------------------------------------------------------------------------------------------------------------------------------
:::

`vim`{.literal} 的行为能够被显著的改变，通过 `Ex`{.literal}-模式 的命令，启用它的内部特性，比如 \"`set ...`{.literal}\" 来设置 vim 选项。

这些 `Ex`{.literal}-模式 的命令，能够在用户的 vimrc 文件里面包括，传统的 \"`~/.vimrc`{.literal}\" 或 git 友好的 \"`~/.vim/vimrc`{.literal}\"。这里有一个非常简单的例子[^\[2\]^](#ftn.id15400){#id15400 .footnote}：

``` screen
""" Generic baseline Vim and Neovim configuration (~/.vimrc)
"""   - For NeoVim, use "nvim -u ~/.vimrc [filename]"
"""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""
let mapleader = ' '             " :h mapleader
""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""
set nocompatible                " :h 'cp -- sensible (n)vim mode
syntax on                       " :h :syn-on
filetype plugin indent on       " :h :filetype-overview
set encoding=utf-8              " :h 'enc (default: latin1) -- sensible encoding
""" current vim option value can be verified by :set encoding?
set backspace=indent,eol,start  " :h 'bs (default: nobs) -- sensible BS
set statusline=%<%f%m%r%h%w%=%y[U+%04B]%2l/%2L=%P,%2c%V
set listchars=eol:¶,tab:⇄\ ,extends:↦,precedes:↤,nbsp:␣
set viminfo=!,'100,<5000,s100,h " :h 'vi -- bigger copy buffer etc.
""" Pick "colorscheme" from blue darkblue default delek desert elflord evening
""" habamax industry koehler lunaperche morning murphy pablo peachpuff quiet ron
""" shine slate torte zellner
colorscheme industry
""" don't pick "colorscheme" as "default" which may kill SpellUnderline settings
set scrolloff=5                 " :h 'scr -- show 5 lines around cursor
set laststatus=2                " :h 'ls (default 1)  k
""" boolean options can be unset by prefixing "no"
set ignorecase                  " :h 'ic
set smartcase                   " :h 'scs
set autoindent                  " :h 'ai
set smartindent                 " :h 'si
set nowrap                      " :h 'wrap
"set list                        " :h 'list (default nolist)
set noerrorbells                " :h 'eb
set novisualbell                " :h 'vb
set t_vb=                       " :h 't_vb -- termcap visual bell
set spell                       " :h 'spell
set spelllang=en_us,cjk         " :h 'spl -- english spell, ignore CJK
set clipboard=unnamedplus       " :h 'cb -- cut/copy/paste with other app
set hidden                      " :h 'hid
set autowrite                   " :h 'aw
set timeoutlen=300              " :h 'tm
```

`vim`{.literal} 的键盘映射能够在用户的 vimrc 文件里面修改。例如：

                                    没有非常好的理由，请不要尝试改变默认的键绑定。
  --------------------------------- ------------------------------------------------
:::

``` screen
""" Popular mappings (imitating LazyVim etc.)
""" Window moves without using CTRL-W which is dangerous in INSERT mode
nnoremap <C-H> <C-W>h
nnoremap <C-J> <C-W>j
nnoremap <C-K> <C-W>k
silent! nnoremap <C-L> <C-W>l
""" Window resize
nnoremap <C-LEFT> <CMD>vertical resize -2<CR>
nnoremap <C-DOWN> <CMD>resize -2<CR>
nnoremap <C-UP> <CMD>resize +2<CR>
nnoremap <C-RIGHT> <CMD>vertical resize +2<CR>
""" Clear hlsearch with <ESC> (<C-L> is mapped as above)
nnoremap <ESC> <CMD>noh<CR><ESC>
inoremap <ESC> <CMD>noh<CR><ESC>
""" center after jump next
nnoremap n nzz
nnoremap N Nzz
""" fast "jk" to get out of INSERT mode (<ESC>)
inoremap  jk <CMD>noh<CR><ESC>
""" fast "<ESC><ESC>" to get out of TERM mode (CTRL-\ CTRL-N)
tnoremap <ESC><ESC> <C-\><C-N>
""" fast "jk" to get out of TERM mode (CTRL-\ CTRL-N)
tnoremap jk <C-\><C-N>
""" previous/next trouble/quickfix item
nnoremap [q <CMD>cprevious<CR>
nnoremap ]q <CMD>cnext<CR>
""" buffers
nnoremap <S-H> <CMD>bprevious<CR>
nnoremap <S-L> <CMD>bnext<CR>
nnoremap [b <CMD>bprevious<CR>
nnoremap ]b <CMD>bnext<CR>
""" Add undo break-points
inoremap  , ,<C-G>u
inoremap  . .<C-G>u
inoremap  ; ;<C-G>u
""" save file
inoremap <C-S> <CMD>w<CR><ESC>
xnoremap <C-S> <CMD>w<CR><ESC>
nnoremap <C-S> <CMD>w<CR><ESC>
snoremap <C-S> <CMD>w<CR><ESC>
""" better indenting
vnoremap < <gv
vnoremap > >gv
""" terminal (Somehow under Linux, <C-/> becomes <C-_> in Vim)
nnoremap <C-_> <CMD>terminal<CR>
"nnoremap <C-/> <CMD>terminal<CR>
""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""
if ! has('nvim')
""" Toggle paste mode with <SPACE>p for Vim (no need for Nvim)
set pastetoggle=<leader>p
""" nvim default mappings for Vim.  See :h default-mappings in nvim
""" copy to EOL (no delete) like D for d
noremap Y y$
""" sets a new undo point before deleting
inoremap <C-U> <C-G>u<C-U>
inoremap <C-W> <C-G>u<C-W>
""" <C-L> is re-purposed as above
""" execute the previous macro recorded with Q
nnoremap Q @@
""" repeat last substitute and *KEEP* flags
nnoremap & :&&<CR>
""" search visual selected string for visual mode
xnoremap * y/\V<C-R>"<CR>
xnoremap # y?\V<C-R>"<CR>
endif
```

其它杂项配置能够在用户的 vimrc 文件里面修改。例如：

``` screen
""" Use faster 'rg' (ripgrep package) for :grep
if executable("rg")
  set grepprg=rg\ --vimgrep\ --smart-case
  set grepformat=%f:%l:%c:%m
endif
""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""
""" Retain last cursor position :h '"
augroup RetainLastCursorPosition
  autocmd!
  autocmd BufReadPost *
    \ if line("'\"") > 0 && line ("'\"") <= line("$") |
    \   exe "normal! g'\"" |
    \ endif
augroup END
""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""
""" Force to use underline for spell check results
augroup SpellUnderline
  autocmd!
  autocmd ColorScheme * highlight SpellBad term=Underline gui=Undercurl
  autocmd ColorScheme * highlight SpellCap term=Underline gui=Undercurl
  autocmd ColorScheme * highlight SpellLocal term=Underline gui=Undercurl
  autocmd ColorScheme * highlight SpellRare term=Underline gui=Undercurl
augroup END
""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""""
""" highlight tailing spaces except when typing as red (set after colorscheme)
highlight TailingWhitespaces ctermbg=red guibg=red
""" \s\+     1 or more whitespace character: <Space> and <Tab>
""" \%#\@<!  Matches with zero width if the cursor position does NOT match.
match TailingWhitespaces /\s\+\%#\@<!$/
```

### []{#_colorized_commands}9.3.6. 有颜色输出的命令 {#有颜色输出的命令 .title}

</div>

``` screen
if [ "$TERM" != "dumb" ]; then
    eval "`dircolors -b`"
    alias ls='ls --color=always'
    alias ll='ls --color=always -l'
    alias la='ls --color=always -A'
    alias less='less -R'
    alias ls='ls --color=always'
    alias grep='grep --color=always'
    alias egrep='egrep --color=always'
    alias fgrep='fgrep --color=always'
    alias zgrep='zgrep --color=always'
else
    alias ll='ls -l'
    alias la='ls -A'
fi
```

### []{#_repeating_a_command_looping_over_files}9.4.9. 使用文件循环来重复一个命令 {#使用文件循环来重复一个命令 .title}

</div>

</div>
:::::

通过匹配某些条件的文件来循环重复一个命令，有几种方法，例如，匹配全局模式\"`*.ext`{.literal}\".

::: itemizedlist
-   Shell 循环方式(参见 [第 12.1.4 节 "shell 循环"](ch12.zh-cn.html#_shell_loops "12.1.4. shell 循环"){.xref}):

``` screen
for x in *.ext; do if [ -f "$x"]; then command "$x" ; fi; done
```

::: itemizedlist
-   `find`{.literal}(1) 和 `xargs`{.literal}(1) 联合:
:::

``` screen
find . -type f -maxdepth 1 -name '*.ext' -print0 | xargs -0 -n 1 command
```

### []{#_starting_a_program_from_gui}9.4.10. 从 GUI 启动一个程序 {#从-gui-启动一个程序 .title}

</div>

</div>
:::::

对于 [命令行界面（command-line interface，CLI）](https://zh.wikipedia.org/wiki/Command-line_interface){.ulink}，`$PATH`{.literal} 环境变量所指定的目录中第一个匹配相应名称的程序会被执行。参见 [第 1.5.3 节 "\"`$PATH`{.literal}\" 变量"](ch01.zh-cn.html#_the_literal_path_literal_variable "1.5.3. "$PATH" 变量"){.xref}。

对于遵从 [freedesktop.org](https://www.freedesktop.org/wiki/){.ulink} 标准的 [图形用户界面（graphical user interface，GUI）](https://zh.wikipedia.org/wiki/Graphical_user_interface){.ulink}，`/usr/share/applications/`{.literal} 目录中的 `*.desktop`{.literal} 文件给每个程序的 GUI 菜单显示提供了必要的属性。遵从Freedesktop.org xdg 菜单系统的每一个软件包，通过 \"/usr/share/applications/\"下 \"\*.desktop\"提供的数据来安装它的菜单。 遵从 Freedesktop.org 标准的现代桌面环境，用 xdg-utils 软件包利用这些数据生成它们的菜单。参见\"/usr/share/doc/xdg-utils/README\"。

举个例子，`chromium.desktop`{.literal} 文件中为 "Chromium 网络浏览器" 定义了相关属性，例如程序名 "Name"，程序执行路径和参数 "Exec"，所使用的图标 "Icon" 等等（参见 [桌面配置项规范](https://specifications.freedesktop.org/desktop-entry-spec/desktop-entry-spec-latest.html){.ulink}）。文件内容如下：

``` screen
[Desktop Entry]
Version=1.0
Name=Chromium Web Browser
GenericName=Web Browser
Comment=Access the Internet
Comment[fr]=Explorer le Web
Exec=/usr/bin/chromium %U
Terminal=false
X-MultipleArgs=false
Type=Application
Icon=chromium
Categories=Network;WebBrowser;
MimeType=text/html;text/xml;application/xhtml_xml;x-scheme-handler/http;x-scheme-handler/https;
StartupWMClass=Chromium
StartupNotify=true
```

这是一个较为简单的说明。`*.desktop`{.literal} 文件像下面那样被搜寻。

桌面环境设置 `$XDG_DATA_HOME`{.literal} 和 `$XDG_DATA_DIR`{.literal} 环境变量。举个例子，在 GNOME 中：

::: itemizedlist
-   未设置 `$XDG_DATA_HOME`{.literal}。（将使用默认值 `$HOME/.local/share`{.literal}。）

-   `$XDG_DATA_DIRS`{.literal} 被设置为 `/usr/share/gnome:/usr/local/share/:/usr/share/`{.literal}。
:::

基准目录（参见 [XDG Base Directory Specification](https://specifications.freedesktop.org/basedir-spec/basedir-spec-latest.html){.ulink}）和`应用程序`{.literal}目录如下所示。

::: itemizedlist
-   `$HOME/.local/share/`{.literal} → `$HOME/.local/share/applications/`{.literal}

-   `/usr/share/gnome/`{.literal} → `/usr/share/gnome/applications/`{.literal}

-   `/usr/local/share/`{.literal} → `/usr/local/share/applications/`{.literal}

-   `/usr/share/`{.literal} → `/usr/share/applications/`{.literal}
:::

`*.desktop`{.literal} 文件将按照这个顺序在这些 `applications`{.literal} 目录中进行搜寻。

``` screen
# run at 14:15 on the first of every month -- output mailed to paul
15 14 1  * *   $HOME/bin/monthly

0 22 *   * 1-5 mail -s "It's 10pm" joe%Joe,%%Where are your kids?%.%%
23 */2 1 2 *   echo "run 23 minutes after 0am, 2am, 4am ..., on Feb 1"
5  4 *   * sun echo "run at 04:05 every Sunday"
```

-   128 = 0x80 - 允许重启、关闭电源（打开）

### []{#_system_and_hardware_time}9.5.5. 系统时间和硬件时间 {#系统时间和硬件时间 .title}

</div>

</div>
:::::

下面设置系统的硬件时间为：MM/DD hh:mm, CCYY.

``` screen
# date MMDDhhmmCCYY
# hwclock --utc --systohc
# hwclock --show
```

Debian 系统的时间通常显示为本地时间，但硬件时间通常使用 [UTC(GMT)](https://zh.wikipedia.org/wiki/UTC){.ulink} 时间。

如果硬件时间设置为 UTC 时间，请在"`/etc/default/rcS`{.literal}"里面设置"`UTC=yes`{.literal}"。

下面是重新配置 Debian 系统使用的时区。

``` screen
# dpkg-reconfigure tzdata
```

如果你希望通过网络来更新系统时间，考虑使用 `ntp`{.literal}, `ntpdate`{.literal} 和 `chrony`{.literal} 这类包提供的 [NTP](https://zh.wikipedia.org/wiki/Network_Time_Protocol){.ulink} 服务。

### []{#_memory_usage}9.5.10. 内存使用 {#内存使用 .title}

</div>

</div>
:::::

对你来说，这里有两种可用的方法来得到内存的使用情况。

::: itemizedlist
-   "`/var/log/dmesg`{.literal}" 中的内核启动信息包含了可用内存的精确总大小。

-   `free`{.literal}(1) 和 `top`{.literal}(1) 显示正在运行的系统中内存资源的相关信息。
:::

下面是一个例子。

``` screen
# grep '\] Memory' /var/log/dmesg
[    0.004000] Memory: 990528k/1016784k available (1975k kernel code, 25868k reserved, 931k data, 296k init)
$ free -k
             total       used       free     shared    buffers     cached
Mem:        997184     976928      20256          0     129592     171932
-/+ buffers/cache:     675404     321780
Swap:      4545576          4    4545572
```

你可能会觉得奇怪："dmesg 告诉你 free 为 990 MB，而 free -k 告诉你 free 为 320 MB。这丢失了超过 600 MB ......"。

别担心 "`Mem:`{.literal}" 这行中 "`used`{.literal}" 较大的值以及 "`free`{.literal}" 较小的值，放轻松，你需要查看的是下面的那个（在上面的例子中它们是 675404 和 321780）。

对于我的 MacBook，有 1GB=1048576k 内存（显卡系统用掉一些），我看到的如下。

::: table-contents

  ----------------------- ------------------------------

### []{#_disk_partition_configuration}9.6.2. 硬盘分区配置 {#硬盘分区配置 .title}

</div>

</div>
:::::

对于[硬盘分区](https://zh.wikipedia.org/wiki/Disk_partitioning){.ulink}配置，尽管 `fdisk`{.literal}(8) 被认为是标准的配置，但是 `parted`{.literal}(8) 工具还是值得注意的。

老的 PC 使用经典的[主引导记录（Master Boot Record，MBR）](https://zh.wikipedia.org/wiki/Master_boot_record){.ulink}方案，将[硬盘分区](https://zh.wikipedia.org/wiki/Disk_partitioning){.ulink}数据保存在第一个扇区，即 [LBA](https://zh.wikipedia.org/wiki/Logical_block_addressing){.ulink} 扇区 0（512 字节）。

一些带有[统一可扩展固件接口（Unified Extensible Firmware Interface，UEFI）](https://zh.wikipedia.org/wiki/Unified_Extensible_Firmware_Interface){.ulink}的近代 PC，包括基于 Intel 的 Mac，使用 [全局唯一标识分区表（GUID Partition Table，GPT）](https://zh.wikipedia.org/wiki/GUID_Partition_Table){.ulink}方案，[硬盘分区](https://zh.wikipedia.org/wiki/Disk_partitioning){.ulink}数据不保存在第一个扇区。

尽管 `fdisk`{.literal}(8) 一直是硬盘分区的标准工具，但现在 `parted`{.literal}(8) 替代了它。

### []{#_optimization_of_filesystem_via_superblock}9.6.8. 通过超级块（superblock）优化文件系统 {#通过超级块superblock优化文件系统 .title}

</div>

</div>
:::::

一个文件系统的特性可以使用 `tune2fs`{.literal}(8) 命令通过超级块来优化。

::: itemizedlist
-   执行 "`sudo tune2fs -l /dev/sda1`{.literal}" 可以显示 "`/dev/sda1`{.literal}" 上的文件系统超级块内容。

-   执行\"`sudo tune2fs -c 50 /dev/sda1`{.literal}\" 改变 \"`/dev/sda1`{.literal}\" 文件系统检查(在启动时执行 `fsck`{.literal})的频率为每 50 次启动.

-   执行 "`sudo tune2fs -j /dev/sda1`{.literal}" 会给文件系统添加日志功能，即 "`/dev/sda1`{.literal}" 的文件系统从 [ext2](https://zh.wikipedia.org/wiki/Ext2){.ulink} 转换为 [ext3](https://zh.wikipedia.org/wiki/Ext3){.ulink}。（对未挂载的文件系统这么做。）

-   执行 "`sudo tune2fs -O extents,uninit_bg,dir_index /dev/sda1 && fsck -pf /dev/sda1`{.literal}" 在 "`/dev/sda1`{.literal}" 上将它从 [ext3](https://zh.wikipedia.org/wiki/Ext3){.ulink} 转换为 [ext4](https://zh.wikipedia.org/wiki/Ext4){.ulink}。（对未挂载的系统这么做。）
:::

### []{#_optimization_of_hard_disk}9.6.9. 硬盘优化 {#硬盘优化 .title}

### []{#_using_smart_to_predict_hard_disk_failure}9.6.11. 使用 SMART 预测硬盘故障 {#使用-smart-预测硬盘故障 .title}

3.  通过 `df`{.literal}(1) 列出硬盘驱动并识别它们。

    ::: itemizedlist
    -   消除 `/etc/default/smartmontools`{.literal}" 文件中 "`start_smartd=yes`{.literal}" 的注释。

    -   通过 "`sudo systemctl restart smartmontools`{.literal}" 重新启动 `smartd`{.literal}(8) 后台守护进程（daemon）。

### []{#_making_the_disk_image_file}9.7.1. 制作磁盘映像文件 {#制作磁盘映像文件 .title}

</div>

</div>
:::::

一个未挂载设备（例如，第二个 SCSI 或 串行 ATA 设备 "`/dev/sdb`{.literal}"）的磁盘映像文件 "`disk.img`{.literal}" ，可以使用下列方式之一建立。

``` screen
# dd if=/dev/sdb of=disk.img; sync
```

``` screen
# cp /dev/sdb disk.img ; sync
```

``` screen
# cat /dev/sdb > disk.img ; sync
```

传统 PC 中位于主 IDE 硬盘第一扇区的[主引导记录（MBR）](https://zh.wikipedia.org/wiki/Master_boot_record){.ulink}（参见 [第 9.6.2 节 "硬盘分区配置"](ch09.zh-cn.html#_disk_partition_configuration "9.6.2. 硬盘分区配置"){.xref}）的磁盘映像可以使用 `dd`{.literal}(1) 通过下列方式建立。

``` screen
# dd if=/dev/sda of=mbr.img bs=512 count=1
# dd if=/dev/sda of=mbr-nopart.img bs=446 count=1
# dd if=/dev/sda of=mbr-part.img skip=446 bs=1 count=66
```

::: itemizedlist
-   "`mbr.img`{.literal}"：带有分区表的 MBR

-   "`mbr-nopart.img`{.literal}"：不带分区表的 MBR

-   "`mbr-part.img`{.literal}"：仅 MBR 的分区表
:::

如果你要建立原始硬盘的一个硬盘分区的映像，你需要使用 "`/dev/sda1`{.literal}" 等替代 "`/dev/sda`{.literal}"。

### []{#_cleaning_a_disk_image_file}9.7.4. 清理磁盘映像文件 {#清理磁盘映像文件 .title}

</div>

</div>
:::::

使用下面的方式，一个磁盘映像文件 \"`disk.img`{.literal}\" 能够清理掉所有已经删除的文件，成为一个干净的稀疏映像 \"`new.img`{.literal}\"。

``` screen
# mkdir old; mkdir new
# mount -t auto -o loop disk.img old
# dd bs=1 count=0 if=/dev/zero of=new.img seek=5G
# mount -t auto -o loop new.img new
# cd old
# cp -a --sparse=always ./ ../new/
# cd ..
# umount new.img
# umount disk.img
```

如果 "`disk.img`{.literal}" 位于 ext2、ext3 或 ext4，你也可以像下面那样使用 `zerofree`{.literal} 软件包中的 `zerofree`{.literal}(8)。

``` screen
# losetup --show -f disk.img
/dev/loop0
# zerofree /dev/loop0
# cp --sparse=always disk.img new.img
# losetup -d /dev/loop0
```

### []{#_making_the_empty_disk_image_file}9.7.5. 制作空的磁盘映像文件 {#制作空的磁盘映像文件 .title}

</div>

</div>
:::::

按下面的方式使用 `dd`{.literal}(1) ，可以制作一个大小为 5GB 的空磁盘映像文件。

``` screen
$ dd bs=1 count=0 if=/dev/zero of=disk.img seek=5G
```

专业的 `fallocate`{.literal}(8) 可以在这里被使用，用来替代使用 `dd`{.literal}(1)。

按下面的方式使用[环回设备](https://zh.wikipedia.org/wiki/Loop_device){.ulink}，你能够在这个磁盘映像\"`disk.img`{.literal}\"上创建 ext4 文件系统。

``` screen
# losetup --show -f disk.img
/dev/loop0
# mkfs.ext4 /dev/loop0
...hack...hack...hack
# losetup -d /dev/loop0
$ du  --apparent-size -h disk.img
5.0G  disk.img
$ du -h disk.img
83M disk.img
```

对于 \"`disk.img`{.literal}\",它的文件大小是5.0 GiB，而它实际磁盘使用仅仅是 83MiB.这个差距可能是由于 [ext4](https://zh.wikipedia.org/wiki/Ext4){.ulink} 里面有[稀疏文件](https://zh.wikipedia.org/wiki/Sparse_file){.ulink}.

::: {.tip style="margin-left: 0.5in; margin-right: 0.5in;"}
  ----------------------------- -----------------------------------------------------------------------------------------------------
   ![\[提示\]](images/tip.png)  提示
                                [稀疏文件](https://zh.wikipedia.org/wiki/Sparse_file){.ulink}的实际磁盘使用会随着数据的写入而增加。
  ----------------------------- -----------------------------------------------------------------------------------------------------
:::

[回环设备](https://zh.wikipedia.org/wiki/Loop_device){.ulink} 或 [设备映射](https://zh.wikipedia.org/wiki/Device_mapper){.ulink} 设备上使用类似的操作，在这些设备按 [第 9.7.3 节 "挂载磁盘映像文件"](ch09.zh-cn.html#_mounting_the_disk_image_file "9.7.3. 挂载磁盘映像文件"){.xref} 挂载后, 你能够使用 `parted`{.literal}(8) 或 `fdisk`{.literal}(8)对这个磁盘映像\"`disk.img`{.literal}\"进行分区，能够使用 `mkfs.ext4`{.literal}(8), `mkswap`{.literal}(8)在上面创建文件系统等.

### []{#_making_the_iso9660_image_file}9.7.6. 制作 ISO9660 镜像文件 {#制作-iso9660-镜像文件 .title}

</div>

</div>
:::::

::: {.tip style="margin-left: 0.5in; margin-right: 0.5in;"}
  ----------------------------- ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
   ![\[提示\]](images/tip.png)  提示
                                [cdrkit](https://en.wikipedia.org/wiki/Cdrkit){.ulink} 提供的 `genisoimage`{.literal}(1) 和 [Libburnia](https://en.wikipedia.org/wiki/Libburnia){.ulink} 提供的 `xorrisofs`{.literal}(1) ，这两个命令共享相同的命令语法，仅命令名称不一样。
  ----------------------------- ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
:::

\"`源目录`{.literal}\"下的目录树可以通过如下所示的 [cdrkit](https://en.wikipedia.org/wiki/Cdrkit){.ulink} 提供的 `genisoimage`{.literal}(1) 命令来制作 [ISO9660](https://zh.wikipedia.org/wiki/ISO_9660){.ulink} 镜像文件，\"`cd.iso`{.literal}\"。

``` screen
#  genisoimage -r -J -T -V volume_id -o cd.iso source_directory
```

类似的，可引导的 ISO9660 镜像文件，\"`cdboot.iso`{.literal}\", 能够从 `debian-installer`{.literal} 类似目录树 \"`source_directory`{.literal}\" 制作，方式如下。

``` screen
#  genisoimage -r -o cdboot.iso -V volume_id \
   -b isolinux/isolinux.bin -c isolinux/boot.cat \
   -no-emul-boot -boot-load-size 4 -boot-info-table source_directory
```

这里的 [Isolinux boot loader](https://zh.wikipedia.org/wiki/SYSLINUX){.ulink} (参见 [第 3.1.2 节 "第二阶段：引载加载程序"](ch03.zh-cn.html#_stage_2_the_boot_loader "3.1.2. 第二阶段：引载加载程序"){.xref}) 是用于启动的.

按下面的方式，你可以直接从光驱设备计算 md5sum 值，并制作 ISO9660 镜像。

``` screen
$ isoinfo -d -i /dev/cdrom
CD-ROM is in ISO 9660 format
...
Logical block size is: 2048
Volume size is: 23150592
...
# dd if=/dev/cdrom bs=2048 count=23150592 conv=notrunc,noerror | md5sum
# dd if=/dev/cdrom bs=2048 count=23150592 conv=notrunc,noerror > cd.iso
```

   ![\[警告\]](images/warning.png)  警告
                                    为了得到正确结果，你必须小心避免 Linux ISO9600 文件系统预读 bug。
  --------------------------------- -------------------------------------------------------------------

### []{#_dummy_files}9.8.7. 样子文件 {#样子文件 .title}

</div>

</div>
:::::

下面的命令创建样子文件或空文件。

``` screen
$ dd if=/dev/zero    of=5kb.file bs=1k count=5
$ dd if=/dev/urandom of=7mb.file bs=1M count=7
$ touch zero.file
$ : > alwayszero.file
```

你将发现下列文件。

::: itemizedlist
-   \"`5kb.file`{.literal}\" 是 5KB 的全零数据。

-   \"`7mb.file`{.literal}\" 是 7MB 随机数据。

-   \"`zero.file`{.literal}\" 也许是一个 0 字节的文件。如果这个文件之前就存在，则它的 `mtime`{.literal} 会被更新，而它的内容和长度保持不变。

-   \"`alwayszero.file`{.literal}\" 一定是一个 0 字节文件。如果这个文件之前存在，则它的 `mtime`{.literal} 会被更新，而它的内容会被清零。
:::

### []{#_erasing_an_entire_hard_disk}9.8.8. 擦除整块硬盘 {#擦除整块硬盘 .title}

</div>

</div>
:::::

有几种方法来完全擦除设备上整个硬盘上数据，比如说，在 \"`/dev/sda`{.literal}\" 上的 [USB 盘](https://zh.wikipedia.org/wiki/USB_flash_drive){.ulink}。

::: {.caution style="margin-left: 0.5in; margin-right: 0.5in;"}
  --------------------------------- ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
   ![\[小心\]](images/caution.png)  小心
                                    在执行这里的命令之前，你应该用 `mount`{.literal}(8) 命令来查看 [USB 盘](https://zh.wikipedia.org/wiki/USB_flash_drive){.ulink} 的挂载位置。\"`/dev/sda`{.literal}\" 指向的设备可能是装有整个系统的 SCSI 硬盘或者 serial-ATA 硬盘。
  --------------------------------- ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
:::

如下所示是通过数据归 0 的方式来擦除硬盘上所有数据的。

``` screen
# dd if=/dev/zero of=/dev/sda
```

如下是用随机数据重写的方式来擦除所有数据的。

``` screen
# dd if=/dev/urandom of=/dev/sda
```

如下是用随机数据重写的方式来高效擦除所有数据。

``` screen
# shred -v -n 1 /dev/sda
```

你或者可以使用 `badblocks`{.literal}(8) 加上 `-t random`{.literal} 选项。

因为 `dd`{.literal}(1) 命令在许多可引导的 Linux CDs (例如 Debian 安装光盘) 上的 shell 环境下都是可用的，你能够在装有系统的硬盘上，例如 \"`/dev/sda`{.literal}\", \"`/dev/sda`{.literal}\" 等等设备上运行擦除命令来完全清除已经安装的系统。

### []{#_erasing_unused_area_of_an_hard_disk}9.8.9. 擦除硬盘上的未使用的区域 {#擦除硬盘上的未使用的区域 .title}

</div>

### []{#_undeleting_deleted_but_still_open_files}9.8.10. 恢复已经删除但仍然被打开的文件 {#恢复已经删除但仍然被打开的文件 .title}

</div>

</div>
:::::

即使你不小心删除了某个文件，只要这个文件仍然被一些应用程序所使用（读或者写），恢复此文件是可能的。

尝试下列例子

``` screen
$ echo foo > bar
$ less bar
$ ps aux | grep ' less[ ]'
bozo    4775  0.0  0.0  92200   884 pts/8    S+   00:18   0:00 less bar
$ rm bar
$ ls -l /proc/4775/fd | grep bar
lr-x------ 1 bozo bozo 64 2008-05-09 00:19 4 -> /home/bozo/bar (deleted)
$ cat /proc/4775/fd/4 >bar
$ ls -l
-rw-r--r-- 1 bozo bozo 4 2008-05-09 00:25 bar
$ cat bar
foo
```

当你安装了 `lsof`{.literal} 软件包的时候，在另外一个终端执行如下命令。

``` screen
$ ls -li bar
2228329 -rw-r--r-- 1 bozo bozo 4 2008-05-11 11:02 bar
$ lsof |grep bar|grep less
less 4775 bozo 4r REG 8,3 4 2228329 /home/bozo/bar
$ rm bar
$ lsof |grep bar|grep less
less 4775 bozo 4r REG 8,3 4 2228329 /home/bozo/bar (deleted)
$ cat /proc/4775/fd/4 >bar
$ ls -li bar
2228302 -rw-r--r-- 1 bozo bozo 4 2008-05-11 11:05 bar
$ cat bar
foo
```

### []{#_searching_all_hardlinks}9.8.11. 查找所有硬链接 {#查找所有硬链接 .title}

</div>

</div>
:::::

有硬链接的文件，能够使用 \"`ls -li`{.literal}\" 确认。

``` screen
$ ls -li
total 0
2738405 -rw-r--r-- 1 root root 0 2008-09-15 20:21 bar
2738404 -rw-r--r-- 2 root root 0 2008-09-15 20:21 baz
2738404 -rw-r--r-- 2 root root 0 2008-09-15 20:21 foo
```

\"`baz`{.literal}\" 和 \"`foo`{.literal}\" 的链接数为 \"2\" (\>1)，表示他们有硬链接。它们的 [inode](https://zh.wikipedia.org/wiki/Inode){.ulink} 号都是\"2738404\".这表示它们是同样的硬链接文件。如果你不想偶然碰巧发现硬链接文件，你可以通过 [inode](https://zh.wikipedia.org/wiki/Inode){.ulink} 号来查找它。比如说, 按下面的方式查找 \"2738404\" 。

``` screen
# find /path/to/mount/point -xdev -inum 2738404
```

### []{#_removable_disk_encryption_with_dm_crypt_luks}9.9.1. 使用 dm-crypt/LUKS 加密移动磁盘 {#使用-dm-cryptluks-加密移动磁盘 .title}

</div>

</div>
:::::

您可以用 [dm-crypt](https://zh.wikipedia.org/wiki/Dm-crypt){.ulink}/[LUKS](https://zh.wikipedia.org/wiki/Linux_Unified_Key_Setup){.ulink} 加密大容量可移动设备上的数据，例如挂载在"`/dev/sdx`{.literal}"上的 [USB 盘](https://zh.wikipedia.org/wiki/USB_flash_drive){.ulink}。你只需按如下步骤简单地把它格式化。

``` screen
# fdisk /dev/sdx
... "n" "p" "1" "return" "return" "w"
# cryptsetup luksFormat /dev/sdx1
...
# cryptsetup open /dev/sdx1 secret
...
# ls -l /dev/mapper/
total 0
crw-rw---- 1 root root  10, 60 2021-10-04 18:44 control
lrwxrwxrwx 1 root root       7 2021-10-04 23:55 secret -> ../dm-0
# mkfs.vfat /dev/mapper/secret
...
# cryptsetup close secret
```

然后，它就可以正常的在现代桌面环境下使用 `udisks2`{.literal} 软件包，挂载到 \"`/media/`{.literal}[*`username/disk_label`{.literal}*]{.emphasis}\"，只不过它会要求输入密码 (参见 [第 10.1.7 节 "可移动存储设备"](ch10.zh-cn.html#_removable_storage_device "10.1.7. 可移动存储设备"){.xref})。不同的是写入的数据都是加密的。密码条目可以自动使用钥匙环（参见 [第 10.3.6 节 "密码密钥环"](ch10.zh-cn.html#_password_keyring "10.3.6. 密码密钥环"){.xref}）。

你可以把它格式化成其他格式的文件系统，例如用 \"`mkfs.ext4 /dev/mapper/sdx1`{.literal}\" 把它格式化为 ext4。如果使用 btrfs，则需要安装 `udisks2-btrfs`{.literal} 软件包。对于这些文件系统，可能需要配置文件的所有者和权限。

-   当一个模块被激活时，通过 `modprobe`{.literal}(8) 参数来设置模块参数。(参见 [第 9.7.3 节 "挂载磁盘映像文件"](ch09.zh-cn.html#_mounting_the_disk_image_file "9.7.3. 挂载磁盘映像文件"){.xref})
:::

### []{#_compiling_the_kernel_source_debian_kernel_team_recommendation}9.10.4. 编译内核源代码：Debian 内核团队推荐 {#编译内核源代码debian-内核团队推荐 .title}

</div>

</div>
:::::

从上游内核源代码编译个性化的内核二进制包，你应当使用由它提供的 \"`deb-pkg`{.literal}\" 对象。

``` screen
$ sudo apt-get build-dep linux
$ cd /usr/src
$ wget https://mirrors.edge.kernel.org/pub/linux/kernel/v6.x/linux-version.tar.xz
$ tar --xz -xvf linux-version.tar.xz
$ cd linux-version
$ cp /boot/config-version .config
$ make menuconfig
 ...
$ make deb-pkg
```

::: {.tip style="margin-left: 0.5in; margin-right: 0.5in;"}
  ----------------------------- -----------------------------------------------------------------------------------------------------------------------------------------------------------------------------
   ![\[提示\]](images/tip.png)  提示
                                linux-source-[*version*]{.emphasis} 软件包使用 \"`/usr/src/linux-`{.literal}[*`version`{.literal}*]{.emphasis}`.tar.bz2`{.literal}\" 提供有 Debian 补丁的 Linux内核源代码。
  ----------------------------- -----------------------------------------------------------------------------------------------------------------------------------------------------------------------------
:::

从 Debian 内核源代码软件包编译特定的二进制包，你应当使用 \"`debian/rules.gen`{.literal}\" 里的 \"`binary-arch_`{.literal}[*`architecture`{.literal}*]{.emphasis}`_`{.literal}[*`featureset`{.literal}*]{.emphasis}`_`{.literal}[*`flavour`{.literal}*]{.emphasis}\" 对象。

``` screen
$ sudo apt-get build-dep linux
$ apt-get source linux
$ cd linux-3.*
$ fakeroot make -f debian/rules.gen binary-arch_i386_none_686
```

### []{#_hardware_drivers_and_firmware}9.10.5. 硬件驱动和固件 {#硬件驱动和固件 .title}

</div>

</div>
:::::

硬件驱动是运行在目标系统上主 CPU 上的代码。大部分硬件驱动现在是自由软件，已经包含在普通的 Debian 内核软件包里，放在 `main`{.literal} 区域。

::: itemizedlist
-   [GPU](https://zh.wikipedia.org/wiki/Graphics_processing_unit){.ulink} 驱动

    ::: itemizedlist
    -   Intel GPU 驱动 (`main`{.literal})

    -   AMD/ATI GPU 驱动 (`main`{.literal})和/

    -   NVIDIA GPU 驱动 ([nouveau](https://zh.wikipedia.org/wiki/Nouveau_(software)){.ulink} 驱动放在 `main`{.literal} , 由厂家支持的二进制驱动，放在 `non-free`{.literal}.)
    :::
:::

固件是加载在连接在目标系统设备上的代码或数据(比如说，CPU [microcode](https://zh.wikipedia.org/wiki/Microcode){.ulink}、 GPU 运行的渲染代码 或 [FPGA](https://zh.wikipedia.org/wiki/FPGA){.ulink} / [CPLD](https://zh.wikipedia.org/wiki/Complex_programmable_logic_device){.ulink} 数据......）部分固件包是作为自由软件存在，但是很多固件包由于包含有没有源代码的数据，二进制不是作为自由软件存在。安装这些固件数据是必需的，这样设备才能按期望运行。

::: itemizedlist
-   固件数据软件包，含有加载到目标设备易失性存储器上的数据。

    ::: itemizedlist
    -   firmware-linux-free (`main`{.literal})

    -   firmware-linux-nonfree (`non-free-firmware`{.literal})

    -   firmware-linux-\* (`non-free-firmware`{.literal})

    -   \*-firmware (`non-free-firmware`{.literal})

    -   intel-microcode (`non-free-firmware`{.literal})

    -   amd64-microcode (`non-free-firmware`{.literal})
    :::

-   固件更新程序软件包，更新在目标设备非易失性存储器上的数据。

    ::: itemizedlist
    -   [fwupd](https://zh.wikipedia.org/wiki/Fwupd){.ulink} (`main`{.literal})：从 [Linux Vendor Firmware Service](https://fwupd.org/){.ulink} 下载固件数据的固件更新后台守护进程（daemon）。

    -   gnome-firmware (`main`{.literal}): fwupd 的 GTK 前端

    -   plasma-discover-backend-fwupd (`main`{.literal}): fwupd 的 Qt 前端
    :::
:::

请注意，从 Debian 12 Bookworm 开始，用户使用由官方安装介质里面提供的 `non-free-firmware`{.literal} 软件包来提供完善的安装体验。 `non-free-firmware`{.literal} 区域在 [第 2.1.5 节 "Debian 档案库基础"](ch02.zh-cn.html#_debian_archive_basics "2.1.5. Debian 档案库基础"){.xref} 里面描述。

也请注意到， [fwupd](https://zh.wikipedia.org/wiki/Fwupd){.ulink} 从 [Linux Vendor Firmware Service](https://fwupd.org/){.ulink} 下载的固件数据并加载到正在运行的 Linux 内核，可能是 `non-free`{.literal}。

### []{#_virtualization_tools}9.11.1. 虚拟化和模拟器工具 {#虚拟化和模拟器工具 .title}

</div>

</div>
:::::

有几个 [虚拟化](https://zh.wikipedia.org/wiki/Virtualization){.ulink} 和模拟器工具平台。

::: itemizedlist
-   完全的 [硬件模拟](https://zh.wikipedia.org/wiki/Hardware_emulation){.ulink} 软件包，比如通过 [games-emulator](https://packages.debian.org/sid/games-emulator){.ulink} 元软件包安装的软件包

-   操作系统级的容器虚拟化，加上内核级的支持，比如 [LXC (Linux Containers)](https://zh.wikipedia.org/wiki/LXC){.ulink}, [Docker](https://zh.wikipedia.org/wiki/Docker_(software)){.ulink}, `systemd-nspawn`{.literal}(1), \...

-   操作系统级的文件系统访问虚拟化，加上在文件路径上的系统库调用，比如 [chroot](https://zh.wikipedia.org/wiki/Chroot){.ulink}

-   操作系统级的文件系统访问虚拟化，加上在文件所有者权限上的系统库调用，比如 [fakeroot](https://zh.wikipedia.org/wiki/){.ulink}

-   操作系统 API 模拟器，比如 [Wine](https://zh.wikipedia.org/wiki/Wine_(software)){.ulink}

-   解释器级的虚拟化，加上它的执行选择和运行时库忽略，比如 Python 的 [virtualenv](https://virtualenv.pypa.io/){.ulink} 和 [venv](https://docs.python.org/3/library/venv.html){.ulink}
:::

容器虚拟化使用 [第 4.7.5 节 "Linux 安全特性"](ch04.zh-cn.html#_linux_security_features "4.7.5. Linux 安全特性"){.xref}，是 [第 7.7 节 "沙盒"](ch07.zh-cn.html#_sandbox "7.7. 沙盒"){.xref} 的后端技术。

### []{#_virtualization_work_flow}9.11.2. 虚拟化工作流 {#虚拟化工作流 .title}

</div>

</div>
:::::

::: {.note style="margin-left: 0.5in; margin-right: 0.5in;"}
  ------------------------------ ----------------------------------------------------------------------------------------------------------------------------------------
   ![\[注意\]](images/note.png)  注意
                                 自从 `lenny`{.literal} 之后，默认的 Debian 内核就是支持 [KVM](https://zh.wikipedia.org/wiki/Kernel-based_Virtual_Machine){.ulink} 的。
  ------------------------------ ----------------------------------------------------------------------------------------------------------------------------------------
:::

典型的[虚拟化](https://zh.wikipedia.org/wiki/Virtualization){.ulink}工作流涉及以下几个步骤。

::: itemizedlist
-   创建空文件系统 (目录树或磁盘映像)。

    ::: itemizedlist
    -   目录树可以通过 \"`mkdir -p /path/to/chroot`{.literal}\" 创建。

    -   原始的磁盘映像文件能够使用 `dd`{.literal}(1) 创建 (参见 [第 9.7.1 节 "制作磁盘映像文件"](ch09.zh-cn.html#_making_the_disk_image_file "9.7.1. 制作磁盘映像文件"){.xref} 和 [第 9.7.5 节 "制作空的磁盘映像文件"](ch09.zh-cn.html#_making_the_empty_disk_image_file "9.7.5. 制作空的磁盘映像文件"){.xref}).

    -   `qemu-img`{.literal}(1) 能够创建和转化 [QEMU](https://zh.wikipedia.org/wiki/QEMU){.ulink} 支持的磁盘映像文件.

    -   原始的格式和 [VMDK](https://zh.wikipedia.org/wiki/VMDK){.ulink} 文件格式,能够作为虚拟化工具的通用格式.
    :::

-   使用 `mount`{.literal}(8) 挂载磁盘映像到文件系统 (可选).

    ::: itemizedlist
    -   对于原始磁盘映像文件,把它作为[回环设备](https://zh.wikipedia.org/wiki/Loop_device){.ulink} 或 [设备映射](https://zh.wikipedia.org/wiki/Device_mapper){.ulink} 设备挂载. (参见 [第 9.7.3 节 "挂载磁盘映像文件"](ch09.zh-cn.html#_mounting_the_disk_image_file "9.7.3. 挂载磁盘映像文件"){.xref}).

    -   对于 [QEMU](https://zh.wikipedia.org/wiki/QEMU){.ulink} 支持的磁盘映像,把它们作为 [network block device 网络块设备](https://zh.wikipedia.org/wiki/Network_block_device){.ulink}挂载 (参见 [第 9.11.3 节 "挂载虚拟磁盘映像文件"](ch09.zh-cn.html#_mounting_the_virtual_disk_image_file "9.11.3. 挂载虚拟磁盘映像文件"){.xref}).
    :::

-   在目标文件系统上部署需要的系统数据。

    ::: itemizedlist
    -   使用 `debootstrap`{.literal} 和 `cdebootstrap`{.literal} 之类的程序来协助处理这个过程(参见 [第 9.11.4 节 "Chroot 系统"](ch09.zh-cn.html#_chroot_system "9.11.4. Chroot 系统"){.xref}).

    -   在全功能系统模拟器下使用操作系统安装器。
    :::

-   在虚拟化环境下运行一个程序。

    ::: itemizedlist
    -   [chroot](https://zh.wikipedia.org/wiki/Chroot){.ulink} 提供基本的虚拟化环境,足够能在里面编译程序,运行控制台应用,运行后台守护程序 daemon.

    -   [QEMU](https://zh.wikipedia.org/wiki/QEMU){.ulink} 提供跨平台的 CPU 模拟器.

    -   [QEMU](https://zh.wikipedia.org/wiki/QEMU){.ulink} 和 [KVM](https://zh.wikipedia.org/wiki/Kernel-based_Virtual_Machine){.ulink} 通过 [hardware-assisted virtualization硬件辅助虚拟化](https://zh.wikipedia.org/wiki/Hardware-assisted_virtualization){.ulink}来提供全功能系统的模拟.

    -   [VirtualBox](https://zh.wikipedia.org/wiki/VirtualBox){.ulink} 可以在 i386 和 amd64 上,使用或者不使用 [hardware-assisted virtualization硬件辅助虚拟化](https://zh.wikipedia.org/wiki/Hardware-assisted_virtualization){.ulink}来提供全功能系统模拟.
    :::
:::

### []{#_mounting_the_virtual_disk_image_file}9.11.3. 挂载虚拟磁盘映像文件 {#挂载虚拟磁盘映像文件 .title}

</div>

</div>
:::::

对于原始磁盘映像文件,参见 [第 9.7 节 "磁盘映像"](ch09.zh-cn.html#_the_disk_image "9.7. 磁盘映像"){.xref}.

对于其它虚拟磁盘映像文件,你能够用使用 [network block device网络块设备](https://zh.wikipedia.org/wiki/Network_block_device){.ulink} 协议的 `qemu-nbd`{.literal}(8) 来导出他们,并使用内核模块 `nbd`{.literal} 来挂载它们.

`qemu-nbd`{.literal}(8) 支持 [QEMU](https://zh.wikipedia.org/wiki/QEMU){.ulink} 所支持的磁盘格式: raw, [qcow2, qcow](https://zh.wikipedia.org/wiki/Qcow){.ulink}, [vmdk](https://zh.wikipedia.org/wiki/VMDK){.ulink}, [vdi](https://zh.wikipedia.org/wiki/VirtualBox){.ulink}, [bochs](https://zh.wikipedia.org/wiki/Bochs){.ulink}, cow (user-mode Linux copy-on-write), [parallels](https://zh.wikipedia.org/wiki/Parallels_Workstation){.ulink}, [dmg](https://zh.wikipedia.org/wiki/Apple_Disk_Image){.ulink}, [cloop](https://zh.wikipedia.org/wiki/Cloop){.ulink}, [vpc](https://zh.wikipedia.org/wiki/VHD_(file_format)){.ulink}, vvfat (virtual VFAT)和主机设备.

[网络块设备](https://zh.wikipedia.org/wiki/Network_block_device){.ulink} 能够用和[回环设备](https://zh.wikipedia.org/wiki/Loop_device){.ulink}一样的方式支持分区 (参见 [第 9.7.3 节 "挂载磁盘映像文件"](ch09.zh-cn.html#_mounting_the_disk_image_file "9.7.3. 挂载磁盘映像文件"){.xref}). 你能够按下面的方式挂载 \"`disk.img`{.literal}\" 的第一个分区.

``` screen
# modprobe nbd max_part=16
# qemu-nbd -v -c /dev/nbd0 disk.img
...
# mkdir /mnt/part1
# mount /dev/nbd0p1 /mnt/part1
```

::: {.tip style="margin-left: 0.5in; margin-right: 0.5in;"}
  ----------------------------- ------------------------------------------------------------------------------------------------------------
   ![\[提示\]](images/tip.png)  提示
                                你可以给 `qemu-nbd`{.literal}(8) 使用 \"`-P 1`{.literal}\" 选项来导出\"`disk.img`{.literal}\"的第一个分区.
  ----------------------------- ------------------------------------------------------------------------------------------------------------

### []{#_chroot_system}9.11.4. Chroot 系统 {#chroot-系统 .title}

</div>

</div>
:::::

如果你希望从终端控制台尝试一个新的 Debian 环境，我推荐你使用 [chroot](https://zh.wikipedia.org/wiki/Chroot){.ulink}。这使你能够运行 `unstable`{.literal} 和 `testing`{.literal} 的控制台应用程序，不会有通常的相关风险，并且不需要重启。`chroot`{.literal}(8) 是最基础的方法。

::: {.caution style="margin-left: 0.5in; margin-right: 0.5in;"}
  --------------------------------- -------------------------------------------------------------------------------
   ![\[小心\]](images/caution.png)  小心
                                    下面的列子假设根源系统和 chroot 系统都共享相同的 `amd64`{.literal} CPU 架构。
  --------------------------------- -------------------------------------------------------------------------------
:::

虽然你能够手工使用 `debootstrap`{.literal}(1)来创建一个 `chroot`{.literal}(8) 环境，这要求琐碎的工作。

[sbuild](https://packages.debian.org/search?keywords=sbuild){.ulink} 软件包从源代码构建一个 Debian 软件包，使用 [schroot](https://packages.debian.org/search?keywords=schroot){.ulink} 软件包管理的 chroot 环境。它和帮助脚本 `sbuild-createchroot`{.literal}(1) 一起。让我们按如下所示的方式运行它，来学会它是怎样工作的。

``` screen
$ sudo mkdir -p /srv/chroot
$ sudo sbuild-createchroot -v --include=eatmydata,ccache unstable /srv/chroot/unstable-amd64-sbuild http://deb.debian.org/debian
 ...
```

你能够看到 `debootstrap`{.literal}(8) 是如何在 \"`/srv/chroot/unstable-amd64-sbuild`{.literal}\" 下部署 `unstable`{.literal} 环境的系统数据,用于一个最小的构建系统。

你可以使用 `schroot`{.literal}(1) 来登录到这个环境。

``` screen
$ sudo schroot -v -c chroot:unstable-amd64-sbuild
```

你可以看到一个运行在 `unstable`{.literal} 环境的系统 shell 是如何创建的。

::: {.note style="margin-left: 0.5in; margin-right: 0.5in;"}
  ------------------------------ --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
   ![\[注意\]](images/note.png)  注意
                                 \"`/usr/sbin/policy-rc.d`{.literal}\" 文件总是用 101 退出，阻止在 Debian 系统上自动启动后台守护程序。参见 \"`/usr/share/doc/init-system-helpers/README.policy-rc.d.gz`{.literal}\"。
  ------------------------------ --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
:::

::: {.note style="margin-left: 0.5in; margin-right: 0.5in;"}
  ------------------------------ --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
   ![\[注意\]](images/note.png)  注意
                                 一些在 chroot 下的程序，需要访问比上面根源系统上的 `sbuild-createchroot`{.literal} 能够提供的文件之外的更多文件才能够运行。例如，\"`/sys`{.literal}\", \"`/etc/passwd`{.literal}\", \"`/etc/group`{.literal}\", \"`/var/run/utmp`{.literal}\", \"`/var/log/wtmp`{.literal}\"等等.也许需要使用 bind-mounted 或拷贝.
  ------------------------------ --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
:::

::: {.tip style="margin-left: 0.5in; margin-right: 0.5in;"}
  ----------------------------- ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
   ![\[提示\]](images/tip.png)  提示
                                `sbuild`{.literal} 软件包帮助建立一个 chroot 系统来构建一个软件包，在 chroot 内使用 `schroot`{.literal} 作为它的后端。它是一个检查构建依赖（build-dependencies）的理想系统。更多信息参见 [sbuild at Debian wiki](https://wiki.debian.org/sbuild){.ulink} 和[在\"Guide for Debian Maintainers\"中的 sbuild 配置样例](https://www.debian.org/doc/manuals/debmake-doc/ch03.en.html#sbuild-setup){.ulink} 。
  ----------------------------- ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
:::

::: {.tip style="margin-left: 0.5in; margin-right: 0.5in;"}
  ----------------------------- --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
   ![\[提示\]](images/tip.png)  提示
                                `systemd-nspawn`{.literal}(1) 命令使用 `chroot`{.literal} 类似的方法帮助运行一个命令，或者轻量级容器内的操作系统。它更强大，因为它使用命名空间来完全虚拟化进程树、进程间通讯、主机名、域名，并可选网络和用户数据库。 参见 [systemd-nspawn](https://wiki.debian.org/nspawn){.ulink}。

### []{#_multiple_desktop_systems}9.11.5. 多桌面系统 {#多桌面系统 .title}

</div>

</div>
:::::

如果你希望尝试任一操作系统的一个新的 GUI 桌面环境，我推荐在 Debian `稳定版`{.literal} 系统上使用 [QEMU](https://zh.wikipedia.org/wiki/QEMU){.ulink}或者[KVM](https://zh.wikipedia.org/wiki/Kernel-based_Virtual_Machine){.ulink} ，这些软件应用[虚拟化技术](https://zh.wikipedia.org/wiki/Virtualization){.ulink}安全的运行多桌面系统。这能让你运行任何桌面应用，包括 Debian `不稳定版`{.literal}和`测试版`{.literal}上的桌面应用，并且没有与之相关的通常意义上的风险，并且不需要重启。

因为单纯的 [QEMU](https://zh.wikipedia.org/wiki/QEMU){.ulink} 工具是非常慢的，当主机系统支持 [KVM](https://zh.wikipedia.org/wiki/Kernel-based_Virtual_Machine){.ulink} 的时候，建议使用它来加速。

[虚拟机管理器](https://zh.wikipedia.org/wiki/Virt-manager){.ulink}，也被称为 `virt-manager`{.literal}，它是一个便利的管理 KVM 虚拟机的 GUI（图形用户界面）工具，它调用 [libvirt](https://zh.wikipedia.org/wiki/Libvirt){.ulink}。

按下面的方法，能够创建一个可以用于[QEMU](https://zh.wikipedia.org/wiki/QEMU){.ulink} 的包含有 Debian 系统的虚拟磁盘映像 \"`virtdisk.qcow2`{.literal}\"，这个 Debian 系统使用 [debian 安装器:小 CD](https://www.debian.org/distrib/netinst){.ulink}安装。

``` screen
$ wget https://cdimage.debian.org/debian-cd/5.0.3/amd64/iso-cd/debian-503-amd64-netinst.iso
$ qemu-img create -f qcow2 virtdisk.qcow2 5G
$ qemu -hda virtdisk.qcow2 -cdrom debian-503-amd64-netinst.iso -boot d -m 256
...
```

::: {.tip style="margin-left: 0.5in; margin-right: 0.5in;"}
  ----------------------------- -----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
   ![\[提示\]](images/tip.png)  提示
                                在[虚拟化](https://zh.wikipedia.org/wiki/Virtualization){.ulink}下运行 [Ubuntu](https://www.ubuntu.com/){.ulink} 和 [Fedora](https://fedoraproject.org/){.ulink} 之类的其它 GNU/Linux 发行版，是一个不错的学习其配置技巧的方法。其它专有操作系统也可以在这个 GNU/Linux [虚拟化](https://zh.wikipedia.org/wiki/Virtualization){.ulink}下很好的运行。
  ----------------------------- -----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
:::

在 [Debian wiki: SystemVirtualization](https://wiki.debian.org/SystemVirtualization){.ulink} 参看更多技巧。
