我认为学习一个计算机系统，就像学习一门新的外语。虽然教程和文档是有帮助的，但你必须自己练习。为了帮助你平滑起步，我详细说明一些基本要点。

[Debian](https://www.debian.org){.ulink} [GNU](https://zh.wikipedia.org/wiki/GNU){.ulink}/[Linux](https://zh.wikipedia.org/wiki/Linux){.ulink} 中最强大的设计来自 [Unix](https://zh.wikipedia.org/wiki/Unix){.ulink} 操作系统，一个[多用户](https://zh.wikipedia.org/wiki/Multi-user){.ulink}[多任务](https://zh.wikipedia.org/wiki/Computer_multitasking){.ulink}的操作系统。你必须学会利用这些特性以及 Unix 和 GNU/Linux 的相似性。

别回避面向 Unix 的文档，不要只是依赖于 GNU/Linux 文档，这样做会剥夺你了解许多有用的信息。

### []{#_the_shell_prompt}1.1.1. shell 提示符 {#shell-提示符 .title}

</div>

</div>
:::::

启动系统之后，如果你没有安装 [GUI](https://zh.wikipedia.org/wiki/Graphical_user_interface){.ulink}（例如[GNOME](https://zh.wikipedia.org/wiki/GNOME){.ulink} 或者 [KDE](https://zh.wikipedia.org/wiki/KDE){.ulink}），那么你会看到字符登录界面。假设你的主机名为`foo`{.literal}，那么登录提示符将如下所示。

如果你安装了一个 [GUI](https://zh.wikipedia.org/wiki/Graphical_user_interface){.ulink} 环境，那么你仍然能够用 Ctrl-Alt-F3 进入基于字符的登录提示符，同时你能通过 Ctrl-Alt-F2 回到 GUI 环境（更多详情请参阅下文 [第 1.1.6 节 "虚拟控制台"](ch01.zh-cn.html#_virtual_consoles "1.1.6. 虚拟控制台"){.xref}）。

``` screen
foo login:
```

在登录提示符下，你输入你的用户名，例如 `penguin`{.literal}，然后按回车键，接下来输入你的密码并再次按回车键。

::: {.note style="margin-left: 0.5in; margin-right: 0.5in;"}
  ------------------------------ ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
   ![\[注意\]](images/note.png)  注意
                                 遵循 Unix 传统，Debian 系统下的用户名和密码是大小写敏感的。用户名通常由小写字母组成。第一个用户账号通常在安装期间进行创建。额外的用户账号由 root 用户用 `adduser`{.literal}(8) 创建。
  ------------------------------ ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
:::

系统以保存在 \"`/etc/motd`{.literal}\" 中的欢迎信息（Message Of The Day）来开始，同时显示一个命令提示符。

``` screen
Debian GNU/Linux 12 foo tty3

foo login: penguin
Password:

Linux foo 6.5.0-0.deb12.4-amd64 #1 SMP PREEMPT_DYNAMIC Debian 6.5.10-1~bpo12+1 (2023-11-23) x86_64

The programs included with the Debian GNU/Linux system are free software;
the exact distribution terms for each program are described in the
individual files in /usr/share/doc/*/copyright.

Debian GNU/Linux comes with ABSOLUTELY NO WARRANTY, to the extent
permitted by applicable law.

Last login: Wed Dec 20 09:39:00 JST 2023 on tty3
foo:~$
```

现在，你就在 [shell](https://zh.wikipedia.org/wiki/Shell_(computing)){.ulink} 下。shell 解析你的命令。

### []{#_the_shell_prompt_under_gui}1.1.2. GUI 下的 shell 提示符 {#gui-下的-shell-提示符 .title}

</div>

</div>
:::::

如果你在安装 Debian 的过程中，安装了一个 [GUI](https://zh.wikipedia.org/wiki/Graphical_user_interface){.ulink} 环境，那么你在启动系统后将使用图形登录界面。输入你的用户名和密码可以登录到非特权用户帐号。使用 Tab 键（跳格键）可以在用户名和密码之间移动，也可以使用鼠标主要键点击。

要在 GUI（图形用户界面）环境下获得 shell 提示符，你必须启动一个`x 终端模拟器`{.literal}程序，例如`gnome-terminal`{.literal}(1)、`rxvt`{.literal}(1)或`xterm`{.literal}(1)。在 GNOME 桌面环境下，你可以按 超级键（Windows 键），在搜索提示里输入\"terminal\"来打开终端。

在其它一些桌面系统（如 `fluxbox`{.literal}）下面，可能没有明显的开始菜单入口。如果是这种情况，试下右击桌面屏幕并希望能有弹出菜单。

### []{#_the_root_account}1.1.3. root 账户 {#root-账户 .title}

</div>

</div>
:::::

root 账户也被称作[超级用户](https://zh.wikipedia.org/wiki/Superuser){.ulink}或特权用户。用这个账户，你能够履行下面的系统管理任务。

::: itemizedlist
-   读、写和删除系统上的任何文件，不顾它们的文件权限

-   设置系统上任何文件的所有者和权限

-   设置系统上任何非特权用户的密码

-   免用户密码登录任何帐户
:::

无限权力的 root 账户，要求你慎重和负责任的使用。

::: {.warning style="margin-left: 0.5in; margin-right: 0.5in;"}
  --------------------------------- ---------------------------------
   ![\[警告\]](images/warning.png)  警告
                                    千万不要和其他人共享 root 密码.
  --------------------------------- ---------------------------------
:::

::: {.note style="margin-left: 0.5in; margin-right: 0.5in;"}
  ------------------------------ -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
   ![\[注意\]](images/note.png)  注意
                                 一个文件（包括硬件设备，如 CD-ROM 等，这些对 Debian 系统来说都只是一个文件）的权限可能会导致非 root 用户无法使用或访问它 。虽然在这种情况下，使用 root 帐户是一个快速的方法，但正确的解决方法应该是对文件权限和用户组的成员进行合适的设置（参见[第 1.2.3 节 "文件系统权限"](ch01.zh-cn.html#_filesystem_permissions "1.2.3. 文件系统权限"){.xref}）。
  ------------------------------ -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

### []{#_virtual_consoles}1.1.6. 虚拟控制台 {#虚拟控制台 .title}

</div>

</div>
:::::

在默认的 Debian 系统中，有6个可切换的[类VT100](https://zh.wikipedia.org/wiki/VT100){.ulink}字符控制台，可以直接在 Linux 主机上启动 shell。除非你处于 GUI 环境下，否则你可以同时按下`左 Alt 键`{.literal}和`F1`{.literal}---`F6`{.literal}之一的键在虚拟控制台间切换。每一个字符控制台都允许独立登录账户并提供多用户环境。这个多用户环境是伟大的 Unix 的特性，很容易上瘾。

如果你处于 GUI 环境中，你可以通过 `Ctrl-Alt-F3`{.literal} 键前往字符控制台 3，也就是同时按下`左 Ctrl 键`{.literal}、`左 Alt 键`{.literal}和`F3 键`{.literal}。你可以按下 `Alt-F2`{.literal} 回到 GUI 环境，它一般运行在虚拟控制台 2。

你也可以使用命令行切换到另一个虚拟控制台，例如切换到控制台 3。

``` screen
# chvt 3
```

### []{#_how_to_leave_the_command_prompt}1.1.7. 怎样退出命令行提示符 {#怎样退出命令行提示符 .title}

</div>

</div>
:::::

在命令行输入 `Ctrl-D`{.literal}，即同时按下`左侧-Ctrl-键`{.literal}和`d-键`{.literal}，即可关闭 shell 活动。如果你正处于字符控制台，你将会返回到登录提示行。尽管这些控制字符 "control D\" 使用了大写字母，你并不需要按住 Shift-键。`Ctrl-D`{.literal} 也可以简写为 `^D`{.literal}。或者，你也可以键入 "exit" 退出命令行。

如果你位于`x 终端模拟器`{.literal}(1)中，你可以使用这个关闭`x 终端模拟器`{.literal}窗口。

### []{#_how_to_shutdown_the_system}1.1.8. 怎样关闭系统 {#怎样关闭系统 .title}

</div>

</div>
:::::

就像任何其他的现代操作系统一样，Debian 会通过内存中的[缓存数据](https://zh.wikipedia.org/wiki/缓存){.ulink}进行文件操作以提高性能，因此在电源被安全地关闭前需要适当的关机过程，通过将内存中的数据强制写入硬盘来维持文件的完整性。如果软件的电源控制可用，那么关机过程中会自动关闭系统电源。（否则，你可能需要在关机过程之后按电源键几秒钟。）

在普通多用户模式模式下，可以使用命令行关闭系统。

``` screen
# shutdown -h now
```

在单用户模式下，可以使用命令行关闭系统。

``` screen
# poweroff -i -f
```

参见[第 6.3.8 节 "怎样通过 SSH 关闭远程系统"](ch06.zh-cn.html#_how_to_shutdown_the_remote_system_on_ssh "6.3.8. 怎样通过 SSH 关闭远程系统"){.xref}。

### []{#_additional_package_suggestions_for_the_newbie}1.1.10. 建议新手安装的额外软件包 {#建议新手安装的额外软件包 .title}

</div>

</div>
:::::

尽管连无需任何桌面环境的 Debian 系统最小安装都提供了基本的 Unix 功能，但对新手而言，使用 `apt-get`{.literal}(8) 安装一些基于字符终端的命令行和 curses 软件包（例如 `mc`{.literal} 和 `vim`{.literal}）依旧是一个不错的主意。

``` screen
# apt-get update
 ...
# apt-get install mc vim sudo aptitude

对于典型的单用户工作站，例如运行在笔记本电脑上的桌面 Debian 系统，通常简单地配置 `sudo`{.literal}(8) 来使为非特权用户（例如用户 `penguin`{.literal}）只需输入用户密码而非 root 密码就能获得管理员权限。

# echo "penguin  ALL=(ALL) ALL" >> /etc/sudoers

# echo "penguin  ALL=(ALL) NOPASSWD:ALL" >> /etc/sudoers
```

这些技巧只对你管理的单用户工作站中那个唯一的用户有用。

## []{#_unix_like_filesystem}1.2. 类 Unix 文件系统 {#类-unix-文件系统 .title}

</div>

</div>
:::::

在GNU/Linux和其他[类 Unix](https://zh.wikipedia.org/wiki/Unix-like){.ulink}操作系统中，[文件](https://zh.wikipedia.org/wiki/Computer_file){.ulink}被组织到[目录](https://zh.wikipedia.org/wiki/Directory_(file_systems)){.ulink}中。所有的文件和目录排放在以"`/`{.literal}"为根的巨大的树里。叫它树是因为如果你画出文件系统，它看起来就像一棵树，但是它是颠倒过来的。

这些文件和目录可以分散在多个设备中。`mount`{.literal}(8)用于把某个设备上找到的文件系统附着到巨大的文件树上。相反的，`umount`{.literal}(8)把它再次分离。在最近的 Linux 内核里，`mount`{.literal}(8)带某些参数时可以把文件树的一部分绑定到另外的地方，或者可以把文件系统挂载为共享的、私有的、从设备、或不可绑定的。对每个文件系统支持的挂载选项可以在`/usr/share/doc/linux-doc-*/Documentation/filesystems/`{.literal}找到。

Unix系统上叫做[**目录**]{.strong}，某些其他系统上叫做[**文件夹**]{.strong}。请同样留意，在任何Unix系统上，没有的[**驱动器**]{.strong}的概念，例如"`A:`{.literal}"。这只有一个文件系统，并且所有东西都包含在内。这相对于 Windows 来说是一个巨大的优点。

### []{#_unix_file_basics}1.2.1. Unix 文件基础 {#unix-文件基础 .title}

</div>

</div>
:::::

下面是一些 Unix 文件基础。

::: itemizedlist
-   文件名是 [**区分大小写**]{.strong} 的。也就是说， \"`MYFILE`{.literal}\" 和 \"`MyFile`{.literal}\" 是不同的文件。

-   [**根目录**]{.strong}意味着文件系统的根，简单的称为"`/`{.literal}"，不要把它跟 root 用户的家目录"`/root`{.literal}"混淆了。

-   每个目录都有一个名字，它可以包含任意字母或[**除了`/`{.literal}"以外的**]{.strong}符号。根目录是个特例。它的名字是"`/`{.literal}"（称作"斜线"或"根目录"），并且它不能被重命名。

-   每个文件或目录都被指定一个[**全限定文件名**]{.strong}，[**绝对文件名**]{.strong}，或[**路径**]{.strong}，按顺序给出必须经过的目录从而到达相应目录。这三个术语是同义的。

-   所有的[**全限定文件名**]{.strong}以"`/`{.literal}"目录开始，并且在每个目录或文件名之间有一个"`/`{.literal}"。第一个"`/`{.literal}"是最顶层目录，其他的"`/`{.literal}"用于分隔跟着的子目录。直到到达最后的入口，即实际文件的名称。这些话可能会令人困惑。用下面这个[**全限定文件名**]{.strong}作为例子："`/usr/share/keytables/us.map.gz`{.literal}"。不过，人们也把它的基名"`us.map.gz`{.literal}"单独作为文件名。

-   根目录有很多分支，例如"`/etc/`{.literal}"和"`/usr/`{.literal}"。这些子目录依次分出更多的子目录，例如"`/etc/systemd/`{.literal}"和"`/usr/local/`{.literal}"。这整体叫做"[**目录树**]{.strong}"。你可以把一个绝对文件名想象成从"`/`{.literal}"这棵树的基到某个分支（一个文件）的结尾的一条路径。你也听到人们谈论目录树，就好像它是一个包含所有直系后代的"[**家庭**]{.strong}"树的一个图，这个图叫做根目录（"`/`{.literal}"）：因此子目录有[**父**]{.strong}目录，并且一条路径显示了一个文件完整的祖先。也有相对路径从其他地方开始，而不是从根目录。 你应该还记得目录"`../`{.literal}"指向父目录。这个术语也适用于其他类似目录的结构，如分层数据结构。

-   对于一个物理设备, 是没有一个特定的目录路径名来对应的组成部分. 这不同于 [RT-11](https://zh.wikipedia.org/wiki/RT-11){.ulink}, [CP/M](https://zh.wikipedia.org/wiki/CP/M){.ulink},[OpenVMS](https://zh.wikipedia.org/wiki/OpenVMS){.ulink},[MS-DOS](https://zh.wikipedia.org/wiki/MS-DOS){.ulink},[AmigaOS](https://zh.wikipedia.org/wiki/AmigaOS){.ulink}, 以及[微软的 Windows](https://zh.wikipedia.org/wiki/Microsoft_Windows){.ulink}，这些系统存在一个路径包含了一个设备名字，比如\"`C:\`{.literal}\"。(尽管如此, 路径条目确实存在引用了物理设备作为正常的文件系统的一部分. 参考[第 1.2.2 节 "文件系统深入解析"](ch01.zh-cn.html#_filesystem_internals "1.2.2. 文件系统深入解析"){.xref}。)

### []{#_filesystem_internals}1.2.2. 文件系统深入解析 {#文件系统深入解析 .title}

</div>

</div>
:::::

按照[**UNIX系统的传统**]{.strong}，Debian GNU / Linux 的[文件系统](https://zh.wikipedia.org/wiki/File_system){.ulink}是在物理数据存储设备诸如磁盘或其他存储设备上，与硬件设备的交互，如控制台和远程串口终端都是以统一的方式呈现在 "`/ dev /`{.literal}" 下面。

每个文件、目录、命名管道（一种两个程序间共享数据的方法）或 Debian GNU/Linux 系统上的物理设备都有一个叫做 [索引节点（inode）](https://zh.wikipedia.org/wiki/Inode){.ulink} 的数据结构，描述了其相关属性，例如拥有它的用户（所有者），它属于的组，最后一次访问时间，等等。把所有东西都表示在文件系统中的想法是来源于 Unix，现代的 Linux 内核则将这个思路进行了扩展。现在，甚至有关计算机上正在运行的进程的信息都可以在文件系统中找到。

这个对物理实体和内部进程的统一和抽象是非常强大的，因为这允许我们用同样的命令对许多完全不同的设备进行同样的操作。甚至可以通过向链接到运行进程的特殊文件写入数据来改变内核的运行方式。

### []{#_filesystem_permissions}1.2.3. 文件系统权限 {#文件系统权限 .title}

</div>

</div>
:::::

[类 Unix](https://zh.wikipedia.org/wiki/Unix-like){.ulink} 系统的[文件系统权限](https://zh.wikipedia.org/wiki/File_system_permissions){.ulink}被定义给三类受影响的用户。

::: itemizedlist
-   拥有这个文件的[**用户**]{.strong}（[**u**]{.strong}）

-   这个文件所属[**组**]{.strong}的其他用户（[**g**]{.strong}）

-   所有[**其余**]{.strong}的用户（[**o**]{.strong}），同样称为"世界"和"所有人"
:::

对文件来说，每个对应权限允许下列动作。

::: itemizedlist
-   [**可读**]{.strong}（[**r**]{.strong}）权限允许所有者检查文件的内容。

-   [**可写**]{.strong}（[**w**]{.strong}）权限允许所有者修改文件内容。

-   [**可执行**]{.strong}（[**x**]{.strong}）权限允许所有者把文件当做一个命令运行。
:::

对于目录来说，每个对应权限允许下列动作。

::: itemizedlist
-   [**可读**]{.strong}（[**r**]{.strong}）权限允许所有者列出目录内的内容。

-   [**可写**]{.strong}（[**w**]{.strong}）权限允许所有者添加或删除目录里面的文件。

-   [**可执行**]{.strong}（[**x**]{.strong}）权限允许所有者访问目录里的文件。
:::

在这里，一个目录的[**可执行**]{.strong}权限意味着不仅允许读目录里的文件，还允许显示他们的属性，例如大小和修改时间。

`ls`{.literal}(1)用于显示文件和目录的权限信息（更多）。当运行时带有"`-l`{.literal}"选项，它将按给定顺序显示下列信息。

::: itemizedlist
-   [**文件类型**]{.strong}（第一个字母）

-   文件的访问[**权限**]{.strong}（9个字符，三个字符组成一组按照用户、组、其他的顺序表示）

-   链接到文件的[**硬链接数**]{.strong}

-   文件所有者的[**用户**]{.strong}名

-   这个文件所属的[**组**]{.strong}名

-   以字符（字节）为单位的文件[**大小**]{.strong}

-   文件的[**日期和时间**]{.strong}（mtime）

-   文件的[**名字**]{.strong}
:::

:::: table
[]{#listofthefirstchacteroflsloutput}

**表 1.4. "`ls -l`{.literal}"输出的第一个字符列表**

::: table-contents

  --------------- --------------

  `d`{.literal}   目录

  `s`{.literal}   套接字

`chown`{.literal}(1)用于 root 账户修改文件的所有者。`chgrp`{.literal}(1)用于文件的所有者或 root 账户修改文件所属的组。`chmod`{.literal}(1)用于文件的所有者或 root 账户修改文件和文件夹的访问权限。操作一个 `foo`{.literal} 文件的基本语法如下 。

``` screen
# chown newowner foo
# chgrp newgroup foo
# chmod  [ugoa][+-=][rwxXst][,...] foo
```

例如，你可以按照下面使一个目录树被用户`foo`{.literal}所有，并共享给组`bar`{.literal}。

``` screen
# cd /some/location/
# chown -R foo:bar .
# chmod -R ug+rwX,o=rX .
```

有三个更加特殊的权限位。

::: itemizedlist
-   [**Set-User-ID(SUID)**]{.strong}位（[**s**]{.strong}或[**S**]{.strong}替换用户的[**x**]{.strong}）

-   [**Set-Group-ID(SGID)**]{.strong}位（[**s**]{.strong}或[**S**]{.strong}替换组的[**x**]{.strong}）

-   [**粘滞**]{.strong}位（[**t**]{.strong}或[**T**]{.strong}替代其他用户的[**x**]{.strong}）
:::

如果"`ls -l`{.literal}"对这些位的输出是[**大写字母**]{.strong}，则表示这些输出下面的执行位[**未设置**]{.strong}。

给一个可执行文件设置 [**Set-User-ID**]{.strong} 位将允许一个用户以他自己的ID运行这个可执行文件（例如 [**root 用户**]{.strong}）。类似的，给一个可执行文件设置了[**Set-Group-ID**]{.strong} 位将允许一个用户以文件所属组的 ID 运行该文件。（例如 [**root**]{.strong} 组）。由于这些设置可能导致安全风险，设置它们为可用的时候需要格外留意。

在一个目录上设置"[**Set-Group-ID**]{.strong}"将打开[类 BSD](https://zh.wikipedia.org/wiki/Berkeley_Software_Distribution){.ulink} 的文件创建计划，所有在目录里面创建的文件将属于目录所属的[**组**]{.strong}。

给一个目录设置"[**粘滞位**]{.strong}"将保护该目录内的文件不被其所有者之外的一个用户删除。为了保护一个在像"`/tmp`{.literal}"这样所有人可写或同组可写的目录下文件内容的安全，不仅要去除[**可写**]{.strong}权限，还要给其所在目录设置[**粘滞位**]{.strong}。否则，该文件可以被任意对其所在目录有写权限的用户删除并创建一个同名的新文件。

这里有一点有趣的文件权限例子。

``` screen
$ ls -l /etc/passwd /etc/shadow /dev/ppp /usr/sbin/exim4
crw------T 1 root root   108, 0 Oct 16 20:57 /dev/ppp
-rw-r--r-- 1 root root     2761 Aug 30 10:38 /etc/passwd
-rw-r----- 1 root shadow   1695 Aug 30 10:38 /etc/shadow
-rwsr-xr-x 1 root root   973824 Sep 23 20:04 /usr/sbin/exim4
$ ls -ld /tmp /var/tmp /usr/local /var/mail /usr/src
drwxrwxrwt 14 root root  20480 Oct 16 21:25 /tmp
drwxrwsr-x 10 root staff  4096 Sep 29 22:50 /usr/local
drwxr-xr-x 10 root root   4096 Oct 11 00:28 /usr/src
drwxrwsr-x  2 root mail   4096 Oct 15 21:40 /var/mail
drwxrwxrwt  3 root root   4096 Oct 16 21:20 /var/tmp
```

`chmod`{.literal}(1)有另一种数值模式来描述文件权限。这种数字模式使用3到4位八进制（底为8）数。

你能够增加一个用户 `penguin`{.literal} 到 `bird`{.literal} 组，在两个步骤内：

::: itemizedlist

    ::: itemizedlist

    -   冷重启再登录。（最佳选项）

::: itemizedlist
-   使用下面中的一个改变组配置：

    ::: itemizedlist
    -   执行 \"`sudo usermod -rG bird penguin`{.literal}\"。

    -   执行 \"`sudo deluser penguin bird`{.literal}\"。（只在典型的 Debian 系统）

-   使用下面中的一个来应用配置：

    ::: itemizedlist
    -   冷重启再登录。（最佳选项）

### []{#_links}1.2.7. 链接 {#链接 .title}

</div>

</div>
:::::

有两种方法把一个文件 "`foo`{.literal}" 链接到一个不同的文件名 "`bar`{.literal}"。

::: itemizedlist
-   [硬链接](https://zh.wikipedia.org/wiki/Hard_link){.ulink}

    ::: itemizedlist
    -   对现有文件重复名称

    -   "`ln foo bar`{.literal}"
    :::

-   [符号链接或 symlink](https://zh.wikipedia.org/wiki/Symbolic_link){.ulink}

    ::: itemizedlist
    -   通过名字指向另一个文件的特殊文件

    -   "`ln -s foo bar`{.literal}"
    :::
:::

请参阅下面的示例，`rm`{.literal} 命令结果中链接数的变化和细微的差别。

``` screen
$ umask 002
$ echo "Original Content" > foo
$ ls -li foo
1449840 -rw-rw-r-- 1 penguin penguin 17 Oct 16 21:42 foo
$ ln foo bar     # hard link
$ ln -s foo baz  # symlink
$ ls -li foo bar baz
1449840 -rw-rw-r-- 2 penguin penguin 17 Oct 16 21:42 bar
1450180 lrwxrwxrwx 1 penguin penguin  3 Oct 16 21:47 baz -> foo
1449840 -rw-rw-r-- 2 penguin penguin 17 Oct 16 21:42 foo
$ rm foo
$ echo "New Content" > foo
$ ls -li foo bar baz
1449840 -rw-rw-r-- 1 penguin penguin 17 Oct 16 21:42 bar
1450180 lrwxrwxrwx 1 penguin penguin  3 Oct 16 21:47 baz -> foo
1450183 -rw-rw-r-- 1 penguin penguin 12 Oct 16 21:48 foo
$ cat bar
Original Content
$ cat baz
New Content
```

硬链接可以在同一个文件系统内创建，并共用同一个 inode 号，由`ls`{.literal}(1)带 "`-i`{.literal}"选项显示。

符号链接总是名义上具有"`rwxrwxrwx`{.literal}"的文件访问权限，如上面例子所示，实际的有效访问权限由它所指向的文件确定。

::: {.caution style="margin-left: 0.5in; margin-right: 0.5in;"}
  --------------------------------- ----------------------------------------------------------------------------------------------------------------------------------
   ![\[小心\]](images/caution.png)  小心
                                    除非你有非常好的理由，否则不要创建一个复杂的符号链接或硬链接通常是个好主意。符号链接的逻辑组合可能导致文件系统噩梦般的无限循环。
  --------------------------------- ----------------------------------------------------------------------------------------------------------------------------------
:::

::: {.note style="margin-left: 0.5in; margin-right: 0.5in;"}
  ------------------------------ --------------------------------------------------------------------
   ![\[注意\]](images/note.png)  注意
                                 通常使用符号链接比使用硬链接更合适，除非你有一个好理由使用硬链接。
  ------------------------------ --------------------------------------------------------------------
:::

"`.`{.literal}"目录链接到它所在的目录，因此任何新建目录的链接数从2开始。"`..`{.literal}"目录链接到父目录，因此目录的链接数随着新的子目录的创建而增加。

如果你刚从 Windows 迁移到Linux，你很快将清楚 Unix 的文件名链接相较于 Windows 最相近的"快捷方式"是多么精心设计的。由于它是在文件系统中实现的，应用无法看到链接文件跟原始文件之间的区别。在硬链接这种情况，这真的是毫无差别。

### []{#_named_pipes_fifos}1.2.8. 命名管道（先进先出） {#命名管道先进先出 .title}

</div>

</div>
:::::

[命名管道](https://zh.wikipedia.org/wiki/Named_pipe){.ulink}是一个像管道一样的文件。你把内容放进了文件，它从另一端出来。因此，它被称为FIFO，即先进先出：你从管道这端先放进去的东西会从另一端先出来。

如果对一个命名管道进行写入操作，写入的过程不会被终止，直到写入的信息从管道中被读取出来。读取过程将会持续到没有信息可以读取为止。管道的大小始终是零，它不存储数据，它只是连接两个过程，像 shell 提供的 \"`|`{.literal}\" 语法功能一样。然而，一旦管道有了名称，这两个进程就可以不必在同一个命令行，甚至由同一个用户运行。管道是 Unix 的一个非常有影响力的创新。

尝试下列例子

``` screen
$ cd; mkfifo mypipe
$ echo "hello" >mypipe & # put into background
[1] 8022
$ ls -l mypipe
prw-rw-r-- 1 penguin penguin 0 Oct 16 21:49 mypipe
$ cat mypipe
hello
[1]+  Done                    echo "hello" >mypipe
$ ls mypipe
mypipe
$ rm mypipe
```

</div>
:::::

套接字被广泛应用于所有的互联网通信，数据库和操作系统本身。它类似于命名管道（FIFO）并且允许进程之间甚至不同计算机之间进行信息交换。对于套接字，这些进程不需要在同一时间运行，也不需要是同一个父进程的子进程。它是[进程间通信（IPC）](https://zh.wikipedia.org/wiki/Inter-process_communication){.ulink}的一个节点。信息的交换可能会通过网络发生在不同主机之间。最常见的两种是 [互联网套接字](https://zh.wikipedia.org/wiki/Internet_socket){.ulink} 和 [UNIX域套接字](https://zh.wikipedia.org/wiki/Unix_domain_socket){.ulink} 。

::: {.tip style="margin-left: 0.5in; margin-right: 0.5in;"}
  ----------------------------- ---------------------------------------------------------------------------------
   ![\[提示\]](images/tip.png)  提示
                                通过 \"`netstat -an`{.literal}\" 命令可以很方便的查看系统已经打开了哪些套接字。
  ----------------------------- ---------------------------------------------------------------------------------
:::

### []{#_device_files}1.2.10. 设备文件 {#设备文件 .title}

</div>

</div>
:::::

[设备文件](https://zh.wikipedia.org/wiki/Device_file){.ulink}包括系统的物理设备和虚拟设备，如硬盘、显卡、显示屏、键盘。虚拟设备的一个例子是控制台，用"`/dev/console`{.literal}"来描述。

设备文件有两种类型。

::: itemizedlist
-   [**字符设备**]{.strong}

    ::: itemizedlist
    -   每次访问一个字符

    -   一个字符等于一个字节

    -   如键盘、串口...
    :::

-   [**块设备**]{.strong}

    ::: itemizedlist
    -   通过更大的单元--块，进行访问

    -   一个块\>一个字节

    -   如硬盘等...
    :::
:::

你可以读写块设备文件，尽管该文件可能包含二进制数据，读取后显示出无法理解的乱码。向文件写入数据，有时可以帮助定位硬件连接故障。比如，你可以将文本文件导入打印机设备"`/dev/lp0`{.literal}"，或者将调制解调命令发送到合适的串口"`/dev/ttyS0`{.literal}"。但是，除非这些操作都小心完成，否则可能会导致一场大灾难。所以要特别小心。

::: {.note style="margin-left: 0.5in; margin-right: 0.5in;"}
  ------------------------------ -----------------------------------------
   ![\[注意\]](images/note.png)  注意
                                 常规访问打印机，使用`lp`{.literal}(1)。
  ------------------------------ -----------------------------------------
:::

设备的节点数可以通过执行`ls`{.literal}(1)得到，如下所示。

``` screen
$ ls -l /dev/sda /dev/sr0 /dev/ttyS0 /dev/zero
brw-rw---T  1 root disk     8,  0 Oct 16 20:57 /dev/sda
brw-rw---T+ 1 root cdrom   11,  0 Oct 16 21:53 /dev/sr0
crw-rw---T  1 root dialout  4, 64 Oct 16 20:57 /dev/ttyS0
crw-rw-rw-  1 root root     1,  5 Oct 16 20:57 /dev/zero
```

::: itemizedlist
-   \"`/dev/sda`{.literal}\"的主设备号是8，次设备号是0。它可以被`disk`{.literal}群组的用户读写。

-   \"`/dev/sr0`{.literal}\"的主设备号是11，次设备号是0。它可以被`cdrom`{.literal}群组的用户读写。

-   \"`/dev/ttyS0`{.literal}\"的主设备号是4，次设备号是64。它可以被`dailout`{.literal}群组的用户读写。

-   \"`/dev/zero`{.literal}\"的主设备号是1，次设备号是5。它可以被任意用户读写。
:::

在现代Linux系统中，处在\"`/dev`{.literal}\"之下的文件系统会自动被`udev`{.literal}()机制填充。

### []{#_procfs_and_sysfs}1.2.12. procfs 和 sysfs {#procfs-和-sysfs .title}

</div>

</div>
:::::

[procfs](https://zh.wikipedia.org/wiki/Procfs){.ulink}和[sysfs](https://zh.wikipedia.org/wiki/Sysfs){.ulink}两个伪文件系统，分别加载于\"`/proc`{.literal}\"和\"`/sys`{.literal}\"之上，将内核中的数据结构暴露给用户空间。或者说，这些条目是虚拟的，他们打开了深入了解操作系统运行的方便之门。

目录\"`/proc`{.literal}\"为每个正在运行的进程提供了一个子目录，目录的名字就是进程标识符（PID）。需要读取进程信息的系统工具，如`ps`{.literal}()，可以从这个目录结构获得信息。

\"`/proc/sys`{.literal}\"之下的目录，包含了可以更改某些内核运行参数的接口。（你也可以使用专门的`sysctl`{.literal}()命令修改，或者使用其预加载/配置文件\"`/etc/sysctl.d/*.conf`{.literal}\"。）

当人们看到这个特别大的文件\"`/proc/kcore`{.literal}\"时，常常会惊慌失措。这个文件于你的的电脑内存大小相差不多。它被用来调试内核。它是一个虚拟文件，指向系统内存，所以不必担心它的大小。

\"`/sys`{.literal}\"以下的目录包含了内核输出的数据结构，它们的属性，以及它们之间的链接。它同时也包含了改变某些内核运行时参数的接口。

参考\"`proc.txt(.gz)`{.literal}\"，\"`sysfs.txt(.gz)`{.literal}\"，以及其他相关的Linux内核文档（\"`/usr/share/doc/linux-doc-*/Documentation/filesystems/*`{.literal}\"），这些文件由`linux-doc-*`{.literal}软件包提供。

### []{#_tmpfs}1.2.13. tmpfs {#tmpfs .title}

</div>

</div>
:::::

[tmpfs](https://zh.wikipedia.org/wiki/Tmpfs#Linux){.ulink}是一个临时文件系统，它的文件都保存在[虚拟内存](https://zh.wikipedia.org/wiki/Virtual_memory){.ulink}中。必要时，位于内存[页缓存](https://zh.wikipedia.org/wiki/Page_cache){.ulink}的tmpfs数据可能被交换到硬盘中的[交换分区](https://zh.wikipedia.org/wiki/Paging){.ulink}。

系统启动早期阶段，\"`/run`{.literal}\"目录挂载为tmpfs。这样即使\"`/`{.literal}\"挂载为只读，它也是可以被写入的。它为过渡态文件提供了新的存储空间，同时也替代了[Filesystem Hierarchy Standar](https://zh.wikipedia.org/wiki/Filesystem_Hierarchy_Standard){.ulink}2.3版中说明的目录位置：

::: itemizedlist
-   \"`/var/run`{.literal}\" → \"`/run`{.literal}\"

-   \"`/var/lock`{.literal}\" → \"`/run/lock`{.literal}\"

-   \"`/dev/shm`{.literal}\" → \"`/run/shm`{.literal}\"
:::

参考\"`tmpfs.txt(.gz)`{.literal}\", 文件位于Linux内核文档（\"`/usr/share/doc/linux-doc-*/Documentation/filesystems/*`{.literal}\"）目录之下，由软件包`linux-doc-*`{.literal}提供。

### []{#_the_text_editor}1.4.6. 文本编辑器 {#文本编辑器 .title}

</div>

</div>
:::::

在使用类 Unix 系统过程中， 各种类似于 [Vim](https://zh.wikipedia.org/wiki/Vim_(text_editor)){.ulink} 或 [Emacs](https://zh.wikipedia.org/wiki/Emacs){.ulink} 的工具，你应该精通其中的一个。

我认为习惯于使用 Vim 命令是一个明智的选择，因为 Linux/Unix 系统里一般都附带了 Vi 编辑器。 (实际上最初的 `vi`{.literal} 以及后来的 `nvi`{.literal} 这类工具程序很常见。因为在 Vim 里提供了`F1`{.literal}帮助键，在同类工具中它的功能更强大，所以我选择 Vim 而不是其它新出的一些工具。)

假设你不是用 [Emacs](https://zh.wikipedia.org/wiki/Emacs){.ulink} 就是用 [XEmacs](https://zh.wikipedia.org/wiki/XEmacs){.ulink} 作为你的编辑器，其实还有更好的选择，尤其是在编程的时候。 Emacs 还有很多其他的特点，包括新手导读，目录编辑器，邮件客户端等等。当编写脚本或程序的时候，它能自动识别当前工作模式所对应的格式，让使用更加便利。一些人甚至坚持认为 Linux 系统里最需要配备的就是 Emacs。花十分钟来学习 Emacs 可以为后面的工作剩下更多时间。在此强烈推荐学习使用 Emacs 时候直接使用 GNU Emacs 参考手册。

在实践应用中所有这些程序都会有一个教程，输入 \"`vim`{.literal}\" 和F1键就可以启动Vim。建议你最好阅读一下前面的 35 行。移动光标到 \"`|tutor|`{.literal}\" 并按 `Ctrl-]`{.literal} 就可以看到在线培训教程。

### []{#_the_literal_lang_literal_variable}1.5.2. "`$LANG`{.literal}"变量 {#lang变量 .title}

</div>

默认的语言环境是在 \"`$LANG`{.literal}\" 环境变量中定义，它在安装的时候配置为 \"`LANG=xx_YY.UTF-8`{.literal}\"，或者在接下来的 GUI（图形用户界面）中配置，例如在 GNOME 中是，\"设置\" → \"区域 & 语言\" → \"语言\" / \"格式\"。

使用 shell 命令行按顺序执行下列典型的命令。

``` screen
$ echo $LANG
en_US.UTF-8
$ date -u
Wed 19 May 2021 03:18:43 PM UTC
$ LANG=fr_FR.UTF-8 date -u
mer. 19 mai 2021 15:19:02 UTC
```

这里，`date`{.literal}(1)程序执行时使用了不同的语言环境值。

::: itemizedlist
-   第一个命令，"`$LANG`{.literal}" 设置为系统的默认[语言环境](https://zh.wikipedia.org/wiki/Locale){.ulink}值 "`en_US.UTF-8`{.literal}"。

-   第二个命令，"`$LANG`{.literal}"设置为法语的 UTF-8 [语言环境](https://zh.wikipedia.org/wiki/Locale){.ulink}值"`fr_FR.UTF-8`{.literal}"。
:::

``` screen
$ LANG=fr_FR.UTF-8
$ date -u
mer. 19 mai 2021 15:19:24 UTC
```

### []{#_the_literal_path_literal_variable}1.5.3. \"`$PATH`{.literal}\" 变量 {#path-变量 .title}

可以在 Bash 脚本文件 \"`~/.bash_profile`{.literal}\" 或 \"`~/.bashrc`{.literal}\" 中改变 \"`$PATH`{.literal}\" 环境变量的值。

### []{#_regular_expressions}1.6.2. 正则表达式 {#正则表达式 .title}

</div>

</div>
:::::

[正则表达式](https://zh.wikipedia.org/wiki/Regular_expression){.ulink}被使用在许多文本处理工具中。它们类似 shell 的通配符，但更加复杂和强大。

正则表达式描述要匹配的模式，它是由文本字符和[**元字符**]{.strong}构成的。

[**元字符**]{.strong}仅仅是带有特殊含义的字符。它们有两种主要的形式，[**BRE**]{.strong} 和 [**ERE**]{.strong} ，使用哪种取决于上述的文本工具。

:::: table
[]{#metacharactersforbreandere}

**表 1.25. BRE 和 ERE 中的元字符**

::: table-contents
  BRE                                ERE                         正则表达式的描述
  ---------------------------------- --------------------------- --------------------------------------------------------------------------------------------
  `\ . [ ] ^ $ *`{.literal}          `\ . [ ] ^ $ *`{.literal}   通用的[**元字符**]{.strong}
  `\+ \? \( \) \{ \} \|`{.literal}                               BRE 独有的"`\`{.literal}"转义[**元字符**]{.strong}
                                     `+ ? ( ) { } |`{.literal}   ERE 独有的不需要"`\`{.literal}"转义的[**元字符**]{.strong}
  `c`{.literal}                      `c`{.literal}               匹配[**非元字符**]{.strong} "`c`{.literal}"
  `\c`{.literal}                     `\c`{.literal}              匹配一个字面意义上的字符 "`c`{.literal}"，即使 "`c`{.literal}" 本身是[**元字符**]{.strong}
  `.`{.literal}                      `.`{.literal}               匹配任意字符，包括换行符
  `^`{.literal}                      `^`{.literal}               字符串的开始位置
  `$`{.literal}                      `$`{.literal}               字符串的结束位置
  `\<`{.literal}                     `\<`{.literal}              单词的开始位置
  `\>`{.literal}                     `\>`{.literal}              单词的结束位置
  `[abc…]`{.literal}                 `[abc…]`{.literal}          匹配在 "`abc...`{.literal}" 中的任意字符
  `[^abc…]`{.literal}                `[^abc…]`{.literal}         匹配除了 "`abc...`{.literal}" 中的任意字符
  `r*`{.literal}                     `r*`{.literal}              匹配零个或多个 "`r`{.literal}"
  `r\+`{.literal}                    `r+`{.literal}              匹配一个或多个 "`r`{.literal}"
  `r\?`{.literal}                    `r?`{.literal}              匹配零个或一个 "`r`{.literal}"
  `r1\|r2`{.literal}                 `r1|r2`{.literal}           匹配一个 "`r1`{.literal}" 或 "`r2`{.literal}"
  `\(r1\|r2\)`{.literal}             `(r1|r2)`{.literal}         匹配一个 "`r1`{.literal}" 或 "`r2`{.literal}" ，并作为[**括号**]{.strong}内的正则表达式

</div>

`ed`{.literal}(1) 命令可以在 "`file`{.literal}" 中将所有的 "`FROM_REGEX`{.literal}" 替换成 "`TO_TEXT`{.literal}" 。

``` screen
$ ed file <<EOF
,s/FROM_REGEX/TO_TEXT/g
w
q
EOF
```

`sed`{.literal}(1) 命令可以在 "`file`{.literal}" 中将所有的 "`FROM_REGEX`{.literal}" 替换成 "`TO_TEXT`{.literal}" 。

``` screen
$ sed -i -e 's/FROM_REGEX/TO_TEXT/g' file
```

`vim`{.literal}(1) 命令可以通过使用 `ex`{.literal}(1) 命令在 "`file`{.literal}" 中将所有的 "`FROM_REGEX`{.literal}" 替换成 "`TO_TEXT`{.literal}" 。

``` screen
$ vim '+%s/FROM_REGEX/TO_TEXT/gc' '+update' '+q' file
```

::: {.tip style="margin-left: 0.5in; margin-right: 0.5in;"}
  ----------------------------- ---------------------------------------------------------------------
   ![\[提示\]](images/tip.png)  提示
                                上面的 "`c`{.literal}" 标志可以确保在每次替换时都进行交互式的确认。
  ----------------------------- ---------------------------------------------------------------------
:::

多个文件（ "`file1`{.literal}"，"`file2`{.literal}" 和 "`file3`{.literal}" ）可以使用 `vim`{.literal}(1) 或 `perl`{.literal}(1) 通过正则表达式进行类似的处理。

``` screen
$ vim '+argdo %s/FROM_REGEX/TO_TEXT/gce|update' '+q' file1 file2 file3
```

::: {.tip style="margin-left: 0.5in; margin-right: 0.5in;"}
  ----------------------------- -----------------------------------------------------------------
   ![\[提示\]](images/tip.png)  提示
                                上面的 "`e`{.literal}" 标志是为了防止 "No match" 错误中断替换。
  ----------------------------- -----------------------------------------------------------------
:::

``` screen
$ perl -i -p -e 's/FROM_REGEX/TO_TEXT/g;' file1 file2 file3
```

在 perl(1)例子中 , \"`-i`{.literal}\" 是在每一个目标文件的原处编辑，\"`-p`{.literal}\" 是表示循环所有给定的文件。

::: {.tip style="margin-left: 0.5in; margin-right: 0.5in;"}
  ----------------------------- --------------------------------------------------------------------------------------------------------------------------------------------------
   ![\[提示\]](images/tip.png)  提示
                                使用参数 "`-i.bak`{.literal}" 代替 "`-i`{.literal}" ，可以在文件名后添加 "`.bak`{.literal}" 再保存。对于复杂的替换，这使得从错误中恢复变得容易。
  ----------------------------- --------------------------------------------------------------------------------------------------------------------------------------------------
:::

::: {.note style="margin-left: 0.5in; margin-right: 0.5in;"}
  ------------------------------ -------------------------------------------------------------------------------------------------------------------
   ![\[注意\]](images/note.png)  注意
                                 `ed`{.literal}(1) 和 `vim`{.literal}(1) 使用 [**BRE**]{.strong} ； `perl`{.literal}(1) 使用 [**ERE**]{.strong} 。
  ------------------------------ -------------------------------------------------------------------------------------------------------------------
:::

### []{#_extracting_data_from_text_file_table}1.6.5. 从文本文件的表格中提取数据 {#从文本文件的表格中提取数据 .title}

</div>

</div>
:::::

下面有一个文本文件 "`DPL`{.literal}" ，里面含有 2004 年以前 Debian 项目的领导者名字和起始日期，并以空格分隔。

``` screen
Ian     Murdock   August  1993
Bruce   Perens    April   1996
Ian     Jackson   January 1998
Wichert Akkerman  January 1999
Ben     Collins   April   2001
Bdale   Garbee    April   2002
Martin  Michlmayr March   2003
```

::: {.tip style="margin-left: 0.5in; margin-right: 0.5in;"}
  ----------------------------- ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
   ![\[提示\]](images/tip.png)  提示
                                参见 ["Debian 简史"](https://www.debian.org/doc/manuals/project-history/){.ulink} 获取最新的 [Debian 领导阶层历史](https://www.debian.org/doc/manuals/project-history/leaders){.ulink}。
  ----------------------------- ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
:::

Awk 经常被用来从这种类型的文件中提取数据。

尝试下列例子

``` screen
$ awk '{ print $3 }' <DPL                   # month started
August
April
January
January
April
April
March
$ awk '($1=="Ian") { print }' <DPL          # DPL called Ian
Ian     Murdock   August  1993
Ian     Jackson   January 1998
$ awk '($2=="Perens") { print $3,$4 }' <DPL # When Perens started
April 1996
```

Shell （例如 Bash ）也可以用来分析这种文件。

尝试下列例子

``` screen
$ while read first last month year; do
    echo $month
  done <DPL
... same output as the first Awk example
```

内建命令 `read`{.literal} 使用 "`$IFS`{.literal}" （内部域分隔符）中的字符来将行分隔成多个单词。

如果你将 "`$IFS`{.literal}" 改变为 "`:`{.literal}" ，你可以很好地使用 shell 来分析 "`/etc/passwd`{.literal}"。

``` screen
$ oldIFS="$IFS"   # save old value
$ IFS=':'
$ while read user password uid gid rest_of_line; do
    if [ "$user" = "bozo" ]; then
      echo "$user's ID is $uid"
    fi
  done < /etc/passwd
bozo's ID is 1000
$ IFS="$oldIFS"   # restore old value
```

（如果要用 Awk 做到相同的事，使用 "`FS=':'`{.literal}" 来设置域分隔符。）

IFS 也被 shell 用来分割参数扩展、命令替换和算术扩展的结果。这不会出现在双引号或单引号中。 IFS 的默认值为 [*空格*]{.emphasis}、[*tab*]{.emphasis} 和[*换行符*]{.emphasis}。

请谨慎使用 shell 的 IFS 技巧。当 shell 将脚本的一部分解释为对它的[**输入**]{.strong}时，会发生一些奇怪的事。

``` screen
$ IFS=":,"                        # use ":" and "," as IFS
$ echo IFS=$IFS,   IFS="$IFS"     # echo is a Bash builtin
IFS=  , IFS=:,
$ date -R                         # just a command output
Sat, 23 Aug 2003 08:30:15 +0200
$ echo $(date -R)                 # sub shell --> input to main shell
Sat  23 Aug 2003 08 30 36 +0200
$ unset IFS                       # reset IFS to the default
$ echo $(date -R)
Sat, 23 Aug 2003 08:30:50 +0200
```
