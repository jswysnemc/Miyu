## []{#_gui_communication_protocol}7.2. GUI（图形用户界面）通信协议 {#gui图形用户界面通信协议 .title}

</div>

</div>
:::::

在 GNOME 桌面使用的 GUI 通信协议可以为：

::: itemizedlist
-   [Wayland (服务端显示协议)](https://zh.wikipedia.org/wiki/Wayland_(display_server_protocol)){.ulink} (原生)

-   [X 窗口系统核心协议](https://zh.wikipedia.org/wiki/X_Window_System_core_protocol){.ulink} (通过 `xwayland`{.literal})
:::

请查看 [freedesktop.org 站点来了解 Wayland 架构和 X 窗口架构是如何不同](https://wayland.freedesktop.org/architecture.html){.ulink}。

从用户的观点，不同能够被通俗的概况为：

::: itemizedlist
-   Wayland 是在同一个主机上的 GUI 通信协议：新、简单、快速，不需要 setuid root 二进制

-   X Window 是一个具备网络功能的 GUI 通信协议：传统、复杂、慢，需要 setuid root 二进制
:::

对于使用 Wayland 协议的应用，由 [VNC](https://zh.wikipedia.org/wiki/Virtual_Network_Computing){.ulink} 或 [RDP](https://zh.wikipedia.org/wiki/Remote_Desktop_Protocol){.ulink} 来支持从一个远程主机上访问它们显示的内容。参见 [第 7.8 节 "远程桌面"](ch07.zh-cn.html#_remote_desktop "7.8. 远程桌面"){.xref}

现代 X 服务器具有 [MIT 共享内存扩展](https://zh.wikipedia.org/wiki/MIT-SHM){.ulink} ，他们和本地 X 客户端通过本地共享内存进行通讯. 这就绕过了网络透明的 [Xlib](https://zh.wikipedia.org/wiki/Xlib){.ulink} 进程间通讯通道，获得了性能。这个情况，也是创建 Wayland 作为本地 GUI 通信协议的[背景](https://wayland.freedesktop.org/faq.html){.ulink}。

使用从 GNOME 终端启动的 `xeyes`{.literal} 程序，你能够检查每个 GUI（图形用户界面）应用程序使用的 GUI 通信协议。

``` screen
 $ xeyes
```

::: itemizedlist
-   如果鼠标是在使用 Wayland 服务端显示协议的应用程序上，比如"GNOME 终端"，眼睛不会跟随鼠标移动。

-   如果鼠标是在使用 X 窗口系统核心协议的应用程序上，比如"xterm"，眼睛会跟随鼠标移动，暴露出不是那么孤立的 X 窗口架构的特性。
:::

到 2021 年 4 月，许多流行的 GUI 应用程序，比如 GNOME 和 [LibreOffice (LO)](https://zh.wikipedia.org/wiki/LibreOffice){.ulink} 已经被移植到了 Wayland 服务端显示协议。我发现 `xterm`{.literal}, `gitk`{.literal}, `chromium`{.literal}, `firefox`{.literal}, `gimp`{.literal}, `dia`{.literal} 和 KDE 应用程序任然使用 X 窗口系统核心协议。

::: {.note style="margin-left: 0.5in; margin-right: 0.5in;"}
  ------------------------------ ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
   ![\[注意\]](images/note.png)  注意
                                 对于 Wayland 之上的 xwayland 或 原生的 X 窗口系统，这两个上面的旧的 X 服务端配置文件 \"`/etc/X11/xorg.conf`{.literal}\" 不应当在系统上存在。显卡和输入设备目前是由内核的 [DRM](https://zh.wikipedia.org/wiki/Direct_Rendering_Manager){.ulink}、[KMS](https://wiki.debian.org/KernelModesetting){.ulink} 和 [udev](https://zh.wikipedia.org/wiki/Udev){.ulink} 配置。原生的 X 服务端已经重写来使用它们。参见 Linux 内核文档的 \"[modedb default video mode support](https://www.kernel.org/doc/html/latest/fb/modedb.html){.ulink}\"。
  ------------------------------ ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
:::

## []{#_user_directories}7.5. 用户目录 {#用户目录 .title}

</div>

</div>
:::::

用户目录，比如\"`~/Desktop`{.literal}\"、\"`~/Documents`{.literal}\"......，其默认的名字，使用系统安装时桌面环境的语言。你能够把它们重置为英语：

``` screen
 $ LANGUAGE=C xdg-user-dirs-update --force 
```

然后你手工移到所有数据到新的目录。参加 `xdg-user-dirs-update`{.literal}(1)。

通过编辑 \"`~/.config/user-dirs.dirs`{.literal}\"，你也能够把它们设置为任何名字。参见 `user-dirs.dirs`{.literal}(5)。

## []{#_fonts}7.6. 字体 {#字体 .title}

</div>

</div>
:::::

对于 Debian 的用户，有许多有用的矢量字体存在。用户关注是怎样避免冗余，怎样配置禁用部分已经安装的字体。此外，无用的字体选择可以搞乱你的 GUI（图形用户界面）应用程序菜单。

Debian 系统使用 [FreeType](https://freetype.org){.ulink} 2.0 库来栅格化许多矢量字体格式，用于屏幕和打印：

::: itemizedlist
-   [Type 1 (PostScript) 字体](https://zh.wikipedia.org/wiki/PostScript_fonts#Type_1){.ulink} 使用三次 [贝塞尔曲线](https://en.wikipedia.org/wiki/Bézier_curve){.ulink} (差不多废弃的格式)

-   [TrueType 字体](https://zh.wikipedia.org/wiki/TrueType){.ulink} 使用二次 [贝塞尔曲线](https://en.wikipedia.org/wiki/Bézier_curve){.ulink} (好的选择格式)

-   [OpenType 字体](https://zh.wikipedia.org/wiki/OpenType){.ulink} 使用三次 [贝塞尔曲线](https://en.wikipedia.org/wiki/Bézier_curve){.ulink} (最佳选择格式)
:::

对使用 Debian 的自由软件的狂热爱好者，这些二进制格式的分发，有一些存在的理由。因为这能够得到一个干净的库集合，由 Debian 提供的库和由每个应用程序相应的上游开发者使用的库，独立开来。

运行外部二进制的固有风险，能够使用 [沙盒环境](https://zh.wikipedia.org/wiki/Sandbox_(computer_security)){.ulink} 减少，它有现代 Linux 安全特性的手段。（参见 [第 4.7.5 节 "Linux 安全特性"](ch04.zh-cn.html#_linux_security_features "4.7.5. Linux 安全特性"){.xref}）。

-   对于从 FLATHUB 来的二进制，在 [Flatpak](https://zh.wikipedia.org/wiki/Flatpak){.ulink} 里运行它们。（不需要手工配置。）

-   对于从 snapcraft 来的二进制，在 [Snap](https://zh.wikipedia.org/wiki/Snap_(software)){.ulink} 里面运行它们。（不需要手工配置。和后台守护进程（daemon）兼容。）
:::

### []{#_x_server_remote_connection}7.9.2. X 服务端远程连接 {#x-服务端远程连接 .title}

</div>

</div>
:::::

使用 X 核心协议的远程应用访问本地的 X 服务器显示，由 X11 转发特性支持。

::: itemizedlist
-   在本地主机中打开一个 `gnome 终端`{.literal}。

-   通过下列命令，运行带 `-X`{.literal} 选项的 `ssh`{.literal}(1)，建立与远程站点的连接。

    ``` screen
    localname @ localhost $ ssh -q -X loginname@remotehost.domain
    Password:
    ```

-   通过下列命令，在远程站点运行一个 X 应用程序，例如 "`gitk`{.literal}"。

    ``` screen
    loginname @ remotehost $ gitk
    ```
:::

这个方法可以显示来自远程 X 客户端的输出，相当于它是通过一个本地 UNIX 域名套接字进行本地的连接。

参见介绍 SSH/SSHD 的 [第 6.3 节 "服务器远程访问和工具 (SSH)"](ch06.zh-cn.html#_the_remote_access_server_and_utilities_ssh "6.3. 服务器远程访问和工具 (SSH)"){.xref} 。
