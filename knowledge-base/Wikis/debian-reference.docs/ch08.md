一个应用软件的 [多语言化 (M17N)或本地语言支持](https://zh.wikipedia.org/wiki/Internationalization_and_localization){.ulink}，通过 2 个步骤完成。

::: itemizedlist
-   国际化 (I18N): 使一个软件能够处理多个语言环境。

-   本地化 (L10N)：使一个软件处理一个特定的语言环境。
:::

::: {.tip style="margin-left: 0.5in; margin-right: 0.5in;"}
  ----------------------------- ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

## []{#_the_locale}8.1. 语言环境 {#语言环境 .title}

</div>

</div>
:::::

程序支持国际化的行为，是通过配置环境变量 \"`$LANG`{.literal}\"来支持本地化。语言环境的实际支持，依赖 `libc`{.literal} 库提供的特性，并要求安装 `locales`{.literal} 或 `locales-all`{.literal} 软件包。 `locales`{.literal} 软件包需要被适当的初始化。

如果 `locales`{.literal} 或 `locales-all`{.literal} 软件包均没有安装，支持语言环境的特性丢失，系统使用 US 英语消息，并按 [**ASCII**]{.strong} 处理数据。这个行为和 \"`$LANG`{.literal}\" 被设置为 \"`LANG=`{.literal}\"、\"`LANG=C`{.literal}\" 或 \"`LANG=POSIX`{.literal}\" 相同。

GNOME 和 KDE 等现代软件是多语言的。他们通过处理 [UTF-8](https://zh.wikipedia.org/wiki/UTF-8){.ulink} 数据来实现国际化，并通过 `gettext`{.literal}(1) 架构提供翻译信息来本地化。翻译信息可以由独立的本地化软件包来提供。

目前的 Debian 桌面 GUI（图形用户界面）系统通常在 GUI 环境中设置语言环境为 \"`LANG=xx_YY.UTF-8`{.literal}\"。这里，\"`xx`{.literal}\" 是 [ISO 639 语言代码](https://zh.wikipedia.org/wiki/ISO_639-1){.ulink} ，\"`YY`{.literal}\" 是 [ISO 3166 国家地区代码](https://zh.wikipedia.org/wiki/ISO_3166-1){.ulink}。这些值由配置桌面的 GUI 对话框来设置，并改变程序的行为。参见 [第 1.5.2 节 ""`$LANG`{.literal}"变量"](ch01.zh-cn.html#_the_literal_lang_literal_variable "1.5.2. “$LANG”变量"){.xref}

### []{#_rationale_for_utf_8_locale}8.1.1. UTF-8 语言环境的基本原理 {#utf-8-语言环境的基本原理 .title}

</div>

</div>
:::::

最简单的文本数据表达是 [**ASCII**]{.strong}，它对英语是足够的，少于 127 个字符（使用 7 位描述）。

即使纯英文文本也可能包含非 ASCII 字符，例如微微卷曲的左右引号在 ASCII 中是不可用的。

``` screen
“double quoted text” is not "double quoted ASCII"
‘single quoted text’ is not 'single quoted ASCII'
```

为了支持更多字符，许多字符集和编码系统被用来支持多语言。（参见 [表 11.2 "编码值和用法的列表"](ch11.zh-cn.html#list-of-encoding-values "表 11.2. 编码值和用法的列表"){.xref}）。

[Unicode](https://zh.wikipedia.org/wiki/Unicode){.ulink} 字符集可以用 21 位码点范围来显示几乎所有人类已知的字符（例如，十六进制的 0 到 10FFFF）。

文本编码系统 [UTF-8](https://zh.wikipedia.org/wiki/UTF-8){.ulink} 将 Unicode 码点适配到一个合理的 8 位数据流，并大部分兼容 ASCII 数据处理系统。这个使 [**UTF-8**]{.strong} 作为现代推荐的选择。[**UTF**]{.strong} 表示 Unicode 转换格式（Unicode Transformation Format）。当 [ASCII](https://zh.wikipedia.org/wiki/ASCII){.ulink} 纯文本数据转换为 [UTF-8](https://zh.wikipedia.org/wiki/UTF-8){.ulink} 数据，它有和原始 ASCII 完全一样的内容和大小。所以配置 UTF-8 语言环境不会有任何丢失。

在 [UTF-8](https://zh.wikipedia.org/wiki/UTF-8){.ulink} 语言环境下兼容的应用程序，你可以显示和编辑外语文本数据，在所要求的字体和输入法安装和启用后。例如在\"`LANG=fr_FR.UTF-8`{.literal}\"语言环境下，`gedit`{.literal}(1) (GNOME 桌面的文本编辑器) 能够显示和编辑中文字符文本数据，而显示的菜单是法语。

::: {.tip style="margin-left: 0.5in; margin-right: 0.5in;"}
  ----------------------------- --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
   ![\[提示\]](images/tip.png)  提示
                                新标准的 \"`en_US.UTF-8`{.literal}\"和老标准的\"`C`{.literal}\"/\"`POSIX`{.literal}\"语言环境都使用标准的 US 英文消息，它们在排序等方面有细微的不同。在维护老的\"`C`{.literal}\"本地行为时，如果你不仅想处理 ASCII 字符，同时还想优雅的处理 UTF-8 编码的字符，在 Debian 上使用非标准的 \"`C.UTF-8`{.literal}\"语言环境。
  ----------------------------- --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
:::

::: {.note style="margin-left: 0.5in; margin-right: 0.5in;"}
  ------------------------------ ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
   ![\[注意\]](images/note.png)  注意
                                 一些程序在支持 I8N 后会消耗更多的内存。这是因为它们为了速度优化，而在内部使用 [UTF-32(UCS4)](https://zh.wikipedia.org/wiki/UTF-32/UCS-4){.ulink} 来支持 Unicode，并且每个独立于语言环境所选的 ASCII 字符数据都会消耗 4 个字节。再一次地，使用 UTF-8 语言环境并不会使你损失什么。
  ------------------------------ ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
:::

### []{#_the_reconfiguration_of_the_locale}8.1.2. 语言环境的重新配置 {#语言环境的重新配置 .title}

</div>

</div>
:::::

为了让系统访问某一语言环境，语言环境数据必须从语言环境数据库中编译。

`locales`{.literal} 软件包 [**没有**]{.strong} 包含预先编译的语言环境数据。你需要按下面的方法配置：

``` screen
# dpkg-reconfigure locales
```

该过程包含 2 个步骤。

::: orderedlist
1.  选择所有需要的语言环境数据编译为二进制形式。（请确认至少包含一个 UTF-8 语言环境 ）

2.  通过创建 "`/etc/default/locale`{.literal}" 来设置系统默认的语言环境值给 PAM 使用（参见 [第 4.5 节 "PAM 和 NSS"](ch04.zh-cn.html#_pam_and_nss "4.5. PAM 和 NSS"){.xref}）。
:::

设置在 \"`/etc/default/locale`{.literal}\" 里的系统范围的默认语言环境，可以被 GUI（图形用户界面）应用程序的 GUI 配置覆盖。

::: {.note style="margin-left: 0.5in; margin-right: 0.5in;"}
  ------------------------------ ---------------------------------------------------------------------------------------------------------------------------------------------------------------
   ![\[注意\]](images/note.png)  注意
                                 所使用的确切传统编码系统可以通过 "`/usr/share/i18n/SUPPORTED`{.literal}" 来确认。因此， \"`LANG=en_US`{.literal}\" 是 \"`LANG=en_US.ISO-8859-1`{.literal}\"。
  ------------------------------ ---------------------------------------------------------------------------------------------------------------------------------------------------------------
:::

`locales-all`{.literal} 软件包有所有预编译的语言环境数据，但是不创建\"`/etc/default/locale`{.literal}\"，你可能还需要安装 `locales`{.literal} 软件包。

::: {.tip style="margin-left: 0.5in; margin-right: 0.5in;"}
  ----------------------------- -----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
   ![\[提示\]](images/tip.png)  提示
                                一些 Debian 系发行版的 `locales`{.literal} 软件包，包含有所有语言环境的预先编译好的语言环境数据。为了模拟这样的系统环境，你需要同时在 Debian 安装 `locales`{.literal} 和 `locales-all`{.literal} 软件包。
  ----------------------------- -----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
:::

### []{#_filename_encoding}8.1.3. 文件名编码 {#文件名编码 .title}

</div>

</div>
:::::

对于跨平台的数据交换(参见 [第 10.1.7 节 "可移动存储设备"](ch10.zh-cn.html#_removable_storage_device "10.1.7. 可移动存储设备"){.xref}),你需要使用特殊的编码挂载文件系统.举个例子，不使用选项时，`mount`{.literal}(8) 假设 [vfat 文件系统](https://zh.wikipedia.org/wiki/File_Allocation_Table){.ulink} 使用 [CP437](https://zh.wikipedia.org/wiki/Code_page_437){.ulink}. 你需要给文件名提供明确的挂载选项来使用[UTF-8](https://zh.wikipedia.org/wiki/UTF-8){.ulink} 或 [CP932](https://zh.wikipedia.org/wiki/Code_page_932){.ulink}.

::: {.note style="margin-left: 0.5in; margin-right: 0.5in;"}
  ------------------------------ -----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
   ![\[注意\]](images/note.png)  注意
                                 在 GNOME 这类的现代桌面环境下，当自动挂载一个热拔插 [U 盘](https://zh.wikipedia.org/wiki/USB_flash_drive){.ulink}时，你可以提供这样的挂载选项。右击桌面上的图标，点击 \"Drive\" , \"Setting\", 输入 \"utf8\" 到 \"Mount options:\". 当这个 U 盘下次挂载时，UTF-8 就可以了。
  ------------------------------ -----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
:::

::: {.note style="margin-left: 0.5in; margin-right: 0.5in;"}
  ------------------------------ ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
   ![\[注意\]](images/note.png)  注意
                                 如果你在升级一个系统，或者从老的非 UTF-8 系统迁移磁盘，非 ASCII 字符的文件名也许是使用老旧的 [ISO-8859-1](https://zh.wikipedia.org/wiki/ISO/IEC_8859-1){.ulink} 或 [eucJP](https://zh.wikipedia.org/wiki/Extended_Unix_Code){.ulink} 来编码. 请寻求文本转换工具把他们转换到 [UTF-8](https://zh.wikipedia.org/wiki/UTF-8){.ulink}. 参见 [第 11.1 节 "文本数据转换工具"](ch11.zh-cn.html#_text_data_conversion_tools "11.1. 文本数据转换工具"){.xref}.
  ------------------------------ ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
:::

在默认情况下，[Samba](https://zh.wikipedia.org/wiki/Samba_(software)){.ulink} 对新的客户端(Windows NT, 200x, XP)使用 Unicode，但对老的客户端(DOS 和 Windows 9x/Me)使用 [CP850](https://zh.wikipedia.org/wiki/Code_page_850){.ulink}.可以在 \"`/etc/samba/smb.conf`{.literal}\" 文件里面，使用\"`dos charset`{.literal}\" 来改变老客户端的这个默认编码。比如说，[CP932](https://zh.wikipedia.org/wiki/Code_page_932){.ulink} 表示为日语。

### []{#_localized_messages_and_translated_documentation}8.1.4. 本地化信息和翻译文档 {#本地化信息和翻译文档 .title}

</div>

</div>
:::::

在 Debian 系统中显示的许多文档和文本信息有翻译存在，比如错误信息、标准程序输出、菜单和手册页。[GNU gettext(1) 命令工具链](https://zh.wikipedia.org/wiki/Gettext){.ulink}是大部分翻译活动的后端工具。

`aptitude`{.literal}(8) 里，\"Tasks\" → \"Localization\" 提供一个有用的二进制包扩展列表， 给应用程序增加本地化信息和提供翻译文档。

举个例子，你可以安装 `manpages-`{.literal}[*`LANG`{.literal}*]{.emphasis} 包来获得本地化 man 手册页信息。从 \"`/usr/share/man/it/`{.literal}\" 来读取 [*programname*]{.emphasis} 意大利语的 man 手册页 ,执行下面的操作。

``` screen
LANG=it_IT.UTF-8 man programname
```

通过 `$LANGUAGE`{.literal} 环境变量，GNU gettext 能够适应翻译语言的优先级列表。例如：

``` screen
 $ export LANGUAGE="pt:pt_BR:es:it:fr"
```

获取更多信息，参见 `info gettext`{.literal} ，阅读 \"The LANGUAGE variable\" 章节。

### []{#_effects_of_the_locale}8.1.5. 语言环境的影响 {#语言环境的影响 .title}

</div>

</div>
:::::

`sort`{.literal}(1) 和 `ls`{.literal}(1)的字符排序受语言环境 locale 影响。导出 `LANG=en_US.UTF-8`{.literal}，排序用字典 `A->a->B->b...->Z->z`{.literal} 顺序，而导出 `LANG=C.UTF-8`{.literal}，排序使用 ASCII 二进制 `A->B->...->Z->a->b...`{.literal} 顺序。

`ls`{.literal}(1) 的日期格式受语言环境影响(参见 [第 9.3.4 节 "定制时间和日期的显示"](ch09.zh-cn.html#_customized_display_of_time_and_date "9.3.4. 定制时间和日期的显示"){.xref})。

`date`{.literal}(1)程序的日期格式受语言环境的影响。例如：

``` screen
 $ unset LC_ALL
 $ LANG=en_US.UTF-8 date
Thu Dec 24 08:30:00 PM JST 2023
 $ LANG=en_GB.UTF-8 date
Thu 24 Dec 20:30:10 JST 2023
 $ LANG=es_ES.UTF-8 date
jue 24 dic 2023 20:30:20 JST
 $ LC_TIME=en_DK.UTF-8 date
2023-12-24T20:30:30 JST
```

不同语言环境的数字标点不一样。比如说，英语语言环境中, 一千点一显示为 \"`1,000.1`{.literal}\"，而在德语语言环境中,它显示为 \"`1.000,1`{.literal}\"。你可以在电子表格程序里面看到这个不同。

\"`$LANG`{.literal}\" 环境变量的每一个细节特征能够通过设置 \"`$LC_*`{.literal}\" 变量来覆盖。这些环境变量又能够通过设置 "`$LC_ALL`{.literal}"变量而被再次覆盖。细节参见 `locale`{.literal}(7) man 手册页。除非你有强烈的理由创建复杂的配置，请远离他们并只使用 \"`$LANG`{.literal}\" 变量来设置一个 UTF-8 语言环境。

### []{#_the_keyboard_input_for_linux_console_and_x_window}8.2.1. Linux 控制台和 X 窗口的键盘输入 {#linux-控制台和-x-窗口的键盘输入 .title}

</div>

</div>
:::::

Debian 系统可以使用 `keyboard-configuration`{.literal} 和 `console-setup`{.literal} 软件包配置多个国际化键盘布局。

``` screen
# dpkg-reconfigure keyboard-configuration
# dpkg-reconfigure console-setup
```

对于 Linux 控制台和 X 窗口系统，这将更新在 \"`/etc/default/keyboard`{.literal}\" 和 \"`/etc/default/console-setup`{.literal}\" 里的配置参数。这个也会配置 Linux 控制台字体。许多非 ASCII 字符，包括许多欧洲语言使用的重音字符，可以使用 [死键](https://zh.wikipedia.org/wiki/Dead_key){.ulink}、[AltGr 键](https://en.wikipedia.org/wiki/AltGr_key){.ulink} 和 [组合键](https://zh.wikipedia.org/wiki/Compose_key){.ulink} 来输入它们。

### []{#_the_keyboard_input_for_wayland}8.2.2. Wayland 键盘输入 {#wayland-键盘输入 .title}

</div>

Wayland 桌面系统上的 GNOME, [第 8.2.1 节 "Linux 控制台和 X 窗口的键盘输入"](ch08.zh-cn.html#_the_keyboard_input_for_linux_console_and_x_window "8.2.1. Linux 控制台和 X 窗口的键盘输入"){.xref} 不支持非英语的欧洲语言。[IBus](https://zh.wikipedia.org/wiki/Intelligent_Input_Bus){.ulink} 不仅支持亚洲语言，也支持欧洲语言。GNOME 桌面环境的软件包依赖关系通过\"`gnome-shell`{.literal}\"推荐\"`ibus`{.literal}\" 。\"`ibus`{.literal}\" 的代码已经更新集成 `setxkbmap`{.literal} 和 XKB 选项功能。对多语言键盘输入，你需要从 \"GNOME Settings\" 或 \"GNOME Tweaks\" 配置 `ibus`{.literal}。

### []{#_an_example_for_japanese}8.2.4. 一个日语的例子 {#一个日语的例子 .title}

</div>

</div>
:::::

我发现在英语环境(\"`en_US.UTF-8`{.literal}\")下启动日文输入法非常有用. 下面是在 Wayland 上的 GNOME下使用 IBus 的做法:

::: orderedlist
1.  安装日文输入法软件包 `ibus-mozc`{.literal}（或 `ibus-anthy`{.literal} ），以及 `im-config`{.literal} 等推荐的软件包。

2.  选择 \"Settings\" → \"Keyboard\" → \"Input Sources\" → 在\"Input Sources\"中单击 \"`+`{.literal}\" → \"Japanese\" → \"Japanese mozc (or anthy)\"， 然后单击\"Add\"。如果它没有被激活。

3.  你可以选择许多输入源。

4.  重新登录用户账户。

5.  右键单击 GUI 工具条图标，设置每一个输入源。

6.  使用 SUPER-SPACE 在安装的输入法之间进行切换. (SUPER 键通常是 Windows 键.)
:::

::: {.tip style="margin-left: 0.5in; margin-right: 0.5in;"}
  ----------------------------- ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
   ![\[提示\]](images/tip.png)  提示
                                如果你希望在日本物理键盘（shift-`2`{.literal} 按键刻有有一个`"`{.literal} 双引号标记 ）上访问只有字母表的键盘环境，在上面的过程中选择 \"Japanese\" 。你能够在物理的"US"键盘（shift-`2`{.literal} 按键刻有有一个 `@`{.literal} 标记）上使用\"Japanese mozc (或 anthy)\"输入日文。
  ----------------------------- ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
:::

::: itemizedlist
-   `im-config`{.literal}(8) 的用户界面菜单入口是 \"Input method\"。

-   此外，从用户的 shell 来执行 \"`im-config`{.literal}\" 。

-   如果命令从 root 账户或 非 root 账号执行，`im-config`{.literal}(8) 表现会有所不同。

-   `im-config`{.literal}(8) 让最佳的输入法作为系统默认而不需要用户干预。
:::

## []{#_the_display_output}8.3. 显示输出 {#显示输出 .title}

</div>

</div>
:::::

Linux 控制台只能显示有限的字符。（你需要使用特殊的终端程序，例如 `jfbterm`{.literal}(1)，从而在非 GUI 控制台中显示非欧洲语言。）

只要需要的字体安装并被启用，GUI（图形用户界面）环境（[第 7 章 *GUI（图形用户界面）系统*](ch07.zh-cn.html "第 7 章 GUI（图形用户界面）系统"){.xref}）能够显示任意 UTF-8 字符。（原始字体数据的编码会被处理，并对用户来说是透明的。）

## []{#_east_asian_ambiguous_character_width_characters}8.4. 东亚环境下宽度有歧义的字符 {#东亚环境下宽度有歧义的字符 .title}

</div>

</div>
:::::

在东亚语言环境下，方框绘制、希腊字符和西里尔字符可能会显示得比你预期的样子更宽，这样会导致终端输出排列不再整齐（参见 [Unicode 标准附录 #11，4.2 Ambiguous 字符](https://unicode.org/reports/tr11/#Ambiguous){.ulink}）。

您可以绕过这个问题：

::: itemizedlist
-   `gnome-terminal`{.literal}：首选项 → 配置文件 → [*配置名*]{.emphasis} → 兼容性 → 宽度有歧义的字符 → 窄

-   `ncurses`{.literal}：设置环境变量 `export NCURSES_NO_UTF8_ACS=0`{.literal}。
:::
