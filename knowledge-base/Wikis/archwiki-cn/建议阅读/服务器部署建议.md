##  网页/网络服务器

在互联网中存放各种网站的服务器，主要用于企业或个人网站在互联网上的发布和运用。 

###  Apache HTTP 服务器

[Apache HTTP 服务器](<https://www.apache.org/>)，简称 **Apache** ，是非常流行的[网络服务器](<../../zh-cn/%E5%BA%94%E7%94%A8%E7%A8%8B%E5%BA%8F%E5%88%97%E8%A1%A8/%E4%BA%92%E8%81%94%E7%BD%91.html#%E7%BD%91%E7%BB%9C%E6%9C%8D%E5%8A%A1%E5%99%A8> "网络服务器")软件。通常和脚本语言比如 [PHP](<../../zh-cn/PHP.html> "PHP")、数据库 [MySQL](<../../zh-cn/MySQL.html> "MySQL") 一起工作，合称为 LAMP 栈(Linux、Apache、MySQL、PHP)。 

[Apache HTTP 服务器](<../../zh-cn/Apache_HTTP_%E6%9C%8D%E5%8A%A1%E5%99%A8.html> "Apache HTTP 服务器")一文介绍了在 Arch Linux 上安装/设置 Apache 网页服务器，以及选择安装 [PHP](<../../zh-cn/PHP.html> "PHP") 和 [MySQL](<../../zh-cn/MySQL.html> "MySQL") 并集成到 Apache 服务器中的方法。 

### GNU Artanis

[GNU Artanis](<https://artanis.dev/>) 是 [Scheme](<../../zh-cn/Scheme.html> "Scheme") 编程语言的第一个产品级现代网络框架。它的设计和维护都以稳健、快速和易于使用为目标，适用于专业的网络开发。 

它支持的数据库（通过 guile-dbi）有 [MySQL](<../../zh-cn/MySQL.html> "MySQL")、[SQLite](<../../zh-cn/SQLite.html> "SQLite")、[PostgreSQL](<../../zh-cn/PostgreSQL.html> "PostgreSQL")。 

GNU Artanis 主要与 [Guile](<https://www.gnu.org/software/guile/>) 一起使用。 

参见 [Artanis](<../../zh-cn/Artanis.html> "Artanis")。 

### Caddy

[Caddy](<https://caddyserver.com/>) 是具有 HTTP/2 功能的[网页服务器](<../../zh-cn/%E5%BA%94%E7%94%A8%E7%A8%8B%E5%BA%8F%E5%88%97%E8%A1%A8/%E4%BA%92%E8%81%94%E7%BD%91.html#%E7%BD%91%E7%BB%9C%E6%9C%8D%E5%8A%A1%E5%99%A8> "网络服务器")，具有自动 HTTPS 配置功能。 

有关 Caddy 在 Arch Linux 上的安装、配置、用法等，参见 [Caddy](<../../zh-cn/Caddy.html> "Caddy")。 

### Lighttpd

[Lighttpd](<https://www.lighttpd.net/>) 是一个安全、快速、标准且非常灵活的[网页服务器](<../../zh-cn/%E5%BA%94%E7%94%A8%E7%A8%8B%E5%BA%8F%E5%88%97%E8%A1%A8/%E4%BA%92%E8%81%94%E7%BD%91.html#%E7%BD%91%E7%BB%9C%E6%9C%8D%E5%8A%A1%E5%99%A8> "网络服务器")，并对高性能环境做了最佳化。 

参见 [Lighttpd](<../../zh-cn/Lighttpd.html> "Lighttpd") 了解有关信息。 

### Nginx

Nginx（读作 "Engine X"）是一个免费、开源、高性能的 HTTP [网络服务器](<../../zh-cn/%E5%BA%94%E7%94%A8%E7%A8%8B%E5%BA%8F%E5%88%97%E8%A1%A8/%E4%BA%92%E8%81%94%E7%BD%91.html#%E7%BD%91%E7%BB%9C%E6%9C%8D%E5%8A%A1%E5%99%A8> "网络服务器")和反向代理，也是一个 IMAP/POP3 代理服务器，由 Igor Sysoev 于 2005 年编写。 

[Nginx](<../../zh-cn/Nginx.html> "Nginx") 一文描述了如何设置 nginx 并且如何通过 [FastCGI](<../../zh-cn/Nginx.html#FastCGI> "Nginx") 集成 [PHP](<../../zh-cn/PHP.html> "PHP")。

### BIND

伯克利互联网名称服务 (Berkeley Internet Name Daemon，简称 [BIND](<https://www.isc.org/downloads/bind/>)) 是 DNS 协议的一个参考实现。 

参见 [BIND](<../../zh-cn/BIND.html> "BIND")。 

### Cloudflared

[Cloudflared](<https://developers.cloudflare.com/1.1.1.1/dns-over-https/cloudflared-proxy/>) 可被用于运行本地 [DNS over HTTPS](<../../zh-cn/DNS_over_HTTPS.html> "DNS over HTTPS")（DoH）服务器，也就是存根解析器。 

参见 [Cloudflared](<../../zh-cn/Cloudflared.html> "Cloudflared")。 

### DNS-over-HTTPS

[DNS-over-HTTPS](<https://github.com/m13253/dns-over-https>)是 [DNS over HTTPS](<../../zh-cn/DNS_over_HTTPS.html> "DNS over HTTPS") 的一个实现。它可以作为一个本地解析器。 

参见 [DNS-over-HTTPS](<../../zh-cn/DNS-over-HTTPS.html> "DNS-over-HTTPS")。 

### Dnscrypt-proxy

[dnscrypt-proxy](<https://github.com/jedisct1/dnscrypt-proxy>) 可以加密和认证用户和 DNS 解析服务器之间的数据传输，支持 [DNS over HTTPS](<https://en.wikipedia.org/wiki/DNS_over_HTTPS> "wikipedia:DNS over HTTPS") 和 [DNSCrypt](<https://dnscrypt.info/>)，可以避免中间人攻击和窃听。 _dnscrypt-proxy_ 兼容 [DNSSEC](</wzh/index.php?title=DNSSEC&action=edit&redlink=1> "DNSSEC（页面不存在）")。 

参见 [Dnscrypt-proxy](<../../zh-cn/Dnscrypt-proxy.html> "Dnscrypt-proxy")。 

### dnsmasq

[dnsmasq](<https://www.thekelleys.org.uk/dnsmasq/doc.html>) 提供 [DNS 服务器](<https://en.wikipedia.org/wiki/Name_server> "wikipedia:Name server")、支持 [DHCPv6](<https://en.wikipedia.org/wiki/DHCPv6> "wikipedia:DHCPv6") 和 [PXE](<https://en.wikipedia.org/wiki/Preboot_Execution_Environment> "wikipedia:Preboot Execution Environment") 的 [DHCP 服务器](<https://en.wikipedia.org/wiki/Dynamic_Host_Configuration_Protocol> "wikipedia:Dynamic Host Configuration Protocol")、[TFTP 服务器](<https://en.wikipedia.org/wiki/Trivial_File_Transfer_Protocol> "wikipedia:Trivial File Transfer Protocol")。它设计为轻量且占用空间小，适用于资源受限的路由器和防火墙。还可以将 dnsmasq 配置为 DNS 缓存查询，以提高对以前访问过站点的 DNS 查找速度。 

参见 [dnsmasq](<../../zh-cn/Dnsmasq.html> "Dnsmasq")。 

### systemd-resolved

_systemd-resolved_ 是一个通过 [D-Bus](<../../zh-cn/D-Bus.html> "D-Bus") 接口、[NSS](</wzh/index.php?title=Name_Service_Switch&action=edit&redlink=1> "Name Service Switch（页面不存在）") ([nss-resolve(8)](<https://man.archlinux.org/man/nss-resolve.8>))解析服务和一个监听 `127.0.0.53` 的本地DNS解析器为本地应用提供网络名称解析的 [systemd](<../../zh-cn/Systemd.html> "Systemd") 服务。 

参见 [systemd-resolved](<../../zh-cn/Systemd-resolved.html> "Systemd-resolved")。 

### Unbound

[Unbound](<https://unbound.net/>) 是一个具有验证，递归和缓存等功能的 DNS 解析器。 

参见 [Unbound](<../../zh-cn/Unbound.html> "Unbound")。

Percona 也有 Oracle 的 [InnoDB](<https://en.wikipedia.org/wiki/InnoDB> "wikipedia:InnoDB") 存储引擎的分支，称为 [XtraDB](<https://en.wikipedia.org/wiki/XtraDB> "wikipedia:XtraDB")。[MariaDB](<../../zh-cn/MariaDB.html> "MariaDB") 和 Percona Server 都使用此分支。

[PostgreSQL](<https://www.postgresql.org/>) 是一个开源的，社区驱动的，符合标准的对象-关系型数据库系统。

### SQLite

[SQLite](<https://www.sqlite.org/>) 是一个实现自包含、无服务器、零配置、事务性 SQL 数据库引擎的软件库。SQLite 是世界上部署最广泛的 SQL 数据库引擎。SQLite 的源代码在公共领域。 

参见 [SQLite](<../../zh-cn/SQLite.html> "SQLite")。 

### Adminer

[Adminer](<https://www.adminer.org/>) 是一个用 PHP 编写的基于网络的数据库管理工具。它可以管理 [MySQL](<../../zh-cn/MySQL.html> "MySQL")、[PostgreSQL](<../../zh-cn/PostgreSQL.html> "PostgreSQL")、[Sqlite3](<../../zh-cn/SQLite.html> "Sqlite3")、MS SQL、Oracle 数据库和 [Elasticsearch](<../../zh-cn/Elasticsearch.html> "Elasticsearch")。 

参见 [Adminer](<../../zh-cn/Adminer.html> "Adminer")。 

### Elasticsearch

[Elasticsearch](<https://www.elastic.co/products/elasticsearch>) 是基于 [Lucene](<https://lucene.apache.org/>) 的搜索引擎。它提供了一个分布式、支持多租户的全文搜索引擎，具有 HTTP 网络接口和无模式 JSON 文档。Elasticsearch 采用 Java 开发，根据 Apache License 条款开源发布。 

参见 [Elasticsearch](<../../zh-cn/Elasticsearch.html> "Elasticsearch")。
