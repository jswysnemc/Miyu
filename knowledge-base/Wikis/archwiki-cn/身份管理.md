[身份管理](<https://en.wikipedia.org/wiki/Identity_management> "wikipedia:Identity management")（Identity Management，IDM），又称作身份与访问管理（Identity and Access Management，IAM），负责控制用户用户如何获得数字身份及其角色，并有时负责为身份授予权限。 

##  软件清单

有许多软件可以帮助进行身份管理，下列是其中一部分。 

  * [Google 身份验证器](<../zh-cn/Google_%E8%BA%AB%E4%BB%BD%E9%AA%8C%E8%AF%81%E5%99%A8.html> "Google 身份验证器")，使用一次性密码(OTP)进行两步验证
  * [Initiative for Open Authentication](</wzh/index.php?title=Initiative_for_Open_Authentication&action=edit&redlink=1> "Initiative for Open Authentication（页面不存在）")，对网页 API 调用获取访问令牌的方法进行了标准化。包含了密码方法，但未来会转用 WebAuthn
  * kanidm，相关信息可参考[此处](<https://fy.blackhats.net.au/blog/html/2019/09/18/announcing_kanidm_a_new_idm_project.html>)，是 LDAP 和 Kerberos 的后继者，实现了服务器端的 WebAuthn
  * [Kerberos](<../zh-cn/Kerberos.html> "Kerberos"), with [Active Directory integration](</wzh/index.php?title=Active_Directory_integration&action=edit&redlink=1> "Active Directory integration（页面不存在）"), [FreeIPA](<../zh-cn/FreeIPA.html> "FreeIPA")
  * [LDAP 认证](<../zh-cn/LDAP_%E8%AE%A4%E8%AF%81.html> "LDAP 认证")，可将密码存放在中心化服务器上
  * [PAM](<../zh-cn/PAM.html> "PAM") Linux 可插拔认证模块（Pluggable Authentication Module），可将 Kerberos，LDAP，kanidm 等接入到 Linux
  * [通用第二因素](<../zh-cn/%E9%80%9A%E7%94%A8%E7%AC%AC%E4%BA%8C%E5%9B%A0%E7%B4%A0.html> "通用第二因素")（U2F），被 WebAuthn 替代
  * [用户和用户组](<../zh-cn/%E7%94%A8%E6%88%B7%E5%92%8C%E7%94%A8%E6%88%B7%E7%BB%84.html> "用户和用户组")，可存放在系统上或中心化的 LDAP 和 kanidm 平台上
  * [WebAuthn](</wzh/index.php?title=WebAuthn&action=edit&redlink=1> "WebAuthn（页面不存在）") 浏览器网页认证 API（Web Authentication API），用于替代密码认证，可向后兼容 U2F
