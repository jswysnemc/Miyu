##  安装前的准备

###  获取安装映像

打开[下载](<https://archlinux.org/download/>)页面，并根据需要的引导方式，获取 ISO 文件或 netboot 映像以及相应的 PGP 签名。 

###  验证签名

建议使用前先验证所下载文件的签名，特别是从 **HTTP 镜像源/非官方镜像仓库** 下载的文件，因为 HTTP 连接一般来说容易遭到拦截而[提供恶意文件](<http://www2.cs.arizona.edu/stork/packagemanagersecurity/attacks-on-package-managers.html>)，而非官方镜像仓库可能会对镜像的文件进行恶意修改。验证签名也能及时发现文件损坏。 

####  验证哈希校验和

在 Arch Linux 网站的[下载页面的 HTTP Direct Downloads 一节](<https://archlinux.org/download/#http-downloads>)提供了安装映像的校验和，包括 SHA256 和 BLAKE2b 两种算法，您可以使用任何支持其中一个算法的检验工具来检查文件校验和是否正确。例如，[7-Zip](<https://www.7-zip.org/>) 具有此功能。 

如果您正在使用的系统上安装了 `sha256sum` 命令，则可以下载网站提供的 _sha256sums.txt_ 文件，并将它放在前面下载的安装映像的同一个目录下，然后运行以下命令来验证： 
    
    $ sha256sum -c sha256sums.txt --ignore-missing
    
####  验证 PGP 签名

从[下载页面](<https://archlinux.org/download/#checksums>) 下载 **PGP 签名** 到 ISO 文件所在的路径，并根据该页面提供的指引来校验 ISO 文件。 

另外，在一台已经安装 Arch Linux 的计算机上可以通过以下方式验证： 
    
    $ pacman-key -v archlinux-_版本_ -x86_64.iso.sig
    
**注意：**

  * 如果安装映像是从镜像站点下载，而不是从 [ArchLinux.org](<https://archlinux.org/download/>) 下载的话，其签名有被伪造的风险。在这种情况下，请您确保用来解码签名的公钥是被另一个可信的密钥签署的。`gpg` 命令将会输出公钥的指纹。
  * 另一种验证签名的方法是确保公钥的指纹等于其中一位签署了 ISO 文件 [Arch Linux 开发者](<https://archlinux.org/people/developers/>)的指纹。请您参阅[维基百科](<https://zh.wikipedia.org/wiki/%E5%85%AC%E5%BC%80%E5%AF%86%E9%92%A5%E5%8A%A0%E5%AF%86> "zhwp:公开密钥加密")获取更多关于公钥加密的信息。

**提示：** 在 Windows 上，您可尝试[Gpg4win](<https://gpg4win.org/index.html>)，注意：您所[下载](<https://files.gpg4win.org/>)、安装和使用的 Gpg4win 程序需要您自行甄别其真伪。

###  准备安装介质

Arch Linux 的ISO文件可以被制作成多种类型安装介质，如 [U 盘](<../zh-cn/U_%E7%9B%98%E5%AE%89%E8%A3%85%E4%BB%8B%E8%B4%A8.html> "U 盘安装介质")、[光盘](<../zh-cn/%E5%85%89%E7%9B%98%E9%A9%B1%E5%8A%A8%E5%99%A8.html#%E5%88%BB%E5%BD%95> "光盘驱动器")和带有 [PXE](<../zh-cn/PXE.html> "PXE") 的网络安装映像。请按照合适的文章与教程，使用ISO文件为自己准备安装介质。 

下载archiso文件时请考虑使用镜像站，比如[中国境内的镜像站](<https://archlinux.org/download/#:~:text=China>)。可以考虑使用 [Ventoy](<../zh-cn/Ventoy.html> "Ventoy") 制作安装介质。 

对于 Netboot 映像，请按照[网络引导#从U盘启动](<../zh-cn/%E7%BD%91%E7%BB%9C%E5%BC%95%E5%AF%BC.html#%E4%BB%8EU%E7%9B%98%E5%90%AF%E5%8A%A8> "网络引导")准备用于 UEFI 引导的 USB 闪存驱动器。 

###  启动到 live 环境

**注意：** Arch Linux 安装镜像不支持 UEFI 安全启动（Secure Boot）功能。如果要引导安装介质，需要[禁用安全启动](<../zh-cn/UEFI/%E5%AE%89%E5%85%A8%E5%90%AF%E5%8A%A8.html#%E7%A6%81%E7%94%A8%E5%AE%89%E5%85%A8%E5%90%AF%E5%8A%A8> "UEFI/安全启动")。如果需要，可在完成安装后重新[配置](<../zh-cn/UEFI/%E5%AE%89%E5%85%A8%E5%90%AF%E5%8A%A8.html#%E5%AE%9E%E6%96%BD%E5%AE%89%E5%85%A8%E5%90%AF%E5%8A%A8> "UEFI/安全启动")。

  1. 选择从带有 Arch 安装文件的介质启动，通常是需要在[计算机启动加电自检](<https://zh.wikipedia.org/wiki/%E5%8A%A0%E7%94%B5%E8%87%AA%E6%A3%80> "zhwp:加电自检")时快速按下某个热键（比如部分主板会是 `F12` 键或 `F8` 键）。启动时的画面也可能会有提示，详情请参考自己的计算机的说明书或主板说明书。
  2. 当引导加载程序菜单出现时， 
     * 如果你使用了 ISO 映像，选择 _Arch Linux install medium_ 并按 `Enter` 进入安装环境。
     * 如果您使用了 Netboot 映像，请从 _Mirror_ 菜单中选择一个地理位置接近的镜像，然后选择 _Boot Arch Linux_ 并按 `Enter`。
  3. 您将会以 root 身份登录进入一个[虚拟控制台](<https://en.wikipedia.org/wiki/Virtual_console> "wikipedia:Virtual console")，默认的 Shell 是 [Zsh](<../zh-cn/Zsh.html> "Zsh")。

**提示：**

  * 安装映像在 UEFI 模式下使用 [systemd-boot](<../zh-cn/Systemd-boot.html> "Systemd-boot") 引导，在 BIOS 模式下使用 [Syslinux](<../zh-cn/Syslinux.html> "Syslinux") 引导。分别使用 `e` 或 `Tab` 来输入[引导参数](<../zh-cn/%E5%86%85%E6%A0%B8%E5%8F%82%E6%95%B0.html#%E5%BC%95%E5%AF%BC%E5%8A%A0%E8%BD%BD%E7%A8%8B%E5%BA%8F%E9%85%8D%E7%BD%AE> "内核参数")。Netboot 映像使用 iPXE，可以在 _启动选项_ 菜单中指定启动参数。请参阅 [/usr/share/doc/mkinitcpio-archiso/README.bootparams](<https://gitlab.archlinux.org/archlinux/mkinitcpio/mkinitcpio-archiso/blob/master/docs/README.bootparams>) 获取[引导参数](<../zh-cn/%E5%86%85%E6%A0%B8%E5%8F%82%E6%95%B0.html#%E5%BC%95%E5%AF%BC%E5%8A%A0%E8%BD%BD%E7%A8%8B%E5%BA%8F%E9%85%8D%E7%BD%AE> "内核参数")列表。
  * 手动定义启动参数的一个常见例子是改变系统显示在超高分辨率（HiDPI）屏幕的字体的大小。为使系统在 HiDPI 屏幕上显示的字体有更好的可读性——当 Live 系统启动时屏幕还没有被识别为 HiDPI 的时候——使用 `fbcon=font:TER16x32` 会有帮助。参见 [HiDPI#Linux 控制台](<../zh-cn/HiDPI.html#Linux_%E6%8E%A7%E5%88%B6%E5%8F%B0> "HiDPI")的详细解释。

如果要使用其他控制台（例如在安装时使用 [Lynx](<https://lynx.invisible-island.net/lynx_help/Lynx_users_guide.html>) 查看本指南），可以使用 `Alt+_箭头_` [键盘快捷键](<../zh-cn/Linux_%E6%8E%A7%E5%88%B6%E5%8F%B0.html#%E9%94%AE%E7%9B%98%E5%BF%AB%E6%8D%B7%E9%94%AE> "Linux 控制台")切换不同的控制台。要[编辑](<../zh-cn/Help:%E9%98%85%E8%AF%BB.html#%E6%B7%BB%E5%8A%A0%E3%80%81%E5%88%9B%E5%BB%BA%E3%80%81%E7%BC%96%E8%BE%91%E6%96%87%E4%BB%B6> "Help:阅读")配置文件，可以使用 [mcedit(1)](<https://man.archlinux.org/man/mcedit.1>)、[nano](<../zh-cn/Nano.html#%E4%BD%BF%E7%94%A8> "Nano") 和 [vim](<../zh-cn/Vim.html#%E7%94%A8%E6%B3%95> "Vim") 等文本编辑软件。请参阅 [packages.x86_64](<https://geo.mirror.pkgbuild.com/iso/latest/arch/pkglist.x86_64.txt>) 获取安装介质中包含的软件包列表。 

####  关于 archinstall 的提示

如果您觉得用命令行安装过于繁琐，可以试试官方的安装脚本 `archinstall`。详情可参见 [archinstall](<../zh-cn/Archinstall.html> "Archinstall")。 

但**不建议** 初学者用 archinstall 走捷径，因为手动安装 Arch Linux 本身就是 Arch Linux 教学实操中**重要的一环** ，少了这个环节，会让初学者面对一些基础问题时束手无策，为自己和社区带来不必要的麻烦。 
    
    # archinstall
    
**提示：** 使用 archinstall 安装基础系统以后，可能需要进行一些配置，才能使用户体验得以优化提升。参见[#安装后的工作](<#%E5%AE%89%E8%A3%85%E5%90%8E%E7%9A%84%E5%B7%A5%E4%BD%9C>)与[建议阅读](<../zh-cn/%E5%BB%BA%E8%AE%AE%E9%98%85%E8%AF%BB.html> "建议阅读")等页面。

###  配置控制台键盘布局和字体

系统启动后，[控制台键盘布局](<../zh-cn/Linux_%E6%8E%A7%E5%88%B6%E5%8F%B0/%E9%94%AE%E7%9B%98%E9%85%8D%E7%BD%AE.html> "Linux 控制台/键盘配置")默认为 `us`（美式键盘）。如果您需要列出所有可用的键盘布局，可以使用以下命令： 
    
    # localectl list-keymaps
    
如果您想要更改键盘布局，可以使用 [loadkeys(1)](<https://man.archlinux.org/man/loadkeys.1>)。比如，要添加[德语](<https://en.wikipedia.org/wiki/File:KB_Germany.svg> "wikipedia:File:KB Germany.svg")键盘布局： 
    
    # loadkeys de-latin1
    
**注意：** 中文环境用户请勿照抄以上设置，默认键盘布局（us/C）已经可以正常地使用。

[控制台字体](<../zh-cn/Linux_console.html#%E5%AD%97%E4%BD%93> "Linux console")位于 `/usr/share/kbd/consolefonts/` 目录中，设置方式请参考 [setfont(8)](<https://man.archlinux.org/man/setfont.8>)。例如，要使用适合 [HiDPI 屏幕](<../zh-cn/HiDPI.html#Linux_console_\(tty\)> "HiDPI")的最大字体之一，请运行以下命令： 
    
    # setfont ter-132b
    
###  验证引导模式

要验证系统目前的引导模式，请检查 [UEFI 固件位数](<../zh-cn/UEFI.html#UEFI_%E5%9B%BA%E4%BB%B6%E4%BD%8D%E6%95%B0> "UEFI")： 
    
    # cat /sys/firmware/efi/fw_platform_size
    
  * 如果命令结果为 `64`，则系统是以 UEFI 模式引导且使用 64 位 x64 UEFI。
  * 如果命令结果为 `32`，则系统是以 UEFI 模式引导且使用 32 位 IA32 UEFI，虽然其受支持，但引导加载程序只能使用 [systemd-boot](<../zh-cn/Systemd-boot.html> "Systemd-boot") 和 [GRUB](<../zh-cn/GRUB.html> "GRUB")。
  * 如果命令结果为`No such file or directory`，则系统可能是以 [BIOS](<https://zh.wikipedia.org/wiki/BIOS> "zhwp:BIOS") 模式（或 [CSM](<https://en.wikipedia.org/wiki/Compatibility_Support_Module> "wikipedia:Compatibility Support Module") 模式，这两种模式通常出现在**老旧** 的电脑或**未经配置的虚拟机** 上）引导。

如果系统没有以您想要的模式（UEFI 或 BIOS）引导启动，请您参考自己的计算机或主板说明书。 

###  连接到互联网

要在 Live 环境中配置网络连接，请遵循以下步骤： 

  1. 确保系统已经列出并启用了[网络接口](<../zh-cn/%E7%BD%91%E7%BB%9C%E9%85%8D%E7%BD%AE.html#%E7%BD%91%E7%BB%9C%E6%8E%A5%E5%8F%A3> "网络配置")，用 [ip-link(8)](<https://man.archlinux.org/man/ip-link.8>) 检查：
         
         # ip link

  2. 对于无线局域网（Wi-Fi）和无线广域网（WWAN），请确保网卡未被 [rfkill](<../zh-cn/%E7%BD%91%E7%BB%9C%E9%85%8D%E7%BD%AE/%E6%97%A0%E7%BA%BF%E7%BD%91%E7%BB%9C%E9%85%8D%E7%BD%AE.html#Rfkill_%E8%AF%B4%E6%98%8E> "Rfkill") 禁用。
  3. 连接到网络： 
     * 有线以太网——连接网线。
     * WiFi——使用 [iwctl](<../zh-cn/Iwd.html#iwctl> "Iwd") 认证无线网络。
     * 移动宽带调制解调器（移动网卡） - 使用 [mmcli](</wzh/index.php?title=Mmcli&action=edit&redlink=1> "Mmcli（页面不存在）")（英语：[mmcli](<https://wiki.archlinux.org/title/mmcli> "en:mmcli")） 连接到移动网络。
  4. 配置网络连接： 
     * [DHCP](<../zh-cn/%E7%BD%91%E7%BB%9C%E9%85%8D%E7%BD%AE.html#DHCP> "网络配置")：对于有线以太网、无线局域网（WLAN）和无线广域网（WWAN）网络接口来说，动态 IP 地址和 DNS 服务器分配（由 [systemd-networkd](<../zh-cn/Systemd-networkd.html> "Systemd-networkd") 和 [systemd-resolved](<../zh-cn/Systemd-resolved.html> "Systemd-resolved") 提供功能）能够开箱即用。
     * 静态 IP 地址：按照[网络配置#静态 IP 地址](<../zh-cn/%E7%BD%91%E7%BB%9C%E9%85%8D%E7%BD%AE.html#%E9%9D%99%E6%80%81_IP_%E5%9C%B0%E5%9D%80> "网络配置")进行操作。
  5. 用 [ping](<../zh-cn/%E7%BD%91%E7%BB%9C%E9%85%8D%E7%BD%AE.html#Ping> "网络配置") 检查网络连接：
         
         # ping ping.archlinux.org

**注意：** 默认情况下，安装映像在启动时已经预先配置好并启用了 [systemd-networkd](<../zh-cn/Systemd-networkd.html> "Systemd-networkd")、[systemd-resolved](<../zh-cn/Systemd-resolved.html> "Systemd-resolved")、[iwd](<../zh-cn/Iwd.html> "Iwd") 和 [ModemManager](</wzh/index.php?title=ModemManager&action=edit&redlink=1> "ModemManager（页面不存在）")（英语：[ModemManager](<https://wiki.archlinux.org/title/ModemManager> "en:ModemManager")）。但在已经安装完成了的系统之中并非如此。

###  更新系统时间

为确保软件包签名校验成功以及防止 TLS 证书错误，Live 系统需要准确的时间，为此 [systemd-timesyncd](<../zh-cn/Systemd-timesyncd.html> "Systemd-timesyncd") 默认启用，也就是说当系统已经创建互联网连接后，系统时间将自动同步。 

使用 [timedatectl(1)](<https://man.archlinux.org/man/timedatectl.1>) 确保系统时间是同步的： 
    
    # timedatectl
    
###  创建硬盘分区

系统如果识别到计算机的内置硬盘、U盘或者移动硬盘等类型磁盘，就会将其分配为一个[块设备](<../zh-cn/%E8%AE%BE%E5%A4%87%E6%96%87%E4%BB%B6.html#%E5%9D%97%E8%AE%BE%E5%A4%87> "设备文件")，如 `/dev/sda`、`/dev/nvme0n1` 或 `/dev/mmcblk0`。可以使用 [lsblk](<../zh-cn/%E8%AE%BE%E5%A4%87%E6%96%87%E4%BB%B6.html#lsblk> "Lsblk") 或者 [fdisk](<../zh-cn/Fdisk.html> "Fdisk") 等工具查看： 
    
    # fdisk -l	#此处为小写字母l
    
结果中以 `rom`、`loop` 或者 `airootfs` 结尾的设备可以被忽略。结果中以 `rpmb`、`boot0` 或者 `boot1` 结尾的 `mmcblk*` 设备也可以被忽略。 

**注意：** 如果您想要的磁盘没有显示出来，请确保[您的磁盘控制器未处于 RAID 模式](<../zh-cn/%E5%88%86%E5%8C%BA.html#%E5%90%AF%E7%94%A8%E5%9B%BA%E4%BB%B6_RAID_%E6%97%B6%E6%97%A0%E6%B3%95%E6%89%BE%E5%88%B0%E7%A1%AC%E7%9B%98> "分区")。

**提示：** 在分区之前，请您检查 NVMe 驱动器和[先进格式化](<../zh-cn/%E5%85%88%E8%BF%9B%E6%A0%BC%E5%BC%8F%E5%8C%96.html> "先进格式化")硬盘是否使用了[最佳逻辑扇区大小](<../zh-cn/%E5%85%88%E8%BF%9B%E6%A0%BC%E5%BC%8F%E5%8C%96.html> "先进格式化")。需要注意的是，更改逻辑扇区大小后，可能会导致在 Windows 系统中出现兼容性问题。

对于一个选定的设备，以下分区是**必需** 的： 

  * 一个根分区（挂载在[根目录](<../zh-cn/%E5%88%86%E5%8C%BA.html#%E6%A0%B9%E5%88%86%E5%8C%BA> "分区")）`/`；
  * 要在 [UEFI](<../zh-cn/Unified_Extensible_Firmware_Interface.html> "Unified Extensible Firmware Interface") 模式中启动，还需要一个 [EFI 系统分区](<../zh-cn/EFI_%E7%B3%BB%E7%BB%9F%E5%88%86%E5%8C%BA.html> "EFI 系统分区")。

请使用分区工具（[fdisk](<../zh-cn/Fdisk.html> "Fdisk") 、[parted](<../zh-cn/Parted.html> "Parted")、[cfdisk](<../zh-cn/Cfdisk.html> "Cfdisk") 等等）修改分区表。例如： 
    
    # cfdisk /dev/_要被分区的磁盘_
    
**注意：**

  * 花时间规划一个长期的分区方案，以避免将来需要有风险和复杂的转换或重新分区。
  * 如果您想为 [LVM](<../zh-cn/LVM.html> "LVM")、[系统加密](<../zh-cn/Dm-crypt.html> "Dm-crypt") 或 [RAID](<../zh-cn/RAID.html> "RAID") 创建任何堆叠块设备，请立即执行。
  * 如果要从中启动的磁盘[已经有一个 EFI 分区](<../zh-cn/EFI_%E7%B3%BB%E7%BB%9F%E5%88%86%E5%8C%BA.html#%E6%A3%80%E6%9F%A5%E7%8E%B0%E6%9C%89%E7%9A%84%E5%88%86%E5%8C%BA> "EFI 系统分区")，请不要创建另一个 EFI 分区，而是使用现有分区。
  * [Swap](<../zh-cn/Swap.html> "Swap") 交换空间可以在 [Swap#交换文件](<../zh-cn/Swap.html#%E4%BA%A4%E6%8D%A2%E6%96%87%E4%BB%B6> "Swap") 上为支持它的文件系统设置。或者，你也可以在安装完毕后配置[基于 zram 的 swap](<../zh-cn/Zram.html#%E4%BD%9C%E4%B8%BA_swap_%E7%9A%84%E4%BD%BF%E7%94%A8> "Zram") 以避免使用基于磁盘的 swap。

####  分区方案示例

对于 UEFI 与 [GPT](<../zh-cn/%E5%88%86%E5%8C%BA.html#GUID_%E5%88%86%E5%8C%BA%E8%A1%A8> "Partitioning") 分区表的磁盘分区方案  已安装系统上的挂载点  | 分区  |  [分区类型](<https://en.wikipedia.org/wiki/GUID_Partition_Table#Partition_type_GUIDs> "wikipedia:GUID Partition Table") | 注释  | 建议大小   
---|---|---|---|---  
`/boot`1 |  `/dev/_efi_system_partition_` | EFI System  |  [EFI 系统分区](<../zh-cn/EFI_%E7%B3%BB%E7%BB%9F%E5%88%86%E5%8C%BA.html> "EFI 系统分区") | 1 GiB   
`[SWAP]` |  `/dev/_swap_partition_` | Linux swap  |  [交换空间](<../zh-cn/Swap.html> "Swap") | 至少 4 GiB   
`/` |  `/dev/_root_partition_` | Linux root (x86-64)  | 根目录  | 设备剩余空间，至少 23-32 GiB   
  
  1. 如果使用的引导加载程序能够从根磁盘卷中加载内核和 initramfs 映像，则可以使用[其他挂载点](<../zh-cn/EFI_%E7%B3%BB%E7%BB%9F%E5%88%86%E5%8C%BA.html#%E5%85%B8%E5%9E%8B%E6%8C%82%E8%BD%BD%E7%82%B9> "EFI 系统分区")（例如 `/efi`）。请您参阅[引导加载程序](<../zh-cn/Arch_%E7%9A%84%E5%90%AF%E5%8A%A8%E6%B5%81%E7%A8%8B.html#%E5%BC%95%E5%AF%BC%E5%8A%A0%E8%BD%BD%E7%A8%8B%E5%BA%8F> "Arch 的启动流程")中的警告部分。

对于传统 BIOS 与 [MBR](<../zh-cn/%E5%88%86%E5%8C%BA.html#%E4%B8%BB%E5%BC%95%E5%AF%BC%E8%AE%B0%E5%BD%95> "分区") 分区表的磁盘分区方案  已安装系统上的挂载点  | 分区  |  [分区类型](<https://en.wikipedia.org/wiki/GUID_Partition_Table#Partition_type_GUIDs> "wikipedia:GUID Partition Table") | 建议大小   
---|---|---|---  
`[SWAP]` |  `/dev/_swap_partition_` | Linux swap (交换空间)  | 至少 4GiB   
`/` |  `/dev/_root_partition_` | Linux  | 设备剩余空间，至少23-32GiB   
  
另请参阅[布局示例](<../zh-cn/%E5%88%86%E5%8C%BA.html#%E5%B8%83%E5%B1%80%E7%A4%BA%E4%BE%8B> "分区")。 

###  格式化分区

创建分区后，必须使用合适的[文件系统](<../zh-cn/File_systems.html> "File systems")对每个新创建的分区进行格式化。详情请参阅[文件系统#创建文件系统](<../zh-cn/File_systems.html#%E5%88%9B%E5%BB%BA%E6%96%87%E4%BB%B6%E7%B3%BB%E7%BB%9F> "File systems")。 

**注意：** 对于堆叠式块设备（stacked block devices）请使用恰当的块设备路径替换下文中的 `/dev/_*_partition_` 处。

例如，要在根分区 `/dev/_root_partition_` 上创建一个 [Ext4](<../zh-cn/Ext4.html> "Ext4") 文件系统，请运行： 
    
    # mkfs.ext4 /dev/_root_partition（根分区）_
    
如果你要创建一个 [EFI 系统分区](<../zh-cn/EFI_%E7%B3%BB%E7%BB%9F%E5%88%86%E5%8C%BA.html> "EFI 系统分区")，使用 [mkfs.fat(8)](<https://man.archlinux.org/man/mkfs.fat.8>) 将其[格式化](<../zh-cn/EFI_system_partition.html#Format_the_partition> "EFI system partition")为 Fat32。 

**警告：** 只有在分区步骤中创建 EFI 系统分区时才需要格式化。如果这个磁盘上已经有一个 EFI 系统分区了，将它重新格式化会破坏其他已安装操作系统的引导加载程序。
    
    # mkfs.fat -F 32 /dev/_efi_system_partition（EFI 系统分区）_
    
如果创建了[交换分区](<../zh-cn/Swap.html> "Swap")，请使用 [mkswap(8)](<https://man.archlinux.org/man/mkswap.8>) 将其初始化： 
    
    # mkswap /dev/_swap_partition（交换空间分区）_
    
###  挂载分区

将根磁盘卷[挂载](<../zh-cn/%E6%96%87%E4%BB%B6%E7%B3%BB%E7%BB%9F.html#%E6%8C%82%E8%BD%BD%E6%96%87%E4%BB%B6%E7%B3%BB%E7%BB%9F> "文件系统")到 `/mnt`，例如： 
    
    # mount /dev/_root_partition（根分区）_ /mnt
    
然后使用 [mkdir(1)](<https://man.archlinux.org/man/mkdir.1>) 在 `/mnt` 下创建任何剩余的挂载点（例如，为`/boot` 而创建`/mnt/boot`），并按相应的层级顺序挂载相应的磁盘卷。 

**提示：** 使用 `--mkdir` 选项运行 [mount(8)](<https://man.archlinux.org/man/mount.8>) 来创建指定的挂载点。或者，先使用 [mkdir(1)](<https://man.archlinux.org/man/mkdir.1>) 创建挂载点再挂载。

**注意：** 挂载分区一定要遵循顺序，先挂载根（root）分区（到 `/mnt`），再挂载引导（boot）分区（到 `/mnt/boot` 或 `/mnt/efi`，如果单独分出来了的话），最后再挂载其他分区。否则您可能遇到安装完成后无法启动系统的问题。参见 [en:Talk:Installation guide#Clarify root mount](<https://wiki.archlinux.org/title/Talk:Installation_guide#Clarify_root_mount> "en:Talk:Installation guide")。

对于 UEFI 系统，挂载 EFI 系统分区，例如： 
    
    # mount --mkdir /dev/_efi_system_partition_ /mnt/boot
    
如果创建了[交换空间](<../zh-cn/Swap.html> "Swap")卷，请使用 [swapon(8)](<https://man.archlinux.org/man/swapon.8>) 启用它： 
    
    # swapon /dev/_swap_partition（交换空间分区）_
    
稍后 [genfstab](<../zh-cn/Genfstab.html> "Genfstab") 将自动检测挂载的文件系统和交换空间。 

##  开始安装系统

###  选择镜像站

需安装的软件包会从文件 `/etc/pacman.d/mirrorlist` 中所列的[镜像](<../zh-cn/%E9%95%9C%E5%83%8F%E6%BA%90.html> "镜像")站下载。下载软件包时，列表中越靠前的镜像站会拥有越高的优先级。 

在从 LiveCD 启动的系统中，所有未注释的 HTTPS 镜像站都会启用，位于顶端的全球镜像站对大多用户来说应当足够快，但仍**请您检查`/etc/pacman.d/mirrorlist` 文件**，看看列出的镜像站的顺序是否合适。如果不合适，可以手动[编辑](<../zh-cn/Help:%E9%98%85%E8%AF%BB.html#%E6%B7%BB%E5%8A%A0%E3%80%81%E5%88%9B%E5%BB%BA%E3%80%81%E7%BC%96%E8%BE%91%E6%96%87%E4%BB%B6> "编辑")文件，将离您所处地理位置最近的镜像移到文件的头部，同时也应该考虑一些其他的评判标准。此外，您可使用 [reflector](<../zh-cn/Reflector.html> "Reflector") 根据几种不同的评判标准来创建新的 mirrorlist 文件。 

如果 `/etc/pacman.d/mirrorlist` 文件中没有合适的镜像站，可以手动从 [archlinux 官方网站的镜像站列表](<https://archlinux.org/mirrorlist/>)下载一份。这需要您所在的网络能够正常访问 Arch Linux 官方网站。例如，使用 [curl(1)](<https://man.archlinux.org/man/curl.1>) 下载位于中国大陆的 HTTPS 镜像站： 
    
    # curl -L 'https://archlinux.org/mirrorlist/?country=CN&protocol=https' -o /etc/pacman.d/mirrorlist
    
或者，也可以通过安装 [pacman-mirrorlist](<https://archlinux.org/packages/?name=pacman-mirrorlist>)包 来获取按国家分列的原始镜像列表。这么做的缺点是，pacman 仍然会使用当前配置中的镜像站来下载软件包数据库和 [pacman-mirrorlist](<https://archlinux.org/packages/?name=pacman-mirrorlist>)包，这可能会很慢。在挑选了能用的镜像之后，可以执行 
    
    # pacman -Sy pacman-mirrorlist
    
再将 `/etc/pacman.d/mirrorlist.pacnew` 复制到 `/etc/pacman.d/mirrorlist` 并进行编辑。 

或者，如果您记得想要使用的镜像站的 URL，那么可以手动编辑 `/etc/pacman.d/mirrorlist` 文件并手动输入 URL。 

这个文件接下来还会被 _pacstrap_ 拷贝到新系统里，所以请您确保设置正确。 

###  安装必需的软件包

除 `/etc/pacman.d/mirrorlist` 之外的配置不会从 Live 环境传递到安装的系统中。因此，若您在 Live 环境的`/etc/pacman.conf` 里启用了[非官方用户仓库](<../zh-cn/%E9%9D%9E%E5%AE%98%E6%96%B9%E7%94%A8%E6%88%B7%E4%BB%93%E5%BA%93.html> "非官方用户仓库")（如 [archlinuxcn](<../zh-cn/Arch_Linux_%E4%B8%AD%E6%96%87%E7%A4%BE%E5%8C%BA%E4%BB%93%E5%BA%93.html> "Archlinuxcn") 仓库），请在新安装的系统手动启用之。[base](<https://archlinux.org/packages/?name=base>)包 是唯一的必需软件包，其**不包含** 来自 Live 系统的所有工具，因此需要安装其他的软件包。

  * [文件系统的用户空间工具](<../zh-cn/%E6%96%87%E4%BB%B6%E7%B3%BB%E7%BB%9F.html> "文件系统")（比如 [XFS](<../zh-cn/XFS.html> "XFS") 和 [Btrfs](<../zh-cn/Btrfs.html> "Btrfs") 文件系统对应的管理工具），可用于创建和管理[文件系统](<../zh-cn/%E6%96%87%E4%BB%B6%E7%B3%BB%E7%BB%9F.html> "文件系统")以及 [fsck](<../zh-cn/Fsck.html> "Fsck")。
  * 访问和管理 [RAID](<../zh-cn/RAID.html#%E5%AE%89%E8%A3%85> "RAID") 或 [LVM](<../zh-cn/LVM.html#%E5%87%86%E5%A4%87> "LVM") 分区的工具。
  * 未包含在 [linux-firmware](<https://archlinux.org/packages/?name=linux-firmware>)包 中的额外必需[固件](<../zh-cn/Linux_%E5%9B%BA%E4%BB%B6.html> "Linux 固件")（如用于[内建音频](<../zh-cn/ALSA.html#%E5%9B%BA%E4%BB%B6> "ALSA")的 [sof-firmware](<https://archlinux.org/packages/?name=sof-firmware>)包、用于 Marvell 无线的 [linux-firmware-marvell](<https://archlinux.org/packages/?name=linux-firmware-marvell>)包 以及[博通无线网卡](<../zh-cn/%E5%8D%9A%E9%80%9A%E6%97%A0%E7%BA%BF%E7%BD%91%E5%8D%A1.html> "博通无线网卡")的固件包)。

  * [GPG 密钥环](<../zh-cn/Pacman/%E8%BD%AF%E4%BB%B6%E5%8C%85%E7%AD%BE%E5%90%8D.html> "Pacman/软件包签名")（如 [archlinuxcn-keyring](<https://github.com/archlinuxcn/repo/tree/master/archlinuxcn/archlinuxcn-keyring>)[CNRepo](<../zh-cn/Arch_Linux_%E4%B8%AD%E6%96%87%E7%A4%BE%E5%8C%BA%E4%BB%93%E5%BA%93.html> "Arch Linux 中文社区仓库")，若启用了某些[非官方用户仓库](<../zh-cn/%E9%9D%9E%E5%AE%98%E6%96%B9%E7%94%A8%E6%88%B7%E4%BB%93%E5%BA%93.html> "非官方用户仓库")。）

  * 访问 [man](<../zh-cn/Man_%E6%89%8B%E5%86%8C.html> "Man") 和 [info](<../zh-cn/GNU/%E6%96%87%E6%A1%A3.html#Texinfo> "GNU/文档") 文档的工具：[man-db](<https://archlinux.org/packages/?name=man-db>)包, [man-pages](<https://archlinux.org/packages/?name=man-pages>)包 和 [texinfo](<https://archlinux.org/packages/?name=texinfo>)包。

要安装您选择的其他软件包或软件包组，请将它们的名字追加到下面的 [pacstrap(8)](<https://man.archlinux.org/man/pacstrap.8>) 命令后（以空格分隔），或者也可以在 [Chroot 进入新系统](<#Chroot>)后使用 [pacman](<../zh-cn/Pacman.html> "Pacman") 手动[安装](<../zh-cn/Help:%E9%98%85%E8%AF%BB.html#%E5%AE%89%E8%A3%85%E8%BD%AF%E4%BB%B6%E5%8C%85> "安装")。 

例如，一个包含 Linux [内核](<../zh-cn/%E5%86%85%E6%A0%B8.html> "内核")和常见硬件固件的基本安装： 
    
    # pacstrap -K /mnt base linux linux-firmware
    
**注意：** 有时，由于安装映像中的软件包版本过旧，会导致在运行 _pacstrap_ 的时候无法正常安装软件包（提示为文件签名损坏）。 请参阅 [Pacman/软件包签名#手动更新密钥环](<../zh-cn/Pacman/%E8%BD%AF%E4%BB%B6%E5%8C%85%E7%AD%BE%E5%90%8D.html#%E6%89%8B%E5%8A%A8%E6%9B%B4%E6%96%B0%E5%AF%86%E9%92%A5%E7%8E%AF> "Pacman/软件包签名") 中提到的解决方案。 如果您启用了[中文社区仓库](<../zh-cn/Arch_Linux_%E4%B8%AD%E6%96%87%E7%A4%BE%E5%8C%BA%E4%BB%93%E5%BA%93.html> "Arch Linux 中文社区仓库")或其他[非官方用户仓库](<../zh-cn/%E9%9D%9E%E5%AE%98%E6%96%B9%E7%94%A8%E6%88%B7%E4%BB%93%E5%BA%93.html> "非官方用户仓库")，也不要忘了一同更新其密钥环。

**提示：**

  * 您可将 [linux](<https://archlinux.org/packages/?name=linux>)包 软件包替换为[内核](<../zh-cn/%E5%86%85%E6%A0%B8.html> "内核")页面中介绍的其他内核软件包，而在[容器](<https://en.wikipedia.org/wiki/Container_\(virtualization\)> "wikipedia:Container \(virtualization\)")中安装时可将其忽略。
  * 在虚拟机或容器中安装时可以不需要固件包。
  * 使用 pacstrap 安装的软件包只需保证系统能够启动即可，其他软件包可以之后再行安装或替换。
  * 参见 [Pacman](<../zh-cn/Pacman.html> "Pacman") 以增强您的 pacman 使用体验。

##  配置系统

###  生成 fstab 文件

生成 [fstab](<../zh-cn/Fstab.html> "Fstab") 文件以使需要的文件系统（如启动目录 `/boot`）在启动时被自动挂载，用 `-U` 或 `-L` 选项分别设置 UUID 或卷标： 
    
    # genfstab -U /mnt > /mnt/etc/fstab
    
**强烈建议** 在执行完以上命令后，检查一下生成的 `/mnt/etc/fstab` 文件是否正确。如果有问题，最好在现在手动修改。对于熟练者，可以自行编辑 `/mnt/etc/fstab` 文件。 

可以手动修改 `/mnt/etc/fstab` 添加挂载选项（参见 [mount(8) § FILESYSTEM-INDEPENDENT_MOUNT_OPTIONS](<https://man.archlinux.org/man/mount.8#FILESYSTEM-INDEPENDENT_MOUNT_OPTIONS>) 和 [ext4(5) § MOUNT_OPTIONS](<https://man.archlinux.org/man/ext4.5#MOUNT_OPTIONS>)）。例如，给[固态硬盘](<../zh-cn/%E5%9B%BA%E6%80%81%E7%A1%AC%E7%9B%98.html> "固态硬盘")的 [ext4](<../zh-cn/Ext4.html> "Ext4") 文件系统添加 `discard` 选项以启用 [TRIM](<../zh-cn/%E5%9B%BA%E6%80%81%E7%A1%AC%E7%9B%98.html#TRIM> "TRIM")。 

**警告：** 在启用 TRIM 功能前，请确保 SSD 支持 TRIM，否则可能导致数据丢失！

###  chroot 到新安装的系统

接下来的步骤需要像启动到新安装的系统一样直接与其环境、工具和配置进行交互，请 [chroot](<../zh-cn/Change_root.html> "Change root") 到新安装的系统： 
    
    # arch-chroot /mnt
    
**注意：** 由于部分 [systemd](<../zh-cn/Systemd.html> "Systemd") 工具如 [hostnamectl(1)](<https://man.archlinux.org/man/hostnamectl.1>)、[localectl(1)](<https://man.archlinux.org/man/localectl.1>) 和 [timedatectl(1)](<https://man.archlinux.org/man/timedatectl.1>) 需要有效的 [dbus](<../zh-cn/D-Bus.html> "Dbus") 连接，因此无法在 [chroot(1)](<https://man.archlinux.org/man/chroot.1>) 内使用这些工具[[2]](<https://github.com/systemd/systemd/issues/798#issuecomment-126568596>)。

**提示：** 此处使用的是`[arch-chroot](<../zh-cn/Chroot.html#%E4%BD%BF%E7%94%A8_arch-chroot> "Chroot")`而不是直接使用`[chroot](<../zh-cn/Chroot.html#%E4%BD%BF%E7%94%A8_chroot> "Chroot")`，注意不要输错了。

###  设置时间和时区

为顺应人类习惯（如显示正确的当地时间及处理夏令时），请设置[时区](<../zh-cn/%E7%B3%BB%E7%BB%9F%E6%97%B6%E9%97%B4.html#%E6%97%B6%E5%8C%BA> "系统时间")： 
    
    # ln -sf /usr/share/zoneinfo/_地区名_ /_城市名_ /etc/localtime
    
**提示：** 例如，在中国大陆需要将时区设置为北京时间，那么请运行 `# ln -sf /usr/share/zoneinfo/Asia/Shanghai /etc/localtime`。时区名称是上海而非北京，是因为上海是[该时区内人口最多的城市](<https://en.wikipedia.org/wiki/Time_in_China#IANA_time_zone_database> "wikipedia:Time in China")。

然后运行 [hwclock(8)](<https://man.archlinux.org/man/hwclock.8>) 以生成 `/etc/adjtime`： 
    
    # hwclock --systohc
    
这个命令假定已设置硬件时间为 [UTC 时间](<https://zh.wikipedia.org/wiki/UTC> "zhwp:UTC")。详细信息请查看[系统时间#时间标准](<../zh-cn/%E7%B3%BB%E7%BB%9F%E6%97%B6%E9%97%B4.html#%E6%97%B6%E9%97%B4%E6%A0%87%E5%87%86> "系统时间")。 

为了防止时钟漂移并确保时间准确，请手动启用 [NTP](<../zh-cn/Category:NTP.html> "NTP")（网络时间协议，Network Time Protocol）客户端（例如 [systemd-timesyncd](<../zh-cn/Systemd-timesyncd.html> "Systemd-timesyncd")）设置[时间同步](<../zh-cn/%E7%B3%BB%E7%BB%9F%E6%97%B6%E9%97%B4.html#%E6%97%B6%E9%92%9F%E5%90%8C%E6%AD%A5> "系统时间")。 

###  区域和本地化设置

程序和库如果需要本地化文本，都依赖[区域设置](<../zh-cn/Locale.html> "Locale")，后者明确规定了地域、货币、时区日期的格式、字符排列方式和其他本地化标准。 

需要设置这两个文件：`locale.gen` 与 `locale.conf`。 

[编辑](<../zh-cn/Help:%E9%98%85%E8%AF%BB.html#%E6%B7%BB%E5%8A%A0%E3%80%81%E5%88%9B%E5%BB%BA%E3%80%81%E7%BC%96%E8%BE%91%E6%96%87%E4%BB%B6> "编辑") `/etc/locale.gen`，然后取消掉 `en_US.UTF-8 UTF-8` 和其他需要的 UTF-8 [区域设置](<../zh-cn/Locale.html> "Locale")前的注释（**#** ）。 

接着执行 `locale-gen` 以生成 locale 信息： 
    
    # locale-gen
    
然后创建 [locale.conf(5)](<https://man.archlinux.org/man/locale.conf.5>) 文件，并[编辑设定 LANG 变量](<../zh-cn/Locale.html#%E7%B3%BB%E7%BB%9F%E5%8C%BA%E5%9F%9F%E8%AE%BE%E7%BD%AE> "Locale")，比如： 
    
    /etc/locale.conf
    
    LANG=_en_US.UTF-8_

另外对于中文用户： 

**提示：**

  * 用户可以设置自己的 [locale](<../zh-cn/Locale.html> "Locale")，详情请参阅 [在用户会话中覆盖系统区域设置](<../zh-cn/Locale.html#%E5%9C%A8%E7%94%A8%E6%88%B7%E4%BC%9A%E8%AF%9D%E4%B8%AD%E8%A6%86%E7%9B%96%E7%B3%BB%E7%BB%9F%E5%8C%BA%E5%9F%9F%E8%AE%BE%E7%BD%AE> "Locale") 或 [设置当前区域](<../zh-cn/Locale.html#%E8%AE%BE%E7%BD%AE%E5%BD%93%E5%89%8D%E5%8C%BA%E5%9F%9F> "Locale")；
  * 将系统 locale 设置为 `en_US.UTF-8` ，系统的[日志](<../zh-cn/Systemd.html#Journal> "Systemd")就会用英文显示，这样更容易判断和处理问题； 
    * 也可以设置为 `en_GB.UTF-8`［英语（英国）］或 `en_SG.UTF-8`［英语（新加坡）］，附带以下特点： 
      * 进入桌面环境后以 24 小时制显示时间；
      * [LibreOffice](<../zh-cn/LibreOffice.html> "LibreOffice") 等办公软件的纸张尺寸会默认为 `A4` 而非 `Letter(US)`；
      * 可尽量避免不必要且可能造成处理麻烦的英制单位。
    * 设置的 LANG 变量需与 locale 设置一致，否则会出现以下错误：

    `Cannot set LC_CTYPE to default locale: No such file or directory`

**注意：** 如果在此处设置为中文 locale，这可能会导致 [tty](<../zh-cn/Getty.html> "Getty") 上中文显示为方块（因为 TTY 下没有 CJK 字体）。如果您主要使用[桌面环境](<../zh-cn/%E6%A1%8C%E9%9D%A2%E7%8E%AF%E5%A2%83.html> "桌面环境")而非 TTY，则不用在意这个提醒。您也可以参照[简体中文本地化#终端中文支持](<../zh-cn/%E7%AE%80%E4%BD%93%E4%B8%AD%E6%96%87%E6%9C%AC%E5%9C%B0%E5%8C%96.html#%E7%BB%88%E7%AB%AF%E4%B8%AD%E6%96%87%E6%94%AF%E6%8C%81> "简体中文本地化")

如果需要修改[#控制台键盘布局和字体](<#%E6%8E%A7%E5%88%B6%E5%8F%B0%E9%94%AE%E7%9B%98%E5%B8%83%E5%B1%80%E5%92%8C%E5%AD%97%E4%BD%93>)，可编辑 [vconsole.conf(5)](<https://man.archlinux.org/man/vconsole.conf.5>) 使其长期生效，以德语键盘（qwertz）为例： 
    
    /etc/vconsole.conf
    
    KEYMAP=_de-latin1_

**注意：** 如果键盘为标准美式键盘（中国最常见的键盘布局），请勿照抄如上设置。

###  网络配置

为您的系统设置一个固定且易于辨识的名称（在联网环境中尤其有用），请[创建](<../zh-cn/Help:%E9%98%85%E8%AF%BB.html#%E6%B7%BB%E5%8A%A0%E3%80%81%E5%88%9B%E5%BB%BA%E3%80%81%E7%BC%96%E8%BE%91%E6%96%87%E4%BB%B6> "创建") [hostname](<../zh-cn/%E7%BD%91%E7%BB%9C%E9%85%8D%E7%BD%AE.html#%E8%AE%BE%E7%BD%AE%E8%AE%A1%E7%AE%97%E6%9C%BA%E5%90%8D> "网络配置") 文件： 
    
    /etc/hostname
    
    _主机名_
    
**提示：** 请参阅 [RFC 1178](<https://tools.ietf.org/html/rfc1178> "rfc:1178") 以获取一些关于为计算机取名的建议。如 [hostname(7)](<https://man.archlinux.org/man/hostname.7>) 所述，其必须包含 1-63 个字符，仅使用小写的 `a-z`、`0-9` 以及 `-`，但不得以 `-` 开头。

请接着完成新安装的环境的[网络配置](<../zh-cn/%E7%BD%91%E7%BB%9C%E9%85%8D%E7%BD%AE.html> "网络配置")，配置过程中可能需要安装合适的[网络管理](<../zh-cn/%E7%BD%91%E7%BB%9C%E9%85%8D%E7%BD%AE.html#%E7%BD%91%E7%BB%9C%E7%AE%A1%E7%90%86> "网络配置")软件。 

**警告：** 请按上述网络配置指引正确配置好网络后再重新启动，否则系统重新启动后可能无法连接网络（不过可以用 LiveCD 重新进入 [arch-chroot](<../zh-cn/Chroot.html#%E4%BD%BF%E7%94%A8_arch-chroot> "Chroot") 进行配置）。例如在虚拟机软件 [VirtualBox](<../zh-cn/VirtualBox.html> "VirtualBox") [安装](<../zh-cn/VirtualBox/%E5%9C%A8%E8%99%9A%E6%8B%9F%E6%9C%BA%E4%B8%AD%E5%AE%89%E8%A3%85_Arch_Linux.html> "VirtualBox/在虚拟机中安装 Arch Linux")并使用桥接模式时就需要配置 DHCP 。

###  关于 initramfs

通常不需要自己创建新的 _initramfs_ ，因为在执行 _pacstrap_ 时已经安装 [linux](<https://archlinux.org/packages/?name=linux>)包，这时已经运行过 [mkinitcpio](<../zh-cn/Mkinitcpio.html> "Mkinitcpio") 了。 

如果是 [LVM](</wzh/index.php?title=%E5%9C%A8_LVM_%E4%B8%8A%E5%AE%89%E8%A3%85_Arch_Linux&action=edit&redlink=1> "在 LVM 上安装 Arch Linux（页面不存在）")（英语：[Install Arch Linux on LVM#Adding mkinitcpio hooks](<https://wiki.archlinux.org/title/Install_Arch_Linux_on_LVM#Adding_mkinitcpio_hooks> "en:Install Arch Linux on LVM")）、[系统加密](<../zh-cn/Dm-crypt.html> "Dm-crypt")或 [RAID](<../zh-cn/RAID.html#%E9%85%8D%E7%BD%AE_mkinitcpio> "RAID") 等分区配置，请修改 [mkinitcpio.conf](<../zh-cn/Mkinitcpio.html> "Mkinitcpio") 并用以下命令重新创建 Initramfs。此外，如果您[更改了默认的控制台键盘映射](<#%E5%8C%BA%E5%9F%9F%E5%92%8C%E6%9C%AC%E5%9C%B0%E5%8C%96%E8%AE%BE%E7%BD%AE>)，只需要重新创建 initramfs 即可： 
    
    # mkinitcpio -P
    
###  设置 root 密码

使用以下命令为 root 用户设置一个安全的[密码](<../zh-cn/%E7%94%A8%E6%88%B7%E5%92%8C%E7%94%A8%E6%88%B7%E7%BB%84.html#%E7%94%A8%E6%88%B7%E4%BF%A1%E6%81%AF%E5%AD%98%E5%82%A8> "用户和用户组")以便进行管理操作： 
    
    # passwd
    
**提示：** 建议为 [root](<../zh-cn/%E7%94%A8%E6%88%B7%E5%92%8C%E7%94%A8%E6%88%B7%E7%BB%84.html#%E6%A6%82%E8%A7%88> "Root") 用户选择一个[强密码](<../zh-cn/%E5%AE%89%E5%85%A8.html#%E9%80%89%E6%8B%A9%E5%AE%89%E5%85%A8%E7%9A%84%E5%AF%86%E7%A0%81> "安全")并[保护其安全](<../zh-cn/%E5%AE%89%E5%85%A8.html#%E7%BB%B4%E6%8A%A4%E5%AF%86%E7%A0%81%E5%AE%89%E5%85%A8> "安全")。

###  安装引导程序

需要正确安装 Linux 引导加载程序，才能在安装后启动系统。您应当已在[前文](<#%E5%AE%89%E8%A3%85%E5%BF%85%E9%9C%80%E7%9A%84%E8%BD%AF%E4%BB%B6%E5%8C%85>)安装了一个[引导加载程序](<../zh-cn/Arch_%E7%9A%84%E5%90%AF%E5%8A%A8%E6%B5%81%E7%A8%8B.html#%E5%BC%95%E5%AF%BC%E5%8A%A0%E8%BD%BD%E7%A8%8B%E5%BA%8F> "引导加载程序")软件包，请参阅其 wiki 页面以配置它。 

**警告：** 这是安装的最后一步也是关键的一步，请点击上述链接并按指引正确安装好引导加载程序后再重新启动。否则计算机重新启动后将无法正常进入 Arch Linux 系统。

##  重新启动计算机

输入 `exit` 或按 `Ctrl+d` 退出 chroot 环境。 

可选用 `umount -R /mnt` 手动卸载被挂载的分区：这有助于发现任何“繁忙”的分区，并通过 [fuser(1)](<https://man.archlinux.org/man/fuser.1>) 查找原因。 

最后，通过执行 `reboot` 重启系统， _systemd_ 将自动卸载仍然挂载的任何分区。这时候不要忘记移除安装介质，然后使用 [root](<../zh-cn/%E7%94%A8%E6%88%B7%E5%92%8C%E7%94%A8%E6%88%B7%E7%BB%84.html#%E6%A6%82%E8%A7%88> "Root") 账户登录到新系统。

**提示：** 若系统出现问题，可尝试重新进入[Chroot](<../zh-cn/Chroot.html> "Chroot") 环境以补充、完善安装和配置，或查看[系统日志](<../zh-cn/Systemd.html#Journal> "Systemd")以定位、修复问题。
