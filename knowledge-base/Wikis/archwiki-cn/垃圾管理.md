为防止误删文件，可以使用垃圾桶。为确保多个应用程序之间的兼容性，可以使用遵循 [FreeDesktop.org's Trash specification](<https://specifications.freedesktop.org/trash-spec/latest>) 的软件（CLI、GUI 或库）。 

**注意：** 某些应用程序可以在每个文件系统中使用一个垃圾桶（请参阅规范）。

**警告：** 删除其他文件系统上的文件时，由于文件会在文件系统之间传输，因此使用垃圾桶可能会导致一些延迟。在这种情况下，你可能需要为每个文件系统创建一个垃圾文件夹（参见规范中的 `Trash directories`）。

##  创建垃圾桶

[![](../File:Tango-view-fullscreen.png)](<../File:Tango-view-fullscreen.png>)**这篇文章的某些内容需要扩充。**

**原因：** 添加其它工具 (在 [Talk:垃圾管理](<../zh-cn/Talk:%E5%9E%83%E5%9C%BE%E7%AE%A1%E7%90%86.html>) 中讨论)

一些 GUI 文件管理器，如 [GNOME Files](<../zh-cn/GNOME_Files.html> "GNOME Files") 和 [Dolphin](<../zh-cn/Dolphin.html> "Dolphin") 本身就支持将文件移入垃圾桶。还有一些 CLI 工具： 

  * [glib2](<https://archlinux.org/packages/?name=glib2>)包 提供了一条 `gio trash _file..._`命令，用于将文件移入垃圾桶，而 `gio trash --empty` 则会清空垃圾桶。另请参见 [gio(1) § COMMANDS](<https://man.archlinux.org/man/gio.1#COMMANDS>) 中的 "trash"。要通过 `gio trash` 命令完全支持垃圾管理，需要安装 [gvfs](<https://archlinux.org/packages/?name=gvfs>)包 软件包。
  * [kde-cli-tools](<https://archlinux.org/packages/?name=kde-cli-tools>)包 提供了一条 `kioclient5` 命令，可以使用 `kioclient5 move _file_ trash:/` 将 `_file_` 移至垃圾桶。
  * **trash-cli** — 实现 [FreeDesktop.org's Trash specification](<https://specifications.freedesktop.org/trash-spec/latest>) 的命令行界面。

     <https://github.com/andreafrancia/trash-cli> || [trash-cli](<https://archlinux.org/packages/?name=trash-cli>)包

  * **trashman** — 一个 Python 垃圾管理器。

     <https://github.com/Kwpolska/trashman> || [trashman](<https://aur.archlinux.org/packages/trashman/>)AUR

  * **rmtrash** — trash-put 与 GNUs rm 和 rmdir 兼容。

     <https://github.com/PhrozenByte/rmtrash> || [rmtrash](<https://aur.archlinux.org/packages/rmtrash/>)AUR

  * **trash-d** — GNU rm 的直接替代品，注重兼容性，性能更好，用 D 语言编写。

     <https://github.com/rushsteve1/trash-d> || [trash-d](<https://aur.archlinux.org/packages/trash-d/>)AUR

  * **trashy** — 用 rust 写成的 rm 和 trash-cli 简单、快速、功能丰富的替代品

     <https://github.com/oberblastmeister/trashy> || [trashy](<https://aur.archlinux.org/packages/trashy/>)AUR

  * **gtrash** — 用 Go 编写的功能丰富的 CLI 垃圾管理器

     <https://github.com/umlx5h/gtrash> || [gtrash-bin](<https://aur.archlinux.org/packages/gtrash-bin/>)AUR

  * **rmw（ReMove to Waste）** — 一个可配置的 CLI 垃圾管理工具，可在 X 天后清除垃圾，并与 [FreeDesktop.org's Trash specification](<https://specifications.freedesktop.org/trash-spec/latest>) 集成，用 C 语言编写

     <https://theimpossibleastronaut.com/rmw-website/> || [rmw](<https://aur.archlinux.org/packages/rmw/>)AUR

##  垃圾空间使用管理

为了防止垃圾桶占用过多空间，您可以自己清空垃圾桶，或者制定清理政策

###  标准

[![](../File:Tango-view-fullscreen.png)](<../File:Tango-view-fullscreen.png>)**这篇文章的某些内容需要扩充。**

###  自动化

要自动清空垃圾桶，可以使用 [cron](<../zh-cn/Cron.html> "Cron")、[systemd/Timers](<../zh-cn/Systemd/%E5%AE%9A%E6%97%B6%E5%99%A8.html> "Systemd/Timers") 或 [inotify](</wzh/index.php?title=Inotify&action=edit&redlink=1> "Inotify（页面不存在）")（使用 inotify，只有新的删除才会触发垃圾桶自动清空）。 

##  问题解决

###  无法找到或创建垃圾目录

####  外置硬盘

`/etc/fstab` 可能未正确设置。请注意，`ID` 必须用自己的用户 ID 替换通过 `$ id`。 
    
    /etc/fstab
    
    # <device>                                <dir>              <type>    <options>         <dump> <fsck>
    UUID=75667752-8b06-11ea-bc55-0242ac130003 /externalDrive     ntfs-3g   defaults,uid=ID   0      1

确保驱动器根目录包含 `~/.Trash-1000/{expunged,files,info` 并为该目录和子目录设置相关读取权限。
