[火焰截图（Flameshot）](<https://flameshot.org>)是一款屏幕截图工具。该程序提供了交互式图形界面及多种控制功能，用户可选取指定捕获区域、移动并调整捕获窗口尺寸、使用常见绘图工具（铅笔、直线、矩形、圆形、模糊（像素化）、撤销/重做）进行编辑，并选择输出方式（复制到剪贴板、保存至磁盘、上传至 [Imgur](<https://imgur.com>)、通过其他程序打开）。 

##  安装

[安装](<../zh-cn/Help:%E9%98%85%E8%AF%BB.html#%E5%AE%89%E8%A3%85%E8%BD%AF%E4%BB%B6%E5%8C%85> "安装") [flameshot](<https://archlinux.org/packages/?name=flameshot>)包 软件包。若需开发版本，可安装 [flameshot-git](<https://aur.archlinux.org/packages/flameshot-git/>)AUR。 

##  问题解决

###  子命令立即退出而无输出

在 X 会话期间随时将火焰截图作为后台进程启动： 
    
    $ flameshot &
    
###  无托盘图标

可通过多种方式解决，此处提供一种方案： 

[安装](<../zh-cn/Help:%E9%98%85%E8%AF%BB.html#%E5%AE%89%E8%A3%85%E8%BD%AF%E4%BB%B6%E5%8C%85> "安装") [gnome-shell-extension-topicons-plus](<https://aur.archlinux.org/packages/gnome-shell-extension-topicons-plus/>)AUR。 

然后在 GNOME Tweaks 应用程序中（若未安装需先下载）： 

  1. 点击“扩展”选项卡。
  2. 启用“Topicons plus”以将传统托盘图标移至顶部面板。
  3. 完成！

###  火焰截图无法识别当前可见窗口

若安装了 [xf86-video-intel](<https://archlinux.org/packages/?name=xf86-video-intel>)包 可能出现此问题。 

移除 [xf86-video-intel](<https://archlinux.org/packages/?name=xf86-video-intel>)包 软件包，并确保 `/etc/X11/xorg.conf.d/` 目录下无该软件包的残留 X11 配置。 

随后重启系统。 

此问题曾在火焰截图 GitHub 仓库讨论：[[1]](<https://github.com/flameshot-org/flameshot/issues/1677>)。 

###  Wayland 下火焰截图选区操作卡顿

[PR #2766](<https://github.com/flameshot-org/flameshot/pull/2766>) 导致在 Wayland 中选择截图区域时出现严重卡顿。该问题正在 [issue #3039](<https://github.com/flameshot-org/flameshot/issues/3039>) 中追踪，[PR #3059](<https://github.com/flameshot-org/flameshot/pull/3059>) 可能包含修复方案。临时解决方案是使用无此问题的 12.1.0-1 版本。 

###  Sway/wlroots 环境下火焰截图启动报错

截图时可能显示以下错误： 
    
    flameshot: error: Unable to capture screen
    flameshot: error: Unable to capture screen
    qt.qpa.wayland: Wayland does not support QWindow::requestActivate()
    flameshot: info: Screenshot aborted.
    
需安装以下软件包使火焰截图在 Wayland 下正常工作： 

[xdg-desktop-portal](<https://archlinux.org/packages/?name=xdg-desktop-portal>)包、[xdg-desktop-portal-wlr](<https://archlinux.org/packages/?name=xdg-desktop-portal-wlr>)包、[grim](<https://archlinux.org/packages/?name=grim>)包。 

安装完成后即可正常启动火焰截图。 

更多信息请参阅 [Sway 维基页面](<https://github.com/flameshot-org/flameshot/blob/master/docs/Sway%20and%20wlroots%20support.md>)。 

###  火焰截图在 KDE Plasma Wayland 多显示器环境下只能在单个屏幕生效

火焰截图覆盖层仅在单个显示器显示。GitHub 仓库中有人建议了[解决方案](<https://github.com/flameshot-org/flameshot/issues/3073#issuecomment-1740187784>)。 

基本思路是用 KDE Plasma 配置覆盖火焰截图叠加窗口的初始位置，这样火焰截图叠加窗口才能显示在所有连接的显示器上。为此，启动“窗口规则”应用程序，并创建一个新的窗口规则，设置如下： 

  * **窗口类 (应用程序)** ：flameshot
  * **窗口类型** ：正常窗口
  * **窗口标题** ：完全匹配：flameshot
  * **位置** ：强制 0x0
  * **全屏** ：强制 否
  * **遵守大小和位置限制** ：强制 是
  * **窗口置顶** ：强制 是

###  火焰截图在 Sway 的多显示器环境下只能在单个屏幕生效

火焰截图覆盖层仅在单个显示器显示。GitHub 仓库中有人建议了[解决方案](<https://github.com/flameshot-org/flameshot/issues/2364#issuecomment-1086129055>)。 

该解决方案的核心是强制火焰截图窗口/覆盖层以浮动模式启动，而不是全屏模式。可以在 Sway 配置文件中添加以下行来实现： 
    
    for_window [app_id="flameshot"] border pixel 0, floating enable, fullscreen disable, move absolute position 0 0
    
另外，也可以强制 Flameshot 运行在 XWayland 模式下： 
    
    QT_QPA_PLATFORM=xcb flameshot gui
    
但需要注意，此方法虽然修复了多显示器问题，但会导致键盘快捷键失效，导致火焰截图只能通过鼠标操作。 

###  火焰截图在 KDE Dolphin 中无法生成缩略图

在 Plasma 6 中，使用 Dolphin 文件选择器保存截图时，如果目录中存在其他图片文件，则不会显示缩略图。在终端中运行 Flameshot 时，会看到以下错误信息： 
    
    kf.service.services: KServiceTypeTrader: serviceType "ThumbCreator" not found
    
安装 [kio5-extras](<https://archlinux.org/packages/?name=kio5-extras>)包 可解决该问题。[[2]](<https://forums.opensuse.org/t/dolphin-cant-create-photo-thumbnails-servicetype-thumbcreator-not-found/170259/4.>)
