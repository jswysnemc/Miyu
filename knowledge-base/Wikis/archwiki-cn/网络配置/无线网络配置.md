网络配置的主页面是[网络配置](<../../zh-cn/%E7%BD%91%E7%BB%9C%E9%85%8D%E7%BD%AE.html> "网络配置")。 

配置无线网络一般分两步：第一步是识别硬件、安装正确的驱动程序并进行配置，安装盘中已经包含驱动，但是通常需要额外安装；第二步是选择一种管理无线连接的方式。这篇文章涵盖了这两方面，并提供了无线管理工具的链接地址。 

[#iw](<#iw>) 段落描述了如何使用 [iw](<https://archlinux.org/packages/?name=iw>)包 来手动管理你的网络接口/无线局域网连接。[网络配置#网络管理器](<../../zh-cn/%E7%BD%91%E7%BB%9C%E9%85%8D%E7%BD%AE.html#%E7%BD%91%E7%BB%9C%E7%AE%A1%E7%90%86%E5%99%A8> "网络配置")段落介绍了几个可以自动管理您无线网络接口的程序。这些程序中的某些会包含一个图形界面，而所有的程序都支持网络配置文件（在频繁切换网络时有用，比如使用笔记本电脑的场合）。 

##  设备驱动

默认的 Arch Linux 内核是 _模块化_ 的，硬件的设备驱动作为[内核模块](<../../zh-cn/Kernel_modules.html> "Kernel modules")保存在硬盘上。启动时，[udev](<../../zh-cn/Udev.html> "Udev") 会根据硬件加载不同的驱动模块，从而允许创建网络接口。 

除了相应的驱动程序之外，某些无线芯片组还需要固件。[linux-firmware](<https://archlinux.org/packages/?name=linux-firmware>)包 提供了很多固件，然而专有固件并不包含在内，需要另行安装。这部分指南请参考 [#安装 driver/firmware](<#%E5%AE%89%E8%A3%85_driver/firmware>)。 

**注意：** 如果 udev 在启动时未加载正确的模块，只需 [手动加载](<../../zh-cn/%E5%86%85%E6%A0%B8%E6%A8%A1%E5%9D%97.html#%E6%89%8B%E5%8A%A8%E5%8A%A0%E8%BD%BD> "内核模块") 即可。 如果 udev 为一台设备加载多个驱动程序，则产生的冲突可能会导致配置失败。 确保将不需要的模块加入 [黑名单](<../../zh-cn/%E5%86%85%E6%A0%B8%E6%A8%A1%E5%9D%97.html#%E9%BB%91%E5%90%8D%E5%8D%95> "内核模块")。

###  检查驱动状态

根据设备的连接方式是 PCI(e) 或 USB，对应地执行 `lspci -k` 或 `lsusb -v` 可以检查设备驱动是否已经加载： 
    
    $ lspci -k
    
    06:00.0 Network controller: Intel Corporation WiFi Link 5100
    	Subsystem: Intel Corporation WiFi Link 5100 AGN
    	Kernel driver in use: iwlwifi
    	Kernel modules: iwlwifi
    
**注意：** 如果网卡是 USB 设备，以 root 权限执行 ` grep usbcore` 的输出应当类似于 `usbcore: registered new interface driver rtl8187`。 

If you do not see the card at all, it may not be seated/plugged in properly in its PCI(e) slot/USB port. Try re-plugging it in.

同时也要通过 `ip link` 命令来查看无线接口是否已经创建。通常来说，[设备名](<../../zh-cn/Network_configuration.html#%E6%9B%B4%E6%94%B9%E6%8E%A5%E5%8F%A3%E5%90%8D%E7%A7%B0> "Network configuration")以字母“w”开头，例如`wlan0` 或 `wlp2s1`。确认之后，使用以下命令使设备上线： 
    
    # ip link set _< 设备名>_ up
    
例如，接口设备名是 `wlan0`，此命令就是 `ip link set wlan0 up`。 

**注意：**

  * 如果你从命令中得到如下错误：`RTNETLINK answers: Operation not possible due to RF-kill` 错误，请确保设备没有被 软锁定 或 硬锁定，详情请参考 [Rfkill 说明](<#Rfkill_%E8%AF%B4%E6%98%8E>)。
  * 如果你从命令中得到如下错误：`SIOCSIFFLAGS: No such file or directory`， 那么大概率您的无线芯片组需要额外的固件来运行。

检查内核信息中关于已加载固件的信息： 
    
    # dmesg | grep firmware 
    
    [   7.148259] iwlwifi 0000:02:00.0: loaded firmware version 39.30.4.1 build 35138 op_mode iwldvm
    
如果没有相关的输出，那么需要在消息中查找关于您之前指定的模块的完整输出（本示例中的模块是`iwlwifi`），然后查找对应的错误信息： 
    
    # dmesg | grep iwlwifi 
    
    [   12.342694] iwlwifi 0000:02:00.0: irq 44 for MSI/MSI-X
    [   12.353466] iwlwifi 0000:02:00.0: loaded firmware version 39.31.5.1 build 35138 op_mode iwldvm
    [   12.430317] iwlwifi 0000:02:00.0: CONFIG_IWLWIFI_DEBUG disabled
    ...
    [   12.430341] iwlwifi 0000:02:00.0: Detected Intel(R) Corporation WiFi Link 5100 AGN, REV=0x6B
    
如果需要的内核模块已经加载且网络接口已经上线，您可以跳过下一节的内容。

根据获得的信息，在下面网址查找硬件支持：

如果您的上述列表里 _没有_ 找到您的无线网卡，那么大概率您的网卡仅在windows操作系统下受支持。对于这些设备，您可以尝试使用 [#ndiswrapper](<#ndiswrapper>)。

###  iw 与 wireless_tools 的对比

下面表格给出了 `iw` 和 `wireless_tools` 命令的对比（更多示例参阅[这里](<https://wireless.wiki.kernel.org/en/users/Documentation/iw/replace-iwconfig>)）。 

_iw_ 命令  |  _wireless_tools_ 命令  | 描述   
---|---|---  
iw dev _wlan0_ link  | iwconfig _wlan0_ | 获取连接状态。   
iw dev _wlan0_ scan  | iwlist _wlan0_ scan  | 扫描可用接入点。   
iw dev _wlan0_ set type ibss  | iwconfig _wlan0_ mode ad-hoc  | 设置操作模式为 _ad-hoc_ 。   
iw dev _wlan0_ connect _your_essid_ | iwconfig _wlan0_ essid _your_essid_ | 连接到开放网络。   
iw dev _wlan0_ connect _your_essid_ 2432  | iwconfig _wlan0_ essid _your_essid_ freq 2432M  | 连接到开放网络的特定频道。   
iw dev _wlan0_ connect _your_essid_ key 0:_your_key_ | iwconfig _wlan0_ essid _your_essid_ key _your_key_ | 用16进制密匙连接到 WEP 加密网络。   
iw dev _wlan0_ connect _your_essid_ key 0:_your_key_ | iwconfig _wlan0_ essid _your_essid_ key s:_your_key_ | 用 ASCII 密匙连接到 WEP 加密网络。   
iw dev _wlan0_ set power_save on  | iwconfig _wlan0_ power on  | 启用省电功能。   
  
## iw

**注意：**

  * 多数命令需要使用 [root权限](<../../zh-cn/%E7%94%A8%E6%88%B7%E5%92%8C%E7%94%A8%E6%88%B7%E7%BB%84.html> "用户和用户组") 执行。 使用普通用户执行某些命令,如 _iw list_ ，不会输出任何错误但并不会产生正确的输出，造成疑惑。
  * 根据您硬件或加密类型的不同，某些步骤并非必要。在连接至某个接入点及获取IP地址之前，某些网卡需要首先被 激活 与/或 扫描热点。这可能需要您自己尝试。例如，WPA/WPA2 用户可能会尝试直接从步骤 [连接到接入点](<#%E8%BF%9E%E6%8E%A5%E5%88%B0%E6%8E%A5%E5%85%A5%E7%82%B9>) 开始来激活无线网络。

本章节的示例会假定您的无线接口设备名称是 `_interface_`, 您想要连接到的WiFi接入点名称是 `_your_essid_`。请将这两个字段作相应替换。 

###  获取接口名

**提示：** 更多关于 _iw_ 工具的示例，请查看 [官方文档](<https://wireless.wiki.kernel.org/en/users/documentation/iw>) 。

获取您无线网卡的名称，运行： 
    
    $ iw dev
    
接口的名称会在 "Interface" 单词后显示。比如，常见的无线网卡设备名是 `wlan0`。 

###  检查连接状态

用下面的命令检查连接状态: 
    
    $ iw dev _interface_ link
    
###  获取统计数据

获取一些统计数据，比如上传下载的数据量，信号强度等： 
    
    $ iw dev _interface_ station dump
    
####  激活接口

**提示：** 通常不需要执行此步骤。

一些无线网卡在使用 `iw` 或 `wireless_tools`前需要先激活内核接口： 
    
    # ip link set _interface_ up
    
**注意：** 如果出现错误 `RTNETLINK answers: Operation not possible due to RF-kill`, 请确保硬件开关已经打开。参阅 [#Rfkill 说明](<#Rfkill_%E8%AF%B4%E6%98%8E>)。

如果出现错误 `RTNETLINK answers: Operation not possible due to RF-kill`, 请确保硬件开关已经打开。参阅 [#Rfkill 说明](<#Rfkill_%E8%AF%B4%E6%98%8E>)。 

要验证接口确实打开： 
    
    $ ip link show _interface_
    
    3: wlan0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc mq state DOWN mode DORMANT group default qlen 1000
        link/ether 12:34:56:78:9a:bc brd ff:ff:ff:ff:ff:ff
    
`<BROADCAST,MULTICAST,UP,LOWER_UP>` 中的`UP` 显示接口已经打开，不要和后面的 `state DOWN` 混淆。 

###  扫描接入点

查看可用的接入点： 
    
    # iw dev _interface_ scan | less
    
**注意：** 如果显示 "Interface doesn't support scanning"，可能是忘了安装固件。有时不以 root 运行 `iwlist` 也会产生这个问题。无线网络也可能被软锁定，请运行 `rfkill list all` 进行检查。

**提示：** Depending on your location, you might need to set the correct [regulatory domain](<#Respecting_the_regulatory_domain>) in order to see all available networks.

需要关注的信息: 

  * **SSID:** 网络的名称.
  * **Signal:** 用 dbm (-100 to 0) 报告的无线信号强度。数值越接近零，信号越好。观察高质量连接和低质量连接的数值差异可以了解设备的信号范围。
  * **Security:** 没有直接报告, 检查 `capability` 开头的行，如果有 `Privacy` 信息，例如 `capability: ESS Privacy ShortSlotTime (0x0411)`, 表示网络具有某种程度的保护， 
    * 如果有 `RSN` 信息，网络被 [Robust Security Network](<https://en.wikipedia.org/wiki/IEEE_802.11i-2004> "wikipedia:IEEE 802.11i-2004")(WPA2) 协议保护。
    * 如果有 `WPA` 信息，网络被 [Wi-Fi Protected Access](<https://en.wikipedia.org/wiki/Wi-Fi_Protected_Access> "wikipedia:Wi-Fi Protected Access") 协议保护。
    * 在 `RSN` 和 `WPA` 信息块中，可能看到如下信息： 
      * **Group cipher:** 数值包括 TKIP, CCMP, both, others.
      * **Pairwise ciphers:** 数值包括 TKIP, CCMP, both, others. 可能和 Group cipher 数值不同.
      * **Authentication suites:** 数值包括 PSK, 802.1x, others. 家用路由器通常可以看到 PSK (_i.e._ 密码). 在大学中，通常会链接到需要登录名和密码的 802.1x 网络。需要知道其使用的密码管理方式(例如 EAP), 封装方法 (例如 PEAP). 详情请参考[这里](</wzh/index.php?title=WPA2_Enterprise&action=edit&redlink=1> "WPA2 Enterprise（页面不存在）")和[这里](<https://en.wikipedia.org/wiki/Authentication_protocol> "wikipedia:Authentication protocol").
    * 如果没有看到 `RSN` 或 `WPA`，但是看到了 `Privacy`, 表示使用的是 WEP.

###  运行模式

设置无线网卡的操作模式，如果连接到[漫游网络](<../../zh-cn/Ad-hoc_networking.html> "Ad-hoc networking")，需要设置操作模式为 **ibss**
    
    # iw dev _interface_ set type ibss
    
**注意：** 有些网卡需要先关闭无线接口(`ip link set _interface_ down`)才能修改模式。

###  连接到接入点

根据加密方式不同，需要使用密码将无线设备关联到接入点。 

假设要使用的接入点 ESSID 为 `MyEssid`: 

####  无加密
    
    # iw dev _interface_ connect "_your_essid_ "
    
#### WEP

使用十六进制或 ASCII 密码(格式是自动识别出来的，因为 WEP 密码长度是固定的): 
    
    # iw dev _interface_ connect "_your_essid_ " key 0:_your_key_
    
使用十六进制或 ASCII 密码，第三个是默认 (从0计数，共四个): 
    
    # iw dev _interface_ connect "_your_essid_ " key d:2:_your_key_
    
####  其它

  *     * _iw_ 只能处理 WEP。要使用其他加密方案登录, 请查看下文中的 [#认证](<#%E8%AE%A4%E8%AF%81>)章节。

无论您使用什么认证方法，都可以使用以下命令来确认连接是否成功： 
    
    # iw dev _interface_ link
    
##  认证

###  WPA2 个人模式

WPA2 个人（模式）, 又名 WPA2-PSK, 是一种 受保护Wi-Fi访问（[Wi-Fi存取保护](<https://zh.wikipedia.org/wiki/WPA> "wiki-zh:WPA")）的模式。 

您可以通过 [wpa_supplicant](<../../zh-cn/Wpa_supplicant.html> "Wpa supplicant") 或 [iwd](<../../zh-cn/Iwd.html> "Iwd") 来连接, 或者直接使用[网络管理器](<../../zh-cn/Network_manager.html> "Network manager"). 如果您只进行了认证，那么要获取功能齐全的连接，您仍然需要[进行手动连接](<../../zh-cn/Network_configuration.html#Static_IP_address> "Network configuration")或使用 [DHCP](<../../zh-cn/%E7%BD%91%E7%BB%9C%E9%85%8D%E7%BD%AE.html#DHCP> "网络配置") 客户端。 

###  WPA2 企业模式

_WPA2 企业模式_ 也是一种 受保护Wi-Fi访问（[Wi-Fi存取保护](<https://zh.wikipedia.org/wiki/WPA> "wiki-zh:WPA")）的模式，提供了比 WPA2 个人版更好的安全性和密钥管理，并支持其他的企业型功能，例如 [VLAN](<../../zh-cn/VLAN.html> "VLAN") 和 NAP。然而，与个人模式不同，此种加密需要额外的 [RADIUS](<https://en.wikipedia.org/wiki/RADIUS> "wikipedia:RADIUS") 认证服务器来处理用户的认证。个人模式不需要除无线路由器/接入点以外的设备，对所有用户也只要求一个密码或密码短语。 

企业模式允许用户使用用户名和密码或者数字证书来登录 Wi-Fi。因为每个用户都拥有一个动态唯一的加密密码，因此企业模式可以防止通过无限网络的用户探听，并提高加密强度。 

本章节描述如何使用[网络管理器](<../../zh-cn/%E7%BD%91%E7%BB%9C%E9%85%8D%E7%BD%AE.html#%E7%BD%91%E7%BB%9C%E7%AE%A1%E7%90%86%E5%99%A8> "网络配置")来连接到使用 WPA2 企业模式的无线接入点。 

查看[软件接入点#RADIUS](<../../zh-cn/Software_access_point.html#RADIUS> "Software access point") 来获取关于 _设置接入点自身_ 的相关信息。 

**注意：** 企业级加密模式需要更复杂的客户端配置，个人模式则只需要在出现提示时输入密码。客户端可能需要安装服务器的 签名证书(CA certificate)，如果使用 EAP-TLS, 则另需要为每个用户安装证书，再手动配置无线安全和 802.1X 认证设置。

协议间的对照可以参照此[表格](<http://deployingradius.com/documents/protocols/compatibility.html>)。 

**警告：** 您可以让客户端在使用WPA2企业级时不检查服务器的授权证书。然而，您在任何情况下都该尽最大努力来配置证书，因为如果不验证接入点，连接可能遭受中间人攻击。虽然连接握手可能是加密的，但最常见的密码传输配置会将密码用明文或易于破解的[MS-CHAPv2协议](<#MS-CHAPv2>)协议进行传输。因此，客户端可能会将密码发送到代理连接的恶意接入点。

##  提示和技巧

###  遵循监管区域要求

[监管区域](<https://en.wikipedia.org/wiki/IEEE_802.11#Regulatory_domains_and_legal_compliance> "wikipedia:IEEE 802.11")用于配置无线驱动，确保对无线硬件的使用符合由 FCC、ETSI 及其他组织制定的当地法规。监管区域使用[ISO 3166-1二位字母国家代码](<https://en.wikipedia.org/wiki/zh:ISO_3166-1%E4%BA%8C%E4%BD%8D%E5%AD%97%E6%AF%8D%E4%BB%A3%E7%A0%81> "wikipedia:zh:ISO 3166-1二位字母代码")。例如，美国的监管区域代码是“US”，中国则是“CN”，等等。 

监管区域影响无线信道的可用性。在2.4GHz波段，美国允许使用1-11信道，日本是1-14，世界上多数其它地区则是1-13。在5GHz波段，允许使用信道的规则要复杂得多。这两种情况的更详细信息见[这一WLAN信道列表](<https://en.wikipedia.org/wiki/zh:WLAN%E4%BF%A1%E9%81%93%E5%88%97%E8%A1%A8> "wikipedia:zh:WLAN信道列表")。 

监管区域也影响对来自无线设备的[等效各向同性辐射功率（EIRP）](<https://en.wikipedia.org/wiki/Equivalent_isotropically_radiated_power> "wikipedia:Equivalent isotropically radiated power")所作的限制。这衍生自发射功率（tx power）并以[dBm/mBm（1dBm=100mBm）或mW（对数尺度）](<https://en.wikipedia.org/wiki/zh:%E5%88%86%E8%B4%9D%E6%AF%AB%E7%93%A6> "wikipedia:zh:分贝毫瓦")衡量。在2.4GHz波段，美国和加拿大最大限制30dBm，欧洲大部分是20dBm，世界上其他地区则是20dB-30dBm。在5GHz波段，最大值通常更低。更详细的信息请参考 [wireless-regdb](<https://git.kernel.org/cgit/linux/kernel/git/linville/wireless-regdb.git/tree/db.txt>)（EIRP dBm 值在每行第二对括号里）。 

错误设置监管区域可能会有用——例如，在其他信道拥挤时可以使用未用信道，或者提升发射功率以扩大传输范围。然而**不推荐这种做法** ，因为可能会违反当地法规以及造成与其他无线电设备的干扰。 

要配置监管区域，安装 [crda](<https://archlinux.org/packages/?name=crda>)包[[损坏的链接](<../../zh-cn/Help:%E6%93%8D%E4%BD%9C%E6%B5%81%E7%A8%8B.html#%E4%BF%AE%E5%A4%8D%E6%8D%9F%E5%9D%8F%E7%9A%84%E5%8C%85%E9%93%BE%E6%8E%A5> "Help:操作流程")：replaced by [wireless-regdb](<https://archlinux.org/packages/?name=wireless-regdb>)包] 并重启（以重新加载 `cfg80211` 模块及所有相关驱动）。检查启动日志以确认 `cfg80211` 正在调用CRDA： 
    
    # dmesg | grep cfg80211
    
以下命令可将当前监管区域设为美国： 
    
    # iw reg set US
    
用以下命令查询： 
    
    $ iw reg get
    
**注意：**

  * 你的设备国家代码可能为“00”，它是包含了通用设置的“世界监管区域”。如果无法解除此设置，则CRDA可能配置有误。
  * 根据 [CRDA 的 README](<https://git.kernel.org/pub/scm/linux/kernel/git/mcgrof/crda.git/tree/README>)，从内核版本4.15开始就不再需要它，因为内核会自动从固件加载监管数据库。然而[也有说法称](<https://bugs.gentoo.org/462032#c17>)这一加载操作需要特定的内核设置。

然而设置监管区域可能不会改变您的设置。有些设备在固件/EEPROM中设置了一个监管区域，为设备规定了限制，这意味着通过软件设置监管区域[只能增加限制](<https://wiki.openwrt.org/doc/howto/wireless.utilities#iw>)而不能减少。例如，一个 CN（中国）设备可以用软件设置为 US（美国）监管区域，但由于中国的 EIRP 最大值为20dBm，设备将无法以美国的最大值30dBm进行发射。 

例如，查看一个 Atheros 设备的固件中是否设置了监管区域： 
    
    # dmesg | grep ath:
    
对于其他芯片组，搜索“EEPROM”、“regdomain”或设备驱动的名字可能会有用。 

要查看监管区域更改是否成功，以及查询可用信道数和允许发射功率： 
    
    $ iw list | grep -A 15 Frequencies:
    
监管区域的持久设置可以通过编辑 `/etc/conf.d/wireless-regdom` 并解除相应区域的注释实现。 `wpa_supplicant` 也可以在 `/etc/wpa_supplicant/wpa_supplicant.conf` 中的 `country=` 行应用监管区域。 

也可以通过添加[模块参数](</wzh/index.php?title=Kernel_module_\(%E7%AE%80%E4%BD%93%E4%B8%AD%E6%96%87\)&action=edit&redlink=1> "Kernel module \(简体中文\)（页面不存在）")（如 `options cfg80211 ieee80211_regdom=EU`），配置 [cfg80211](<https://wireless.wiki.kernel.org/en/developers/Documentation/cfg80211>) 内核模块，从而使用指定的监管区域。然而这是一种[老旧的监管实现](<https://wireless.wiki.kernel.org/en/developers/Regulatory#the_ieee80211_regdom_module_parameter>)。 

更多信息请阅读 [wireless.kernel.org 监管文档](<https://wireless.wiki.kernel.org/en/developers/Regulatory/>)。 

###  Rfkill 说明

许多笔记本都有一个硬件按钮或开关用于关闭无线网卡，然而无线网卡也可以通过内核来屏蔽。这一任务可由 _rfkill_ 处理。显示当前状态： 
    
    $ rfkill
    
    ID TYPE      DEVICE      SOFT      HARD
     0 bluetooth hci0   unblocked unblocked
     1 wlan      phy0   unblocked unblocked
    
如果网卡处于 _硬件屏蔽（hard-blocked）_ 状态，请使用硬件按钮或开关来开启它。 如果无线网卡并没有被 _硬件屏蔽_ 但处于 _软件屏蔽（soft-blocked）_ ，请使用以下命令： 
    
    # rfkill unblock wifi
    
**注意：** 按下硬件按钮可能会让无线网卡从硬件屏蔽，软件未屏蔽状态变为硬件未屏蔽，软件屏蔽状态。 可以通过调整 `rfkill` [内核模块](</wzh/index.php?title=Kernel_module_\(%E7%AE%80%E4%BD%93%E4%B8%AD%E6%96%87\)&action=edit&redlink=1> "Kernel module \(简体中文\)（页面不存在）")的一些选项改变这一现象。

切换无线网卡的硬件按钮是厂商专用的[内核模块](</wzh/index.php?title=Kernel_module_\(%E7%AE%80%E4%BD%93%E4%B8%AD%E6%96%87\)&action=edit&redlink=1> "Kernel module \(简体中文\)（页面不存在）")处理的，经常是 [WMI](<https://lwn.net/Articles/391230/>) 模块。特别是非常新的硬件型号，可能还没有得到最新稳定内核的完全支持。这种情况下，在内核 bug 跟踪系统搜索信息，并将型号报告给对应厂商内核模块的维护者（如果还没有报告的话）经常会有帮助。 

参见 <https://askubuntu.com/questions/62166/siocsifflags-operation-not-possible-due-to-rf-kill> 。u

###  设置 RTS 和静态帧分段阈值

无线网卡默认禁用了 RTS 和 分段机制 。这两种方法都以牺牲带宽来换取吞吐量 (i.e. 以速度为代价提高可靠性)。在由于无线干扰和有多个相邻接入点，从而导致连接超时或失败的环境中，这两个设置非常有用。 

数据包分段通过将超过阈值的包分割，从而提高吞吐量。最大值（2346）实际上禁用了分段，因为没有任何包可以超过这个值，最小值（256）最大化了吞吐量，但带宽成本很会大。 
    
    # iw phy0 set frag 512
    
[RTS](<https://en.wikipedia.org/wiki/IEEE_802.11_RTS/CTS> "wikipedia:IEEE 802.11 RTS/CTS") 通过在与接入点传输超过RTS阈值的分组之前来进行握手，从而提高吞吐量。最大值（2347）实际上禁用了RTS，没有分组可以超过这个值。最小值（0）意味着所有分组都启用RTS，但这很极端。 
    
    # iw phy0 set rts 500
    
**注意：**`phy0` 是用`iw phy` 命令 列举出的无线网卡的名称

###  随时可能断开连接

####  Cause #1

如果你在[日志](<../../zh-cn/Systemd/Journal.html> "Journal")里看到：`wlan0: deauthenticating from MAC by local choice (reason=3)` 并且 Wi-Fi 断开连接, 很可能是你的无线网卡的电源节能设置太激进。尝试禁用无线网卡的[省电功能](<../../zh-cn/Power_management.html#Network_interfaces> "Power management")。 

如果你的网卡不支持 启用/禁用 节能模式, 看看 BIOS 的电源管理选项。 比如，在 联想 W520 的BIOS中禁用PCI-Express电源管理就解决了这个问题。 

####  Cause #2

如果你经常断连，并且在[日志](<../../zh-cn/Systemd/Journal.html> "Journal")看到如下信息： 

`ieee80211 phy0: wlan0: No probe response from AP xx:xx:xx:xx:xx:xx after 500ms, disconnecting`

在你路由器设置面板里把频道带宽改成 `20MHz` 。 

####  Cause #3

有些笔记本有硬件 rfkill 开关（如Thinkpad X200系列），因为磨损或有问题，开关（或与主板的连接）可能会随时间的推移而变松，不小心触摸开关或移动电脑时就可能出现断连。这问题没有软件解决方案，除非开关是电子式的，BIOS提供了禁用开关的选项。如果开关是机械式的（大部分机型都是），就有很多解决方案，其中大多数都是要禁用开关：在主板/无线网卡上的接触点进行焊接，用胶水封堵或阻挡，使用螺丝来拧紧开关，或者直接拆除开关。 

####  Cause #4

导致随时断连或完全无法连接的其他原因，比如路由器质量差，配置不完整，其他无线设备干扰，或信号质量差。 

为了排错, 首先要靠近路由器，并且在不需要身份验证的情况下进行连接 

如果有效，那重新启用 WPA/WPA2 ，但是选择固定/有限的路由器配置，比如： 

  * 如果路由器比你用的无线设备客户端老的多，把路由器设置为单一无线模式进行测试。
  * 禁用混合验证模式 ，(e.g. 只启用 WPA2 with AES, o如果是老路由器，则是TKIP)
  * 使用固定频道而不是自动频道 (因为附近的路由器可能老旧而且存在干扰)
  * 禁用 [WPS](<https://en.wikipedia.org/wiki/Wi-Fi_Protected_Setup> "wikipedia:Wi-Fi Protected Setup")
  * 把路由器的 5 GHz 频道切换为 [non-DFS (Dynamic Frequency Selection) 频道](<https://en.wikipedia.org/wiki/List_of_WLAN_channels#5_GHz_.28802.11a.2Fh.2Fj.2Fn.2Fac.2Fax.29> "wikipedia:List of WLAN channels"). 在这种频道的连接[突然被丢弃或者切换](<https://wifinigel.blogspot.com/2018/05/the-5ghz-problem-for-wi-fi-networks-dfs.html>)，这是因为附近天气雷达的干扰。
  * 只使用2.4 GHz ，而不是自动在 5 GHz 和2.4 GHz 切换。 (后者吞吐量低，但是远距离连接更稳定)
  * 禁用 `40MHz` 频道带宽 (吞吐量低，但冲突更少) ，用`cfg80211.cfg80211_disable_40mhz_24ghz=1`
  * 如果路由器有 Qos功能, 检查设置的完整性。(e.g. Wi-Fi 多媒体 (WMM) 是 QoS 流控制中可选的功能， 损坏的路由器固件可能会表明他已经存在，即使没有启用这个功能)

####  Cause #5

对于一些无线网卡适配器 (e.g. Qualcomm Atheros AR9485), 随时断连可能会伴随一个 DMA 报错： 
    
    # journalctl -xb
    
    ath: phy0: DMA failed to stop in 10 ms AR_CR=0x00000024 AR_DIAG_SW=0x02000020 DMADBG_7=0x0000a400
    wlp1s0: authenticate with 56:e7:ee:7b:55:bc
    wlp1s0: send auth to 56:e7:ee:7b:55:bc (try 1/3)
    wlp1s0: send auth to 56:e7:ee:7b:55:bc (try 2/3)
    wlp1s0: send auth to 56:e7:ee:7b:55:bc (try 3/3)
    wlp1s0: authentication with 56:e7:ee:7b:55:bc timed out

可行的方案是禁用 [Intel IOMMU driver (DMA)](<https://docs.kernel.org/x86/iommu.html>), 把`intel_iommu=off` 加入 [kernel parameters](<../../zh-cn/Kernel_parameters.html> "Kernel parameters") [[2]](<https://bbs.archlinux.org/viewtopic.php?pid=1907446#p1907446>). 

**注意：** Intel IOMMU 驱动对于一些高级虚拟机功能是必需的，如PCI直通。

####  Cause #6

如果你用的是带 `iwlwifi` 或 `iwlmvm` 的设备进行无线连接，使用电池电源的时候，无线网卡似乎消失了，（可能是重启或从休眠中恢复后），可以通过 iwlmvm 里的节能模式设置修复。 

创建 `/etc/modprobe.d/iwlmvm.conf` ，再添加以下行： 
    
    /etc/modprobe.d/iwlmvm.conf
    
    options iwlmvm power_scheme=1

`power_scheme` 为 1 表明把 iwlmvm 设置为 _总是活跃_ 。可选项如下： 

值 | 描述   
---|---  
1 | 总是活跃   
2 | 平衡   
3 | 节能   
  
这个修复方法是在 [[3]](<https://forums.debian.net/viewtopic.php?t=121696#p576208>)被找到的。 

####  Cause #7

若设备长期不使用 (e.g. 文件服务器), 可能会因为节能模式导致无法连接，节能模式会阻止流量传入和建立连接。尝试为接口禁用节能模式: 
    
    # iw dev _interface_ set power_save off
    
可以创建udev规则，让开机时自动执行它。参阅 [Power management#Network interfaces](<../../zh-cn/Power_management.html#Network_interfaces> "Power management"). 

####  Cause #8

如果你连接 Mesh 网络时 (e.g., WiFi6) ，会偶尔中断连接，并且你看到了如下日志信息： 
    
    # journalctl -b
    
    kernel: wlan0: disconnect from AP aa:bb:cc:dd:ee:ff for new auth to 11:22:33:44:55:66
    
你遇到的是漫游问题，可采取以下措施： 

  * 如果不需要漫游，在 NetworkManager中锁定BSSID (如上面的 `aa:bb:cc:dd:ee:ff` ) (参阅 [NetworkManager#Regular network disconnects, latency and lost packets (WiFi)](<../../zh-cn/NetworkManager.html#Regular_network_disconnects,_latency_and_lost_packets_\(WiFi\)> "NetworkManager")).
  * 在 [Wpa_supplicant#Roaming](<../../zh-cn/Wpa_supplicant.html#Roaming> "Wpa supplicant") 中调整 `bgscan`

C 

####  Cause #9

如果你使用Intel WiFi 7 BE200网卡，并且WiFi在启动时完全正常，但暂停/恢复后，找不到任何网络。 

问题可能在于，在新版 Linux 内核中，驱动程序或设备本身会声明其具备将设备设置为低功耗 D3cold 睡眠状态的功能，而这种状态本身存在问题。 解决方案是指示 udev 配置该卡，使其不使用该状态。 
    
    $ /etc/udev/rules.d/99-disable-d3cold-be200.rules
    
    $ ACTION=="add", SUBSYSTEM=="pci", ATTR{vendor}=="0x8086", ATTR{device}=="0x272b", ATTR{d3cold_allowed}="0"

###  Wi-Fi 由于错误的信道设置而变得不可见

若电脑 Wi-Fi 信道不符合用户所处国家之规定，部分Wi-Fi即使处于信号覆盖范围内，也可能因信道默认不被允许而不可见。参阅 [#Respecting the regulatory domain](<#Respecting_the_regulatory_domain>).

#####  节能/省电模式

虽然 Linux 无线工具表明对于比 AR9280 更新的 Atheros ath9k 单芯片设备已经启用了动态电源节能功能，虽然 [Linux Wireless](<https://wireless.wiki.kernel.org/en/users/Documentation/dynamic-power-save>) 指出比 AR9280 更新的 Atheros ath9k 单芯片设备已启用动态节能功能，但一些设备仍 (例如 AR9285) [powertop](<https://archlinux.org/packages/?name=powertop>)包 被指出省电被禁用，这种情况需手动开启。

#### iwlwifi

[iwlwifi](<https://wireless.wiki.kernel.org/en/users/drivers/iwlwifi>) 是适用于 Intel 当前网卡的无线驱动，例如 5100AGN、5300AGN、和 5350AGN。[查看所有受支持的文件列表](<https://wireless.wiki.kernel.org/en/users/drivers/iwlwifi#supported_devices>)。该固件已包含在 [linux-firmware](<https://archlinux.org/packages/?name=linux-firmware>)包 里。[linux-firmware-iwlwifi-git](<https://aur.archlinux.org/packages/linux-firmware-iwlwifi-git/>)AUR 可能包含更新的驱动。 

若联网存在问题或连接很弱，尝试启用 802.11n，也可能需要启用软件加密： 
    
    /etc/modprobe.d/iwlwifi.conf
    
    options iwlwifi 11n_disable=1 swcrypto=1

在 802.11n 模式下网络上行速度很慢，比如只有 20Mbps，尝试启用 TX AMPDU： 
    
    /etc/modprobe.d/iwlwifi.conf
    
    options iwlwifi 11n_disable=8

若上行速度很慢，可以禁用无线网卡的[省电模式](<https://wiki.archlinux.org/title/Power_saving#Network_interfaces> "en:Power saving")。 

如果你有 802.11ax (WiFi 6) 接入点并且也有连接不稳定等问题，可参阅 [Intel Article 54799](<https://www.intel.com/content/www/us/en/support/articles/000054799/network-and-i-o/wireless.html>)。 

**注意：** 用 `11n_disable=0` 参数将会阻止 802.11ac，只能使用较慢的协议，（5GHz 频段用 802.11a，2.4 GHz 频段用802.11b/g）。

#####  蓝牙共存

若无法连接蓝牙耳机或无法保持良好的数据接受速度，请禁用蓝牙共存，参考 [[10]](<https://wireless.docs.kernel.org/en/latest/en/users/drivers/iwlwifi.html#wi-fi-bluetooth-coexistence>)： 
    
    /etc/modprobe.d/iwlwifi.conf
    
    options iwlwifi bt_coex_active=0

**注意：** 从内核版本 5.8 起, 对于由 `iwlmvm` 内核模块处理的硬件， `bt_coex_active` 和 `sw_crypto` 模块选项已被禁用，但对于由 `iwldvm` 模块处理的老硬件，这些选项依旧起作用。

#####  固件问题

你可能会遇到驱动程序输出堆栈跟踪与报错，这会导致有点卡顿。 
    
    # dmesg
    
    Microcode SW error detected.  Restarting 0x2000000.

你也可能遇到其他各种各样的问题（例如[连接 5GHz 频段时的问题，随时断连，恢复后无法连接](<https://www.reddit.com/r/archlinux/comments/x0v5jj/rant_intel_wifi_firmwares_are_utter_garbage/>)）。 

要想确认是否是固件的原因，请[降级](<../../zh-cn/%E9%99%8D%E7%BA%A7%E8%BD%AF%E4%BB%B6%E5%8C%85.html> "降级软件包") [linux-firmware](<https://archlinux.org/packages/?name=linux-firmware>)包 包。 

如果确定是固件的问题, 移除有 bug 的固件文件以确保可以加载旧版本固件 (要更新 [linux-firmware](<https://archlinux.org/packages/?name=linux-firmware>)包 包，因为它不仅仅提供 Intel 无线网卡的固件更新): 
    
    # for i in {64..73} ; do mv /usr/lib/firmware/iwlwifi-ty-a0-gf-a0-$i.ucode.xz /usr/lib/firmware/iwlwifi-ty-a0-gf-a0-$i.ucode.xz.bak ; done
    
为了避免每次更新后都需要手动重复这些步骤, 可以在 `pacman.conf` 里用 `NoExtract` 数组搭配通配符以此禁止它们的安装。详见 [pacman#在安装时跳过文件](<../../zh-cn/Pacman.html#%E5%9C%A8%E5%AE%89%E8%A3%85%E6%97%B6%E8%B7%B3%E8%BF%87%E6%96%87%E4%BB%B6> "Pacman")。 

#####  从 Windows 重启后无法检测到 WiFi 适配器

如果从 Windows 重启到 Linux 后，检测不到 WiFi 适配器，可能是由于 Windows 默认启用的快速启动功能。尝试[禁用快速启动](<../../zh-cn/Dual_boot_with_Windows.html#Windows_settings> "Dual boot with Windows")。[iwlwifi 内核驱动维基页面有关于此问题的条目](<https://wireless.wiki.kernel.org/en/users/drivers/iwlwifi#about_dual-boot_with_windows_and_fast-boot_enabled>)。 

#####  禁用 LED 闪烁

**注意：** 适用于 `iwlegacy` 和 `iwlwifi` 驱动

默认设置中 LED 闪烁是开启的，有些人不喜欢，可以用 [systemd-tmpfiles](<../../zh-cn/Systemd.html#%E4%B8%B4%E6%97%B6%E6%96%87%E4%BB%B6> "Systemd") 禁止闪烁： 
    
    /etc/tmpfiles.d/phy0-led.conf
    
    w /sys/class/leds/phy0-led/trigger - - - - phy0radio
    
运行 `systemd-tmpfiles --create phy0-led.conf` 使更改生效，或重启 

查看此 LED 所有可选的触发值： 
    
    # cat /sys/class/leds/phy0-led/trigger
    
**提示：** 如果你没有 `/sys/class/leds/phy0-led`，尝试使用 `led_mode="1"` [模块参数](<../../zh-cn/%E5%86%85%E6%A0%B8%E6%A8%A1%E5%9D%97.html#%E9%85%8D%E7%BD%AE%E6%A8%A1%E5%9D%97%E5%8F%82%E6%95%B0> "内核模块")。它应当同时适用于 `iwlwifi` 和 `iwlegacy` 驱动。
    
    # cat /sys/class/leds/phy0-led/trigger

### ndiswrapper

Ndiswrapper并不是一个真正的驱动,但是如果你无法找到适合你的无线网卡驱动, 它就派上用场了.有的时候, 它是非常有用的.为了使用Ndiswrapper, 你需要 Windows 驱动中的 *.inf 文件(*.sys 文件应该和 *.info 在同一个目录中)。如果你需要从 `*.exe` 文件解压缩，你可以使用 [cabextract](<https://archlinux.org/packages/?name=cabextract>)包。 

**警告：** 确保使用合适架构(也就是 32/64 位)的驱动。

下面是安装 ndiswrapper 的几个步骤: 

1\. 安装 [ndiswrapper-dkms](<https://archlinux.org/packages/?name=ndiswrapper-dkms>)包 2\. 安装驱动到 `/etc/ndiswrapper/*`
    
    ndiswrapper -i filename.inf
    
3\. 列出所有的安装的驱动 
    
    ndiswrapper -l
    
4\. 配置文件写到 `/etc/modprobe.d/ndiswrapper.conf`
    
    ndiswrapper -m
    depmod -a
    
然后依照[内核模块](<../../zh-cn/%E5%86%85%E6%A0%B8%E6%A8%A1%E5%9D%97.html> "内核模块")一文设置启动时加载这个模块。 
    
    modprobe ndiswrapper
    iwconfig
    
如果正常的话, 你应该可以看到wlan0接口了。如果有问题的话, 你可以阅读： [Ndiswrapper installation wiki](<http://ndiswrapper.sourceforge.net/joomla/index.php?/component/option,com_openwiki/Itemid,33/id,installation/>)[[失效链接](<https://zh.wikipedia.org/wiki/Wikipedia:%E5%A4%B1%E6%95%88%E9%93%BE%E6%8E%A5> "zhwp:Wikipedia:失效链接") 2020-08-04 ⓘ]. [ndiswrapper howto](<https://sourceforge.net/p/ndiswrapper/ndiswrapper/HowTos/>) 和 [ndiswrapper FAQ](<https://sourceforge.net/p/ndiswrapper/ndiswrapper/FAQ/>).
