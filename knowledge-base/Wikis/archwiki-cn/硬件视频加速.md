[硬件视频加速](<https://en.wikipedia.org/wiki/Graphics_processing_unit#GPU_accelerated_video_decoding_and_encoding> "wikipedia:Graphics processing unit")让显卡进行视频的解码和编码，以减少CPU的负载并节省电力。 

Linux 中有多种方式可以实现视频加速: 

  * [ 视频加速API (Video Acceleration API, 缩写为VA-API)](<https://en.wikipedia.org/wiki/Video_Acceleration_API> "wikipedia:Video Acceleration API") 是一套 Intel 提供的视频硬件编解码的开源库和标准。
  * [ Unix视频解码及演示 API （Video Decode and Presentation API for Unix，缩写为VDPAU）](<https://en.wikipedia.org/wiki/VDPAU> "wikipedia:VDPAU") (VDPAU) 是一套 Nvidia 提供的把部分视频解码和后期处理工作转移到显卡的开源库和 API。
  * [Advanced Media Framework SDK](<https://gpuopen.com/advanced-media-framework/>) (AMF) 是 AMD 开发的 GPU 开源框架，可以用 [AMDGPU PRO](<../zh-cn/AMDGPU_PRO.html> "AMDGPU PRO") 技术栈实现最优的处理。
  * [NVDEC](<https://en.wikipedia.org/wiki/Nvidia_NVDEC> "wikipedia:Nvidia NVDEC")/[NVENC](<https://en.wikipedia.org/wiki/Nvidia_NVENC> "wikipedia:Nvidia NVENC") NVIDIA 的 费米、开普勒、Maxwell 和 Pascal 系列显卡所使用的专有 API。
  * [Vulkan Video](<https://www.khronos.org/blog/an-introduction-to-vulkan-video>) is an extension of the [Vulkan](<../zh-cn/Vulkan.html> "Vulkan") graphics API designed to support hardware-accelerated video encoding and decoding.

大致的总览对比和应用程序支持，参见页面下方的对照表。 

##  安装

### Intel

#### VA-API

[Intel graphics](<../zh-cn/Intel_graphics.html> "Intel graphics") 开源驱动支持 VA-API: 

  * [Broadwell](<https://github.com/intel/media-driver/#supported-platforms>) [(~2014)](<https://en.wikipedia.org/wiki/Template:Intel_processor_roadmap> "wikipedia:Template:Intel processor roadmap") 及之后的 HD Graphics 系列（例如英特尔 Arc 系列显卡），请使用 [intel-media-driver](<https://archlinux.org/packages/?name=intel-media-driver>)包 。
  * GMA 4500 (2008) 到 [Coffee Lake](<https://en.wikipedia.org/wiki/Coffee_Lake> "wikipedia:Coffee Lake") (2017) 的 GPU, 请使用 [libva-intel-driver](<https://archlinux.org/packages/?name=libva-intel-driver>)包 。
  * [Haswell Refresh](<https://en.wikipedia.org/wiki/Haswell_\(microarchitecture\)#Haswell_Refresh> "wikipedia:Haswell \(microarchitecture\)") 到 Skylake 的 VP9 解码，以及 Broadwell 到 Skylake 的混合 VP8 编码，请使用 [intel-hybrid-codec-driver-git](<https://aur.archlinux.org/packages/intel-hybrid-codec-driver-git/>)AUR.

**注意：** Skylake 及之后的硬件还需要 [linux-firmware](<https://archlinux.org/packages/?name=linux-firmware>)包.

参阅 [VAAPI 支持的硬件和功能](<https://www.intel.com/content/www/us/en/developer/articles/technical/linuxmedia-vaapi.html>)。 

#### Vulkan

ANV 开源 [vulkan](<../zh-cn/Vulkan.html> "Vulkan") 驱动通过 [vulkan-intel](<https://archlinux.org/packages/?name=vulkan-intel>)包 提供 Vulkan 视频支持。 

**注意：** 启用 Vulkan 视频支持需要额外配置，参见 [#Configuring Vulkan Video](<#Configuring_Vulkan_Video>)

####  Intel 视频处理库 (Intel VPL)

对于 [Intel VPL](<https://github.com/intel/libvpl>) ，请[安装](<../zh-cn/Help:%E9%98%85%E8%AF%BB.html#%E5%AE%89%E8%A3%85%E8%BD%AF%E4%BB%B6%E5%8C%85> "安装")基础库 [libvpl](<https://archlinux.org/packages/?name=libvpl>)包 ，并确保安装下列之一的运行时环境： 

  * [vpl-gpu-rt](<https://archlinux.org/packages/?name=vpl-gpu-rt>)包 ：为 Tiger Lake 和更新的GPU提供支持

  * [intel-media-sdk](<https://archlinux.org/packages/?name=intel-media-sdk>)包 (已终止后续支持)：为旧 Intel GPU 提供支持

### NVIDIA

[Nouveau](<../zh-cn/Nouveau.html> "Nouveau") 开源驱动支持 VA-API 和 VDPAU: 

  * GeForce 8 到 GeForce GTX 750 的硬件，请使用 [mesa](<https://archlinux.org/packages/?name=mesa>)包
  * [还需要](<https://nouveau.freedesktop.org/wiki/VideoAcceleration/#firmware>) [nouveau-fw](<https://aur.archlinux.org/packages/nouveau-fw/>)AUR 固件包（目前是从NVIDIA二进制驱动解压出来的）

[NVIDIA](<../zh-cn/NVIDIA.html> "NVIDIA") 专有驱动通过 [nvidia-utils](<https://archlinux.org/packages/?name=nvidia-utils>)包 支持以下硬件: 

  * [GeForce 8 系列](<https://en.wikipedia.org/wiki/GeForce_8_series> "wikipedia:GeForce 8 series")和新 GPU 上的 VDPAU；

  * [Fermi](<https://en.wikipedia.org/wiki/Fermi_\(microarchitecture\)> "wikipedia:Fermi \(microarchitecture\)") 和更新的GPU [[1]](<https://developer.download.nvidia.com/assets/cuda/files/NVIDIA_Video_Decoder.pdf>) 上的NVDEC;

  * [Kepler](<https://en.wikipedia.org/wiki/Kepler_\(microarchitecture\)> "wikipedia:Kepler \(microarchitecture\)") 和更新的GPU上的 NVENC ；

  * [Pascal](<https://en.wikipedia.org/wiki/Pascal_\(microarchitecture\)> "wikipedia:Pascal \(microarchitecture\)") 和更新的GPU [[2]](<https://developer.nvidia.com/vulkan/video/get-started>) 上的 Vulkan 视频。

###  AMD/ATI

[AMDGPU](<../zh-cn/AMDGPU.html> "AMDGPU") 和 [ATI](<../zh-cn/ATI.html> "ATI") 开源驱动通过 [mesa](<https://archlinux.org/packages/?name=mesa>)包 支持 VA-API 和 VDPAU: 

  * VA-API on Radeon HD 2000 and newer GPUs.

  * VDPAU on Radeon R600 and newer GPUs.

RADV 开源 [vulkan](<../zh-cn/Vulkan.html> "Vulkan") 驱动通过 [vulkan-radeon](<https://archlinux.org/packages/?name=vulkan-radeon>)包 提供支持。 

[AMDGPU PRO](<../zh-cn/AMDGPU_PRO.html> "AMDGPU PRO") 专有驱动是基于 AMDGPU 驱动的，所以支持 VA-API、VDPAU 和 AMF。 

  * [Fiji](<https://en.wikipedia.org/wiki/Radeon_Rx_300_series> "wikipedia:Radeon Rx 300 series") 和更新的 GPUs 通过 [amf-amdgpu-pro](<https://aur.archlinux.org/packages/amf-amdgpu-pro/>)AUR 支持 AMF。

**注意：**

  * 你也许需要强制你的应用程序使用 [AMDGPU PRO](<../zh-cn/AMDGPU_PRO.html> "AMDGPU PRO") [Vulkan](<../zh-cn/Vulkan.html> "Vulkan") 驱动。
  * [HEVC](<https://en.wikipedia.org/wiki/High_Efficiency_Video_Coding> "wikipedia:High Efficiency Video Coding") 可能在 [Navi](<https://en.wikipedia.org/wiki/Radeon_RX_5000_series> "wikipedia:Radeon RX 5000 series") 之前的 GPU 上无法使用。

##  验证

你的系统也许不需要任何配置就可以完美地开箱即用。因此，建议阅读本节，以确认是否确实如此。 

**提示：**

  * 有命令行支持的 [mpv](<../zh-cn/Mpv.html#Hardware_video_acceleration> "Mpv") 是测试硬件加速的绝佳工具。查看 `mpv --hwdec=auto _video_filename_` 的日志并参见 [hwdec](<https://mpv.io/manual/stable/#options-hwdec>) 以获得更多信息。
  * 对于 Intel GPU ，例如使用 [intel-gpu-tools](<https://archlinux.org/packages/?name=intel-gpu-tools>)包 并以 root 运行 `intel_gpu_top` 以监控视频播放期间的 GPU 活动。若 “Video” 栏超过 0%，则表示正在使用 GPU 进行视频解码/编码。（如果你使用的是 [新的 Xe 驱动](<../zh-cn/Intel_Graphics.html#Testing_the_new_experimental_Xe_driver> "Intel Graphics") 来运行 `intel_gpu_top` [会抛出错误](<https://gitlab.freedesktop.org/drm/igt-gpu-tools/-/issues/174>)，所以检查并运行 `gputop`，若 [vcs 字段](<https://dri.freedesktop.org/docs/drm/gpu/i915.html#intel-gpu-basics>) 超过 0%，则表示正在使用 GPU 进行视频解码/编码。）
  * 对于 AMD，Intel 和 Nvidia ，在播放视频时使用 [nvtop](<https://archlinux.org/packages/?name=nvtop>)包 来查看(在顶部几行) "DEC"（解码器）的利用率。
  * 对于 AMD GPU ，使用 [radeontop](<https://archlinux.org/packages/?name=radeontop>)包 查看 GPU 利用率。不同于 [intel-gpu-tools](<https://archlinux.org/packages/?name=intel-gpu-tools>)包 ，当前没有任何方法在 [radeontop](<https://archlinux.org/packages/?name=radeontop>)包 [[3]](<https://github.com/clbr/radeontop/issues/29>) 上查看编码器/解码器的利用率。
  * 无论是哪种 GPU，你都可以使用 [htop](<https://archlinux.org/packages/?name=htop>)包 等工具对比 CPU 使用率。特别是对于更高分辨率(4k+)的视频，如果启用了并正常使用 VA-API，CPU 占用应显著降低，尤其是在笔记本等低功耗设备上。

##  配置

Although the video driver should automatically enable hardware video acceleration support for both VA-API and VDPAU, it may be needed to configure VA-API/VDPAU manually. Only continue to this section if you went through [#Verification](<#Verification>). 

The default driver names, used if there is no other configuration present, are guessed by the system. However, they are often hacked together and may not work. The guessed value will be printed in the [Xorg](<../zh-cn/Xorg.html> "Xorg") log file, which is `~/.local/share/xorg/Xorg.0.log` if rootless, or `/var/log/Xorg.0.log` if Xorg is running as root. To search the log file for the values of interest: 
    
    $ grep -iE 'vdpau | dri driver' _xorg_log_file_
    
    (II) RADEON(0): [DRI2] DRI driver: radeonsi
    (II) RADEON(0): [DRI2] VDPAU driver: radeonsi
    
In this case `radeonsi` is the default for both VA-API and VDPAU. 

**注意：** 如果你使用 [GDM](<../zh-cn/GDM.html> "GDM"), 请以 root 权限运行 `journalctl -b --grep='vdpau | dri driver'` 。

This does not represent the _configuration_ however. The values above will not change even if you override them. 

###  配置 VA-API

You can override the driver for VA-API by using the `LIBVA_DRIVER_NAME` [environment variable](<../zh-cn/Environment_variable.html> "Environment variable"): 

  * [Intel graphics](<../zh-cn/Intel_graphics.html> "Intel graphics"): 
    * For [libva-intel-driver](<https://archlinux.org/packages/?name=libva-intel-driver>)包 use `i965`.
    * For [intel-media-driver](<https://archlinux.org/packages/?name=intel-media-driver>)包 use `iHD`.
  * NVIDIA: 
    * For [Nouveau](<../zh-cn/Nouveau.html> "Nouveau") use `nouveau`.
    * For [NVIDIA](<../zh-cn/NVIDIA.html> "NVIDIA") NVDEC use `nvidia`.
  * AMD: 
    * For [AMDGPU](<../zh-cn/AMDGPU.html> "AMDGPU") driver use `radeonsi`.

**注意：**

  * You can find the installed drivers in `/usr/lib/dri/`. They are used as `/usr/lib/dri/**${LIBVA_DRIVER_NAME}** _drv_video.so`.
  * Some drivers are installed several times under different names for compatibility reasons. You can see which by running `sha1sum /usr/lib/dri/* | sort`.
  * `LIBVA_DRIVERS_PATH` can be used to overrule the VA-API drivers location.
  * Since version 12.0.1 Mesa provides `radeonsi` instead of `gallium`.

###  配置 VDPAU

You can override the driver for VDPAU by using the `VDPAU_DRIVER` [environment variable](<../zh-cn/Environment_variable.html> "Environment variable"). 

The correct driver name depends on your setup: 

  * For Intel graphics you [need](<#Failed_to_open_VDPAU_backend>) to set it to `va_gl`.
  * For the open source AMD driver set it to `radeonsi`.
  * For the open source Nouveau driver set it to `nouveau`.
  * For NVIDIA's proprietary version set it to `nvidia`.

**注意：**

  * You can find the installed drivers in `/usr/lib/vdpau/`. They are used as `/usr/lib/vdpau/libvdpau_**${VDPAU_DRIVER}**.so`.
  * Some drivers are installed several times under different names for compatibility reasons. You can see which by running `sha1sum /usr/lib/vdpau/*`.
  * For hybrid setups (both NVIDIA and AMD), it may be necessary to set the `DRI_PRIME` [environment variable](<../zh-cn/Environment_variable.html> "Environment variable"). For more information see [PRIME](<../zh-cn/PRIME.html> "PRIME").

### Configuring Vulkan Video

  * [Intel graphics](<../zh-cn/Intel_graphics.html> "Intel graphics"): Vulkan Video support in [vulkan-intel](<https://archlinux.org/packages/?name=vulkan-intel>)包 can be enabled with the `ANV_DEBUG=video-decode,video-encode` [environment variable](<../zh-cn/Environment_variable.html> "Environment variable").

  * [AMD](</wzh/index.php?title=AMD&action=edit&redlink=1> "AMD（页面不存在）"): Vulkan Video support in [vulkan-radeon](<https://archlinux.org/packages/?name=vulkan-radeon>)包 is enabled by default for VCN 2, 3, and 4+ since Mesa 25. To force-enable support on older cards, set the `RADV_PERFTEST` [environment variable](<../zh-cn/Environment_variable.html> "Environment variable") to `video_decode,video_encode`.

##  疑难解答

###  打开 VDPAU 后端失败

You need to set `VDPAU_DRIVER` variable to point to correct driver. See [#配置 VDPAU](<#%E9%85%8D%E7%BD%AE_VDPAU>). 

###  VA-API 初始化失败

An error along the lines of `libva: /usr/lib/dri/i965_drv_video.so init failed` is encountered. This can happen because of improper detection of Wayland. One solution is to unset `$DISPLAY` so that mpv, MPlayer, VLC, etc. do not assume it is X11. Another mpv-specific solution is to add the parameter `--gpu-context=wayland`. 

This error can also occur if you installed the wrong VA-API driver for your hardware. 

###  使用 AMDGPU 驱动时视频解码损坏或失真

When experiencing video decoding corruption or distortion with [AMDGPU](<../zh-cn/AMDGPU.html> "AMDGPU") driver, set `allow_rgb10_configs=false` as an [environment variable](<../zh-cn/Environment_variable.html> "Environment variable"). [[4]](<https://bugs.freedesktop.org/show_bug.cgi?id=106490>)
