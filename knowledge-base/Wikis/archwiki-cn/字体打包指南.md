##  通用指引

###  包命名

  * [TTF](<https://zh.wikipedia.org/wiki/TrueType> "zhwp:TrueType") 字体使用`ttf-`前缀。
  * [OTF](<https://zh.wikipedia.org/wiki/OpenType> "zhwp:OpenType") 字体使用`otf-`前缀。

若为[可变字体](<https://zh.wikipedia.org/wiki/%E5%8F%AF%E5%8F%98%E5%AD%97%E4%BD%93> "zhwp:可变字体")，使用`-variable`后缀。 

包名应使用全小写。 

###  包描述

至少要表明它是一个字体，并注明字体类别：衬线体、非衬线体、等宽等。 

###  架构

字体是与架构无关的。使用`arch=(any)`。 

###  依赖

字体没有依赖项。但是许多仓库里的包依赖于 [fontconfig](<https://archlinux.org/packages/?name=fontconfig>)包 和 [xorg-mkfontscale](<https://archlinux.org/packages/?name=xorg-mkfontscale>)包，这些依赖是以前字体包还需要使用安装脚本更新字体缓存时的历史遗留。现在可以用 [pacman 钩子](<../zh-cn/Pacman.html#%E9%92%A9%E5%AD%90> "Pacman")来完成这项工作。安装上述两个包时，`/usr/share/fonts/`下的所有字体都会被缓存，因此无需再强制使用 fontconfig 或 mkfontscale。 

###  提供

许多应用依赖于虚拟包`ttf-font`。若你的字体家族满足[该页面](<https://archlinux.org/todo/fix-ttf-font-default-font-setup>)的要求，可添加`provides=('ttf-font')`。 

###  来源

可按下列顺序查询： 

  * 官方网站，或某个如 GitHub 的托管平台
  * [Google Fonts](<https://fonts.google.com>): `https://fonts.google.com/download?family=_Name_`
  * [Font Squirrel](<https://www.fontsquirrel.com>)、[Font Library](<https://fontlibrary.org>)、[FFonts](<https://www.ffonts.net>)、[1001 Fonts](<https://www.1001freefonts.com>)、 [DaFont](<https://www.dafont.com>)

以下来源不推荐： 

  * [FontSpace](<https://www.fontspace.com>)（校验码因不明原因变动）

##  打包

下列片段是以 OFL（SIL Open Font License）授权的 OTF 字体的示例片段： 
    
    license=('OFL')
    ...
    package() {
      install -Dm644 -t "$pkgdir/usr/share/fonts/OTF" "path/to/font/*.otf"
      install -Dm644 -t "$pkgdir/usr/share/licenses/$pkgname" "path/to/license/OFL.txt"
    }
    
  * [OFL](<https://zh.wikipedia.org/wiki/SIL%E5%BC%80%E6%BA%90%E5%AD%97%E4%BD%93%E6%8E%88%E6%9D%83> "zhwp:SIL开源字体授权")理论上是个[自定义许可](<../zh-cn/PKGBUILD.html#license> "PKGBUILD")，因此字体包的许可证文件需要安装在`/usr/share/licenses/[包名]`。
  * 若有许多个文件，目标目录名可考虑使用`包名`而非`OTF`。
