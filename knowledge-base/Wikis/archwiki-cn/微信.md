微信（WeChat）是腾讯公司于 2011 年 1 月 21 日推出的一个为智能终端提供即时通讯服务的免费专有应用程序。 

目前微信官方适配了部分 Linux 发行版（见下），如有兴趣可测试补充其他发行版上的兼容情况。 

除安装原生版本外，本文还将介绍如何通过其他方法安装微信，以满足不同的需求。 

##  Linux 版

**提示：** ✅这是当前比较好的解决方案。

2024年3月，腾讯推出了基于原生跨平台方案的微信（Universal），旨在逐步实现微信 Windows、Mac 和 Linux 版本在功能和更新节奏上的一致性。腾讯官方提供了 rpm、deb 和 appimage 格式的安装包，但目前尚未实现更新源，因此 deb 和 rpm 用户无法通过包管理器实现自动更新。 

微信原版打包于 _[wechat-bin](<https://aur.archlinux.org/packages/wechat-bin/>) AUR_, 并有包 _[wechat](<https://aur.archlinux.org/packages/wechat/>) AUR_ 提供功能增强和修复。后者有进程管理，沙盒（可选），输入法及 HiDPI 修复等功能。也可以使用 [Flatpak](<../zh-cn/Flatpak.html> "Flatpak") 从 Flathub 安装 `com.tencent.WeChat`。 

_[wechat-universal-bwrap](<https://aur.archlinux.org/packages/wechat-universal-bwrap/>) AUR_提供了基本沙盒封装的微信，它最初是基于现已不存在的 _[wechat-uos-qt](<https://aur.archlinux.org/packages/wechat-uos-qt/>) AUR_，因此它不依赖于 _[wechat-bin](<https://aur.archlinux.org/packages/wechat-bin/>) AUR_。其使用[Bubblewrap](<../zh-cn/Bubblewrap.html> "Bubblewrap")沙盒而非[portable](<https://github.com/Kraftland/portable>) 沙盒，不具备 D-Bus 过滤等功能，**有沙盒逃逸风险** 。 

###  沙盒

_[wechat](<https://aur.archlinux.org/packages/wechat/>) AUR_ 可选启用 [portable](<https://github.com/Kraftland/portable>) 沙盒. 它有一些独特功能: 

  * 支持 GNOME 后台监视
  * 简易的文件分享
  * D-Bus 安全过滤 
    * 请注意: 截图功能仅在 GNOME 桌面环境可用, 因为一些桌面实现无须用户授权即可截图
  * 为双显卡用户隐藏独立显卡, 避免唤醒耗电
  * 自动输入法配置
  * 隐藏一些系统和硬件信息

右键任务栏图标以访问实用功能: 

  * 打开或关闭沙盒
  * 向沙盒发送文件 (位于沙盒家目录内的 Shared 文件夹. 向沙盒发送的文件并不是持久性的, 重启程序即会删除)
  * 彻底退出微信

同时, 还可以使用环境变量 `bwBindPar`向沙盒暴露某个路径, 请注意这将会授予微信完全的读写权限. 

_[wechat](<https://aur.archlinux.org/packages/wechat/>) AUR_ 的沙盒需要 [XDG 桌面门户](<../zh-cn/XDG_%E6%A1%8C%E9%9D%A2%E9%97%A8%E6%88%B7.html> "XDG 桌面门户")正常工作. 正常来说, 使用桌面环境 (DE) 时应无须特别处理, 窗口管理器用户可能需要根据文档自行选择实现。 

_[wechat-universal-bwrap](<https://aur.archlinux.org/packages/wechat-universal-bwrap/>) AUR_使用[Bubblewrap](<../zh-cn/Bubblewrap.html> "Bubblewrap")沙盒，以`~/Documents/WeChat_Data/home`为家目录，可以编写`~/.config/wechat-universal/binds.list`来设置更多的被暴露到容器内的文件/文件夹，每行一个路径，要将微信文件的主要路径修改至`~/Documents/WeChat_Data`外的其他路径，请设置环境变量 `WECHAT_DATA_DIR`

###  输入法与 HiDPI

_[wechat](<https://aur.archlinux.org/packages/wechat/>) AUR_ 将会自动设置环境变量以绕过此类问题. 未设置`XMODIFIERS` 变量可能导致探测错误. 

###  Emoji 字体显示问题

安装 [ttf-twemoji](<https://aur.archlinux.org/packages/ttf-twemoji/>)AUR，并升级 _[wechat](<https://aur.archlinux.org/packages/wechat/>) AUR_ 到 1.0.0.241-27 及以上版本。 

##  虚拟机

可以在 [VMware](<../zh-cn/VMware.html> "VMware") 或 [VirtualBox](<../zh-cn/VirtualBox.html> "VirtualBox") 等虚拟机里安装 Windows 系统来使用微信。 

## Wine

**注意：** 目前 Wine-based 微信小程序功能全部不可用，大于一定尺寸的图片无法发送。详情请参阅 [这里](<https://github.com/countstarlight/deepin-wine-wechat-arch/issues/42>)。

###  安装

兼容层 [Wine](<../zh-cn/Wine.html> "Wine") 可用于在 Linux 中运行微信。 

此外，[deepin-wine-wechat](<https://aur.archlinux.org/packages/deepin-wine-wechat/>)AUR（无补丁）也提供了它，这是一个为 Arch 配置的 Wine 容器。版本为最新官方版本。 

####  使用 archlinuxcn 仓库的 wine-for-wechat

如果微信在使用过程中在其他窗口之上有窗口阴影，可以尝试使用 [archlinuxcn](<../zh-cn/Arch_Linux_%E4%B8%AD%E6%96%87%E7%A4%BE%E5%8C%BA%E4%BB%93%E5%BA%93.html> "Archlinuxcn") 仓库中的 _wine-for-wechat_ 。 此版本的 Wine 使用[这个补丁](<https://github.com/archlinuxcn/repo/blob/master/archlinuxcn/wine-for-wechat/wine-wechat.patch>)修复微信窗口阴影。 

另外，建议安装 [wine-wechat-setup](<https://github.com/archlinuxcn/repo/tree/master/archlinuxcn/wine-wechat-setup>)[CNRepo](<../zh-cn/Arch_Linux_%E4%B8%AD%E6%96%87%E7%A4%BE%E5%8C%BA%E4%BB%93%E5%BA%93.html> "Arch Linux 中文社区仓库") 包来安装，运行和配置微信（微信的 Windows 安装包需要自行下载）。 

####  deepin-wine 微信

可以安装 [com.qq.weixin.deepin](<https://aur.archlinux.org/packages/com.qq.weixin.deepin/>)AUR软件包，这是 Deepin 官方配置的 Wine 化应用，但版本较旧 (3.2.1)。该版本的小程序功能不可用。 

若需要更新的版本可选择安装 [com.qq.weixin.spark](<https://aur.archlinux.org/packages/com.qq.weixin.spark/>)AUR或[deepin-wine-wechat](<https://aur.archlinux.org/packages/deepin-wine-wechat/>)AUR。基本保持与官方 Windows 版本更新同步，且可以使用小程序。 

[Deepin-wine](<../zh-cn/Deepin-wine.html> "Deepin-wine") 还提供了其他的相关软件包，比如企业微信[com.qq.weixin.work.deepin](<https://aur.archlinux.org/packages/com.qq.weixin.work.deepin/>)AUR。 

####  在容器中运行

[docker-wechat](<https://github.com/huan/docker-wechat>) 在一个 [Docker](<../zh-cn/Docker.html> "Docker") 容器中运行 Wine wechat。 

###  配置

####  高分辨率屏幕

如果屏幕 DPI 比较高，界面和字体会比较小。需要通过如下方式配置 wine 的屏幕分辨率 DPI: 

  1. 打开对应的 winecfg, 下面的是 [deepin-wine-wechat](<https://aur.archlinux.org/packages/deepin-wine-wechat/>)AUR 的示例，请按实际情况修改 `WINEPREFIX` 和 `deepin-wine6-stable`: 
         
         $ WINEPREFIX=~/.deepinwine/Deepin-WeChat/ deepin-wine6-stable winecfg

  2. 点击显示，调整 _屏幕分辨率_ 为合适的数值。
  3. 微信中，启用`适配系统缩放比例`选项(设置-->通用设置-->适配系统缩放比例)

##  统信 UOS 魔改版

安装 [wechat-uos](<https://aur.archlinux.org/packages/wechat-uos/>)AUR 软件包即可。 

**注意：** 原统信 UOS 软件包会验证运行环境是否是统信 UOS 专业版，在非 UOS 专业版的环境下会工作异常。 

本解决方案通过使用 [bubblewrap](<../zh-cn/Bubblewrap.html> "Bubblewrap") 替换 `os-release` 和 `lsb-release`，在没有修改其分发的二进制文件的前提下，实现模拟 UOS 专业版运行环境，使其能正常运行。 

此包功能较少，基本是微信网页版套壳，建议使用原生版本。
