##  如何降级软件包

###  使用pacman的临时文件

如果一个新包刚刚被安装并且没有删除[pacman cache](<../zh-cn/Pacman.html#%E6%B8%85%E7%90%86%E8%BD%AF%E4%BB%B6%E5%8C%85%E7%BC%93%E5%AD%98> "Pacman"),你可以在`/var/cache/pacman/pkg/`中找到较早版本. 安装替换现有的版本.[pacman](<../zh-cn/Pacman.html> "Pacman")会处理依赖包但不会处理依赖库的版本冲突。如果一个其依赖库因该包降级需要降级，你需要手动降级这些包。 
    
     # pacman -U /var/cache/pacman/pkg/_package_ -_old_version_.pkg.tar._type_
    
对老的软件包，`type` 应该是 `xz`，遵循 [2020 变更](<https://archlinux.org/news/now-using-zstandard-instead-of-xz-for-package-compression/>)的新软件包，`type` 应该是 `zst`。 

当成功降级该包以后，请暂时将其加入`pacman.conf`的[IgnorePkg section](<../zh-cn/Pacman.html#%E5%9C%A8%E5%8D%87%E7%BA%A7%E6%97%B6%E8%B7%B3%E8%BF%87%E8%BD%AF%E4%BB%B6%E5%8C%85> "Pacman")，直到您的问题被解决。 

###  降级内核

新内核出问题时，可以通过使用 pacman 缓存降级内核。在`/var/cache/pacman/pkg` 寻找旧版安装包。必须降级的有[linux](<https://archlinux.org/packages/?name=linux>)包， [linux-headers](<https://archlinux.org/packages/?name=linux-headers>)包和任何内核模块。举例来说： 
    
    # pacman -U linux-4.15.8-1-x86_64.pkg.tar.xz linux-headers-4.15.8-1-x86_64.pkg.tar.xz virtualbox-host-modules-arch-5.2.8-4-x86_64.pkg.tar.xz
    
退出并重启。 

如果您在升级内核后无法启动，您可以通过使用 live CD 来降级内核。方法类似Arch Linux的安装进程。启动后在`/mnt`挂载你的根目录文件系统，别忘了挂载其他分区如`/boot`或`/var`等等。(例子 `mount /dev/sdc3 /mnt/boot`)。然后[chroot](<../zh-cn/Chroot.html> "Chroot")进入系统。 

### Arch Linux Archive

[Arch Linux Archive](<../zh-cn/Arch_Linux_Archive.html> "Arch Linux Archive")是[official repositories](<../zh-cn/Official_repositories.html> "Official repositories")的日更快照。 

_ALA_ 能被用来降级包或者还原整个系统到过去版本。 

###  手动编译

如果找不到编译好的软件包，就需要自己找到 [PKGBUILD](<../zh-cn/PKGBUILD.html> "PKGBUILD") 并通过 [makepkg](<../zh-cn/Makepkg.html> "Makepkg") 编译。 

[官方软件仓库](<../zh-cn/Official_repositories.html> "Official repositories")中的 PKGBUILD 可以通过 [ABS](<../zh-cn/Arch_%E6%9E%84%E5%BB%BA%E7%B3%BB%E7%BB%9F.html> "ABS") 获取并修改软件版本。或者访问[软件包](<https://archlinux.org/packages>)页面搜索你需要的降级的软件包，然后点 "查看修改" 链接，选择“查看日志“。找到需要的版本并通过 _Tree_ 视图下载 `.tar.gz` 快照。 

参阅 [Arch Build System#Checkout an older version of a package](<../zh-cn/Arch_Build_System.html#Checkout_an_older_version_of_a_package> "Arch Build System"). 

老的 AUR 软件包可以从 AUR 软件包 git 仓库提出旧版本。如果是 2015 AUR3 之前的 PKGBUILD，请参阅 [Arch User Repository#Git repositories for AUR3 packages](<../zh-cn/Arch_User_Repository.html#Git_repositories_for_AUR3_packages> "Arch User Repository").
