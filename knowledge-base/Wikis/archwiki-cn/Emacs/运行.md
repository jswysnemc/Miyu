同 [Vim](<../../zh-cn/Vim.html> "Vim")、[Neovim](<../../zh-cn/Neovim.html> "Neovim") 一样，在启动 emacs 之前，您应该知道如何关闭它（尤其是在终端中运行时）：使用 `Ctrl+x``Ctrl+c` 键序。 

要启动 Emacs，请运行： 
    
    $ emacs
    
如果 [X11](<../../zh-cn/Xorg.html> "X11") 或 [Wayland](<../../zh-cn/Wayland.html> "Wayland") 图形界面可用，此命令将会启动 Emacs 的图形用户界面。在这种情况下，建议运行以下命令以使其在后台运行： 
    
    $ emacs &
    
或在纯控制台中使用 Emacs（出于空间原因，菜单通常不会显示，可以按 `F10` 调出菜单。）： 
    
    $ emacs -nw
    
或在命令行中快速加载（无需 .emacs）和编辑： 
    
    $ emacs -Q -nw
    
如果安装的是 nox 版本，`emacs` 和 `emacs -nw` 将启相同作用。 

还可以提供文件名，以便立即打开该文件： 
    
    $ emacs _filename.txt_
    
##  无彩色

默认情况下，Emacs 启动时的颜色主题显示超链接为深蓝色。要在文本终端上启动 Emacs 而不使用任何颜色主题或方案，请执行下列操作： 
    
    $ emacs -nw --color=no
    
这将使所有文本显示为终端的前景色——通常是黑底白字或白底黑字。 

##  作为守护进程

为了避免每次 Emacs 启动时都重新加载 Emacs [配置文件](<#%E9%85%8D%E7%BD%AE>)，可以将 Emacs 作为[守护进程](<../../zh-cn/Systemd.html> "守护进程")运行： 
    
    $ emacs --daemon
    
然后，您可以连接到守护进程通过运行： 
    
    $ emacsclient -nc
    
`-c` 将创建一个新帧缓冲区（如果希望在终端中使用，请使用 `-t`}），并且不会占用终端 `-n` `--no-wait`）。 

许多[窗口管理器](<../../zh-cn/%E7%AA%97%E5%8F%A3%E7%AE%A1%E7%90%86%E5%99%A8.html> "窗口管理器")、[混成器](<../../zh-cn/Wayland.html#%E6%B7%B7%E6%88%90%E5%99%A8> "Wayland")以及[桌面环境](<../../zh-cn/%E6%A1%8C%E9%9D%A2%E7%8E%AF%E5%A2%83.html> "桌面环境")都实现了 [XDG Autostart](<../../zh-cn/XDG_Autostart.html> "XDG Autostart") 规范，您可以利用此特性在[桌面环境](<../../zh-cn/%E6%A1%8C%E9%9D%A2%E7%8E%AF%E5%A2%83.html> "桌面环境")等启动时[自动运行](<../../zh-cn/%E8%87%AA%E5%8A%A8%E5%90%AF%E5%8A%A8.html> "自动启动") `emacs --daemon`。 

需要注意的是，有些程序（如 [Mutt](<../../zh-cn/Mutt.html> "Mutt") 或 [Git](<../../zh-cn/Git.html> "Git")，用于提交信息）会等待编辑器结束，因此不能使用 `-n` 参数。 如果你的默认编辑器设置为使用该参数，则必须为这些程序指定替代编辑器（ _例如_ `emacsclient -a "" -t`）。 

##  作为 systemd 单元

[Emacs 26.1](<https://lists.gnu.org/archive/html/emacs-devel/2018-05/msg00765.html>) 中包含一个 [systemd](<../../zh-cn/Systemd.html> "Systemd") 单元。该单元与 Emacs 一起安装，但必须在[安装](<../../zh-cn/Help:%E9%98%85%E8%AF%BB.html#%E5%AE%89%E8%A3%85%E8%BD%AF%E4%BB%B6%E5%8C%85> "安装") Emacs 后作为[用户单元](<../../zh-cn/Systemd/%E7%94%A8%E6%88%B7.html> "用户单元")（而非全系统单元）[启用](<../../zh-cn/Help:%E9%98%85%E8%AF%BB.html#%E6%8E%A7%E5%88%B6_systemd_%E5%8D%95%E5%85%83> "启用")： 
    
    $ systemctl --user enable --now emacs
    
服务启动后，Emacs 就准备就绪了。 

如果想在 [Wayland](<../../zh-cn/Wayland.html> "Wayland") 上通过 `emacsclient` 启动图形 emacs 框架，则需要特定的[附加配置片段](<../../zh-cn/Systemd.html#%E9%99%84%E5%8A%A0%E9%85%8D%E7%BD%AE%E7%89%87%E6%AE%B5> "附加配置片段")（除非运行的是 [emacs-wayland](<https://archlinux.org/packages/?name=emacs-wayland>)包），如 [EmacsWiki](<https://www.emacswiki.org/emacs/EmacsAsDaemon#h5o-17>) 所示。 

请注意，systemd 用户单元不会从登录 shell（如 `~/.bash_profile`）继承[环境变量](<../../zh-cn/%E7%8E%AF%E5%A2%83%E5%8F%98%E9%87%8F.html> "环境变量")。更多信息请参见 [Systemd/用户#环境变量](<../../zh-cn/Systemd/%E7%94%A8%E6%88%B7.html#%E7%8E%AF%E5%A2%83%E5%8F%98%E9%87%8F> "Systemd/用户")。 

如果将 emacs 作为[守护进程](<../../zh-cn/Systemd.html> "守护进程")启动，可能需要将 `VISUAL` 和 `EDITOR` 环境变量设置为 `emacsclient`，这样启动编辑器的程序就会使用 emacsclient，而不是启动编辑器的完整实例。使用外部编辑器的程序包括电子邮件程序（用于编辑邮件）、[Git](<../../zh-cn/Git.html> "Git")（用于编辑提交信息）和 less（`v` 命令用于编辑显示的文件）。请勿使用 `-n`（`--nowait`）选项，因为程序通常希望编辑器退出时编辑已经完成。 

还建议更改 Emacs 的任何图形用户界面开始菜单条目（或类似条目、[桌面项](<../../zh-cn/%E6%A1%8C%E9%9D%A2%E9%A1%B9.html> "桌面项")），使其指向 emacsclient 而不是 emacs，这样就可以使用 emacs 守护进程而不是启动一个新的 emacs 进程。
