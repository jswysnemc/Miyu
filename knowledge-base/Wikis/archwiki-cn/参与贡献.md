在进化生物学中，[合作](<https://en.wikipedia.org/wiki/Co-operation_\(evolution\)> "wikipedia:Co-operation \(evolution\)")是指个体付出一个小成本，可以为其他同胞带来更大的收益，而其他个体也一样提供回报，这样每个参与者都会获得非常大的收益。Arch 社区也是如此，很多成员希望为自己喜爱的 GNU/Linux 发行版做出贡献。大家的参与，不仅有利于 Archer 同胞，还能造福[自由软件和开源软件](<https://zh.wikipedia.org/wiki/%E8%87%AA%E7%94%B1%E5%8F%8A%E5%BC%80%E6%94%BE%E6%BA%90%E4%BB%A3%E7%A0%81%E8%BD%AF%E4%BB%B6> "zhwp:自由及开放源代码软件")的全体爱好者。 

本文简单介绍了 Archer 为 Arch 社团做贡献的若干途径。开始贡献前请先熟悉一下[行为准则](<../zh-cn/%E8%A1%8C%E4%B8%BA%E5%87%86%E5%88%99.html> "行为准则")。 

##  社区

###  论坛发帖

参与我们的一个最简单的方法，就是逛 [Arch Linux 官方论坛](<https://bbs.archlinux.org/>)。您可以尽情发表您的高见、帮助菜鸟或结交新朋友。 

###  完善维基

[ArchWiki](<../Project:%E5%85%B3%E4%BA%8E.html> "AboutWiki") 是一个**社区** 文档工程。内容**源自** 社区，**服务** 社区。贡献者奉献他们的时间和精力，在社区中共享知识和技能。我们鼓励**全体** 用户[做贡献](<../Project:%E8%B4%A1%E7%8C%AE.html> "Project:Contributing")。 

###  加入聊天室

可以在 [Arch 的 IRC 频道](<../zh-cn/Arch_%E7%9A%84_IRC_%E9%A2%91%E9%81%93.html> "Arch 的 IRC 频道")中帮助新人解决问题。进入前请先阅读[讨论规则](<../zh-cn/General_guidelines.html#IRC> "General guidelines")。[有许多频道](<../zh-cn/Arch_%E7%9A%84_IRC_%E9%A2%91%E9%81%93.html> "Arch 的 IRC 频道")专门用来讨论不同的问题。 

###  加入邮件列表

加入感兴趣的[邮件列表](<https://lists.archlinux.org/mailman3/lists/>)。注意讨论的内容要遵守列表描述。 

###  艺术

共享屏保、启动屏幕、调色盘、主题等，可以加入[艺术子论坛](<https://bbs.archlinux.org/viewforum.php?id=47>)以及中文论坛的[美工与截图](<https://bbs.archlinuxcn.org/viewforum.php?id=32>)版块。 

参阅 [Arch Linux Art](<https://archlinux.org/art/>) 和 [ASCII 艺术](<../zh-cn/ASCII_%E8%89%BA%E6%9C%AF.html> "ASCII 艺术")。 

##  软件包

###  报告安装的软件

[pkgstats](<../zh-cn/Pkgstats.html> "Pkgstats") 使用 [systemd 定时器](<../zh-cn/Systemd/%E5%AE%9A%E6%97%B6%E5%99%A8.html> "Systemd/Timers")定期将系统中安装的软件上报服务器，同时还能提供系统架构和使用镜像的信息。开发者可以通过数据统计，安排自己工作的优先级。信息是匿名发送，不会暴露用户隐私。可以通过[统计页面](<https://pkgstats.archlinux.de/packages>)查看信息。详情请阅读[这个论坛帖子](<https://bbs.archlinux.org/viewtopic.php?id=105431>)。 

###  报告和修正 bug

您还可以通过到 [Arch Linux Bugtracker](<https://gitlab.archlinux.org/archlinux/packaging/packages+GitLab>) 汇报有关 Arch 软件包 bug 来帮忙。 

但若汇报的问题无意义时，反而会浪费大家的时间。如有意参与其中，请先阅读[漏洞报告准则](<../zh-cn/%E6%BC%8F%E6%B4%9E%E6%8A%A5%E5%91%8A%E5%87%86%E5%88%99.html> "漏洞报告准则")和 [General guidelines#Packaging merge requests](<https://wiki.archlinux.org/title/General_guidelines#Packaging_merge_requests> "en:General guidelines")。 

###  安全问题

新的漏洞不停出现，帮助 [Arch 安全小组](<../zh-cn/Arch_Security_Team.html> "Arch Security Team")追踪新的漏洞。 

###  帮助测试软件包

测试仓库中的软件包需要进行测试后才会进入主仓库。请帮助 [Arch 测试组](<../zh-cn/Arch_Testing_Team.html> "Arch Testing Team")测试新软件包。 

###  提出需求

在 [bug 系统](<https://gitlab.archlinux.org/archlinux/packaging/packages+GitLab>)中提出软件包需求，提交前请先阅读[漏洞报告准则#Bug 或者 Feature？](<../zh-cn/%E6%BC%8F%E6%B4%9E%E6%8A%A5%E5%91%8A%E5%87%86%E5%88%99.html#Bug_%E6%88%96%E8%80%85_Feature%EF%BC%9F> "漏洞报告准则")，以确保功能需求是合理的，此外还需阅读 [General guidelines#Packaging merge requests](<https://wiki.archlinux.org/title/General_guidelines#Packaging_merge_requests> "en:General guidelines") 。 

在 AUR 软件包页面的相关页面提出 AUR 软件包需求。 

###  创建或维护 AUR 软件包

[Arch 用户软件仓库](<../zh-cn/Arch_User_Repository.html> "Arch User Repository")是一个大众主导的软件仓库，包含了大量用户提交的 PKGBUILD。与官方软件库二进制包的形式不同，AUR 中的软件包将以 PKGBUILD 的形式供用户使用，并全由用户自行编译。用户可通过 AUR 分享软件，其中一些热门软件会选入官方的 [Extra](<../zh-cn/%E5%AE%98%E6%96%B9%E4%BB%93%E5%BA%93.html#extra> "官方仓库") 软件仓库。 

###  成为 Trusted User

像帮助维护 [AUR](<../zh-cn/Arch_%E7%94%A8%E6%88%B7%E8%BD%AF%E4%BB%B6%E4%BB%93%E5%BA%93.html> "AUR") 和 [extra 软件仓库](<../zh-cn/%E5%AE%98%E6%96%B9%E4%BB%93%E5%BA%93.html#extra> "官方仓库")中的软件包，可以申请成为[软件包维护者](<../zh-cn/%E8%BD%AF%E4%BB%B6%E5%8C%85%E7%BB%B4%E6%8A%A4%E8%80%85.html> "软件包维护者")。 详情参阅[软件包维护者#我如何成为软件包维护者](<../zh-cn/%E8%BD%AF%E4%BB%B6%E5%8C%85%E7%BB%B4%E6%8A%A4%E8%80%85.html#%E6%88%91%E5%A6%82%E4%BD%95%E6%88%90%E4%B8%BA%E8%BD%AF%E4%BB%B6%E5%8C%85%E7%BB%B4%E6%8A%A4%E8%80%85%EF%BC%9F> "软件包维护者")。

##  怎样才能成为 Arch 开发者?

首先，牢记您所做的一切，都应以帮助大众为出发点。其次，牢记您是社区大众的一分子，帮助别人亦是帮助自己。 

通常，新开发者是根据工作内容的增加由已有开发者选出。有时他们会提供一个职位，您可以申请并获得职位。但更多时候，开发者会邀请能力足够，适合开发团队需要的程序员加入。积累 Arch 贡献是成为团队一员的最好方法。 

这里是一个列表，没有顺序之分。包含了可以为社区做贡献的方式: 

  * 通过及时提供帮助获得声誉
  * 在论坛、IRC、和邮件列表回答问题
  * 加入[软件包维护者](<../zh-cn/%E8%BD%AF%E4%BB%B6%E5%8C%85%E7%BB%B4%E6%8A%A4%E8%80%85.html> "软件包维护者")，获得软件打包经验，提升技能
  * 将软件包提交到 AUR
  * 加入以后可能成为 Arch 主流应用的程序开发或者自己建立一个项目
  * 修改 pacman、makepkg 或[GitLab](<https://gitlab.archlinux.org/>) 上的其它 Arch 项目，并通过 bug 管理系统提交补丁
  * 浏览 [bug](<https://gitlab.archlinux.org/groups/archlinux/packaging/-/issues+bug+tracker>) 管理系统，修正问题
  * 找到并提交新 Bug
  * 修正维基错误、添加页面、清理页面并保证步骤符合最新系统
  * 提交翻译
