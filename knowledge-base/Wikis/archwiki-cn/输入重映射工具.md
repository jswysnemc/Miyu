在 X 或 Wayland 环境中，多数重映射任务可通过本地 XKB 配置完成，参见 [X keyboard extension#Local XKB folder](<../zh-cn/X_keyboard_extension.html#Local_XKB_folder> "X keyboard extension")。对于 Linux 控制台的键位重映射，请参阅 [Linux 控制台/键盘配置](<../zh-cn/Linux_%E6%8E%A7%E5%88%B6%E5%8F%B0/%E9%94%AE%E7%9B%98%E9%85%8D%E7%BD%AE.html> "Linux 控制台/键盘配置")。若键盘设备报告了错误的键码，参见[扫描码映射至键位码](<../zh-cn/%E6%89%AB%E6%8F%8F%E7%A0%81%E6%98%A0%E5%B0%84%E8%87%B3%E9%94%AE%E4%BD%8D%E7%A0%81.html> "扫描码映射至键位码")。 

某些特殊配置需通过软件守护进程转换输入事件。本页列出了可用于重映射键盘、鼠标等硬件输入事件的工具。 

##  工具

### evremap

[evremap](<https://github.com/wez/evremap>)（[evremap-git](<https://aur.archlinux.org/packages/evremap-git/>)AUR）⸺Linux 中 Wayland 下的键盘输入重映射工具。支持以下重映射方式： 

  * 将 `CapsLock` 键映射为组合功能：按住时输出 `Ctrl`，轻按时输出 `Esc`。
  * 实现从 n 个键到 m 个键的映射（如 `F3` → `Ctrl+c`，`Alt+Left` → `Home`）。

安装后需创建配置文件 `/etc/evremap.toml`（仓库[示例](<https://github.com/wez/evremap/blob/master/pixelbookgo.toml>)），或[修改](<../zh-cn/Systemd.html#%E4%BF%AE%E6%94%B9%E7%8E%B0%E5%AD%98%E5%8D%95%E5%85%83%E6%96%87%E4%BB%B6> "Systemd") `evremap.service` 指定配置文件路径。最后[启动](<../zh-cn/Help:%E9%98%85%E8%AF%BB.html#%E6%8E%A7%E5%88%B6_systemd_%E5%8D%95%E5%85%83> "启动")该服务。 

### evdevremapkeys

[evdevremapkeys](<https://github.com/philipl/evdevremapkeys>)（[evdevremapkeys-git](<https://aur.archlinux.org/packages/evdevremapkeys-git/>)AUR）⸺Linux 输入设备按键事件重映射守护进程。支持以下功能： 

  * 重映射键盘与鼠标事件。
  * 实现重复动作映射（如模拟双击操作）。
  * 按键持续触发（如按住返回（Back）侧键时持续生成滚轮上滑事件）。

组合键到组合键的映射功能目前尚未合并至主分支，需使用 pronobis 的分叉版本实现，详见[此处](<https://github.com/philipl/evdevremapkeys/issues/4#issuecomment-407684344>)。 

### evsieve

[evsieve](<https://github.com/KarsMulder/evsieve>)（[evsieve](<https://aur.archlinux.org/packages/evsieve/>)AUR）⸺能够读取 Linux 事件设备（evdev）的输入事件，并将其写入虚拟事件设备（uinput），在传递过程中对事件进行简单处理的底层工具。兼容 Wayland 环境。该工具尤其适合与 Qemu 的 `evdev-passthrough` 功能结合使用。 

### kbct

[kbct](<https://github.com/samvel1024/kbct>)（[kbct-git](<https://aur.archlinux.org/packages/kbct-git/>)AUR）⸺**K** ey**b** oard **C** ustomization **T** ool for Linux，Linux 键盘自定义工具。尽管名称如此，亦支持鼠标事件。支持以下功能： 

  * 实现键盘或鼠标按键事件到其他事件的映射。
  * 可定义多组映射层⸺根据输入时按下的修饰键切换不同映射列表。

**注意：** 目前暂不支持生成复合按键事件，详见[该 issue](<https://github.com/samvel1024/kbct/issues/13>)。

安装后按需编辑 `/etc/kbct/config.yml` 配置文件，随后[启动](<../zh-cn/Help:%E9%98%85%E8%AF%BB.html#%E6%8E%A7%E5%88%B6_systemd_%E5%8D%95%E5%85%83> "启动") `kbct.service`。 

### keyd

[keyd](<https://github.com/rvaiya/keyd>)（[keyd](<https://archlinux.org/packages/?name=keyd>)包）⸺基于内核级输入原语（evdev、uinput）的 Linux 系统级键位重映射守护进程。支持特性包括： 

  * 跨环境兼容：适用于 X11、Wayland 图形环境及 Linux 虚拟控制台。
  * 多层映射：通过修饰键实现动态键位配置切换。
  * 低延迟：直接操作内核事件流，无需用户空间转发。

建议阅读项目 README 了解与其他重映射工具的详细对比分析。 

### Input Remapper

[Input Remapper](<https://github.com/sezanzeb/input-remapper/>)（[input-remapper-git](<https://aur.archlinux.org/packages/input-remapper-git/>)AUR）⸺同时提供图形界面和命令行界面的输入设备重映射配置工具，同时兼容 X 与 Wayland 环境。 

### wtype

[wtype](<https://github.com/atx/wtype>)（[wtype](<https://archlinux.org/packages/?name=wtype>)包）⸺[Wayland](<../zh-cn/Wayland.html> "Wayland") 环境下的 xdotool 输入模拟工具（需混成器支持[虚拟键盘协议](<https://github.com/atx/wtype/issues/45>)）。

##  测试

  * [evtest](<https://gitlab.freedesktop.org/libevdev/evtest>)（[evtest](<https://archlinux.org/packages/?name=evtest>)包）⸺实时显示按键名称的调试工具，有助于配置重映射工具时识别输入事件
  * [wev](<https://git.sr.ht/~sircmpwn/wev>)（[wev](<https://archlinux.org/packages/?name=wev>)包）⸺[Wayland](<../zh-cn/Wayland.html> "Wayland") 输入事件查看器，功能类似 [xev](<../zh-cn/%E9%94%AE%E7%9B%98%E8%BE%93%E5%85%A5.html#%E5%9C%A8_Xorg_%E4%B8%AD%E8%AF%86%E5%88%AB%E2%80%9C%E9%94%AE%E7%A0%81%E2%80%9D> "Xev")

通过 `evsieve --input /dev/input/event* --print` 命令可实时查看计算机发出的所有输入事件流。 

另一种测试方案是使用在线键盘检测网站。需注意多数此类工具无法区分左右修饰键，推荐支持左右键检测的示例站点，例如 <https://stendec.io/>。
