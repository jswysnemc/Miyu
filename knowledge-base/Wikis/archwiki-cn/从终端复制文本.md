大多数成熟的终端模拟器允许用户复制或保存其内容。 

##  一般方法

在图形化终端模拟器中，内容通常可以通过鼠标选择，然后通过右键菜单、编辑菜单或快捷键组合（例如 `Ctrl+Shift+c`）进行复制。 

###  不原生支持 CLIPBOARD 选区的终端

#### Xorg

有些终端仿真器不原生支持 [CLIPBOARD 选区](<../zh-cn/%E5%89%AA%E8%B4%B4%E6%9D%BF.html#%E9%80%89%E5%8C%BA> "剪贴板")，而是将数据复制到 PRIMARY 选取。对于这些终端，可以使用 [xclip](<https://archlinux.org/packages/?name=xclip>)包： 
    
    $ xclip -o | xclip -selection clipboard -i
    
上述命令将从 PRIMARY 选区读取数据并写入 CLIPBOARD 选择。 

其他[剪贴板管理器](<../zh-cn/%E5%89%AA%E8%B4%B4%E6%9D%BF.html#%E7%AE%A1%E7%90%86%E5%99%A8> "剪贴板管理器")（如[autocutsel](<https://aur.archlinux.org/packages/autocutsel/>)AUR）提供选择缓冲区之间的自动同步。 

#### Wayland

像 [wl-clipboard](<https://archlinux.org/packages/?name=wl-clipboard>)包 和 [clipboard](<https://aur.archlinux.org/packages/clipboard/>)AUR 这样的工具可以将数据复制到Wayland剪贴板： 
    
    $ command 2>&1 | wl-copy
    
###  截取命令输出

使用 [tee](</wzh/index.php?title=Tee&action=edit&redlink=1> "Tee（页面不存在）")（英语：[tee](<https://wiki.archlinux.org/title/tee> "en:tee")） 截取命令的输出。 
    
    $ command 2>&1 | tee output-file
    
在执行 `command` 之后，`output-file` 将包含其输出，同时显示输出。 

###  访问 Linux 终端历史

可以通过 `/dev/vcs _N_` 访问名为 `/dev/tty _N_` 的本地终端的历史。因此，如果在 `/dev/tty1` 中工作，以下代码段将把历史存储到文件`output-file`中： 
    
    # cat /dev/vcs1 > output-file

##  特殊情况

### putty

[xclip 方法](<#%E4%B8%8D%E5%8E%9F%E7%94%9F%E6%94%AF%E6%8C%81_CLIPBOARD_%E9%80%89%E5%8C%BA%E7%9A%84%E7%BB%88%E7%AB%AF>)适用于 _putty_ : 只需记住， _xclip_ 命令应该在本地计算机上（在另一个终端中）执行，而不是在与 _putty_ 连接的远程计算机上执行。 

### urxvt

将文本选择到 CLIPBOARD 选区需要 `selection-to-clipboard` Perl 扩展。有关详细信息，请参见 [rxvt-unicode#复制和粘贴](<../zh-cn/Rxvt-unicode.html#%E5%A4%8D%E5%88%B6%E5%92%8C%E7%B2%98%E8%B4%B4> "Rxvt-unicode")。 

### xterm

访问 _xterm_ 中的 CLIPBOARD 选区需要[额外的步骤](</wzh/index.php?title=Xterm&action=edit&redlink=1> "Xterm（页面不存在）")（英语：[xterm#PRIMARY or CLIPBOARD](<https://wiki.archlinux.org/title/xterm#PRIMARY_or_CLIPBOARD> "en:xterm")）。 

### mlterm

除了 `Ctrl+Shift+c`，如果不想意外杀死进程，还可以使用 `Ctrl+Insert`，同时，一些版本的 Mlterm 会在选择时自动实时复制选取。
