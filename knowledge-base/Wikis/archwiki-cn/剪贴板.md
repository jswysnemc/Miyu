根据[维基百科](<https://zh.wikipedia.org/wiki/%E5%89%AA%E8%B4%B4%E6%9D%BF> "zhwp:剪贴板")： 

    剪贴板（英语：clipboard），有时也称剪切板、剪贴簿、剪贴本。它是一种软件功能，通常由操作系统提供，作用是使用复制和粘贴操作短期存储数据和在文档或应用程序间转移数据。

##  历史

在X10（[1985](<https://en.wikipedia.org/wiki/X_Window_System#Release_history> "w:X Window System")），引入了[剪切缓冲区](<https://en.wikipedia.org/wiki/X_Window_selection#Cut_buffers> "w:X Window selection")。这些是有限的缓冲区，用于存储任意文本，并被大多数应用程序使用。然而，它们效率低下，且实现方式各异，因此引入了选择功能。剪切缓冲区现在已经被长期弃用，尽管一些应用程序（如[xterm](</wzh/index.php?title=Xterm&action=edit&redlink=1> "Xterm（页面不存在）")（英语：[xterm](<https://wiki.archlinux.org/title/xterm> "en:xterm")））可能仍然有对它们的遗留支持，但不建议使用它们。 

##  选区

[Freedesktop.org](<../zh-cn/Category:Freedesktop.org.html> "Freedesktop.org") 将两种主要[选区](<https://en.wikipedia.org/wiki/X_Window_selection#Selections> "w:X Window selection")描述如下[[4]](<https://specifications.freedesktop.org/clipboard-spec/latest/>)： 

PRIMARY
    用于当前选定的文本，即使该文本没有显式复制，并用于中键粘贴。在某些情况下，使用快捷键也可以粘贴。
CLIPBOARD
    用于涉及键盘快捷键或菜单项的显式复制/粘贴命令。因此，它的行为类似于 Windows 上的单剪贴板系统。与 PRIMARY 不同，它还可以处理[多种数据格式](<https://stackoverflow.com/questions/3571179/how-does-x11-clipboard-handle-multiple-data-formats>)。

大多数 [Xorg](<../zh-cn/Xorg.html> "Xorg") 程序，包括 [Qt](<../zh-cn/Qt.html> "Qt") 和 [GTK](<../zh-cn/GTK.html> "GTK") 应用程序，遵循此行为。尽管 [ICCCM](<https://tronche.com/gui/x/icccm/>) 还定义了 SECONDARY 选区，但它没有一个广泛认同的用途。尽管名称上有三种选区，但它们基本上都是“剪贴板”。与旧的“剪切缓冲区”系统不同，只有一个应用程序可以控制或“拥有”一个选区。这避免了选区操作中的不一致性。 

请参见[快捷键](<../zh-cn/%E5%BF%AB%E6%8D%B7%E9%94%AE.html> "快捷键")页面，列出了许多程序的默认快捷键。 

还需要注意的是，根据选择协议，直到粘贴之前，什么都不会被复制[[5]](<https://unix.stackexchange.com/questions/213840/how-to-toggle-or-turn-off-text-selection-being-sent-to-the-clipboard/213843#213843>)。例如，如果你在终端窗口中选择了一些文字，关闭终端后想要粘贴到其他地方，它将无法工作，因为终端已经消失，文本并没有被复制到任何地方。如果你希望关闭终端窗口后保存文字，考虑安装[剪贴板管理器](<../zh-cn/%E5%89%AA%E8%B4%B4%E6%9D%BF.html#%E7%AE%A1%E7%90%86%E5%99%A8> "剪贴板管理器")。 

**注意：**[剪贴板管理器](<../zh-cn/%E5%89%AA%E8%B4%B4%E6%9D%BF.html#%E7%AE%A1%E7%90%86%E5%99%A8> "剪贴板管理器")可以显著改变用户体验，例如，它们可能会同步 PRIMARY 和 CLIPBOARD 选择，以模拟单一剪贴板系统。

##  禁用中键粘贴

以下方法通过自动清除 PRIMARY 来禁用中键粘贴行为，而不禁用中键按钮或更改其其他功能（如在新标签页中打开或滚动）。 

###  全局禁用

####  使用 sxhkd

使用 [sxhkd](<../zh-cn/Sxhkd.html> "Sxhkd")，在[配置文件](<../zh-cn/Sxhkd.html#%E9%85%8D%E7%BD%AE> "Sxhkd")中添加以下内容： 
    
    ~button2
        ;echo -n | xclip -in
    
该命令使用[xclip](<https://archlinux.org/packages/?name=xclip>)包。请参见 [sxhkd#用法](<../zh-cn/Sxhkd.html#%E7%94%A8%E6%B3%95> "Sxhkd")配置 sxhkd 以实现自动启动。 

####  使用 xsel

运行脚本： 
    
    #!/bin/sh
    while true; do
        xsel --follow --input --nodetach </dev/null
    done
    
###  特定应用程序禁用

  * [GTK](<../zh-cn/GTK.html> "GTK")——参见 [GTK#禁用鼠标中键粘贴](<../zh-cn/GTK.html#%E7%A6%81%E7%94%A8%E9%BC%A0%E6%A0%87%E4%B8%AD%E9%94%AE%E7%B2%98%E8%B4%B4> "GTK")
  * [Firefox](<../zh-cn/Firefox.html> "Firefox")——参见[Firefox#中键单击行为](<../zh-cn/Firefox.html#%E4%B8%AD%E9%94%AE%E5%8D%95%E5%87%BB%E8%A1%8C%E4%B8%BA> "Firefox")
  * [LibreOffice](<../zh-cn/LibreOffice.html> "LibreOffice")——可以在**工具 > 选项 > LibreOffice > 查看 > 鼠标**中禁用
