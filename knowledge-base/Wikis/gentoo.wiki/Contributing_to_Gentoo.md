This article explains how users can **contribute to the development of Gentoo**. All users are welcome to contribute, even those who are new! There is always room for improvement, and any contributions can help make the Gentoo experience even better.

There are many different ways to help out, so there should be something for everyone. Some areas for contribution have a low bar for entry, so anyone should be able to contribute on some level!

Anyone interested in becoming a Gentoo developer can start by contributing to any of the areas listed below. It\'s good to have a track record of contributions to build a profile, and [recruiters](https://wiki.gentoo.org/wiki/Project:Recruiters "Project:Recruiters") will be able to see past contributions.

** Tip**\
Gentoo accepts **[donations](https://www.gentoo.org/donate/)**! Give a lump sum, or set up a monthly contribution, with PayPal. This helps keep the lights on for the core services powering Gentoo.

** See also**\
See also the [contribute to Gentoo](https://www.gentoo.org/get-involved/contribute/) page on the main website.

## [Support the community]

Help users in the various [support](https://wiki.gentoo.org/wiki/Support "Support") mediums:

-   [Answer unanswered forums questions](https://forums.gentoo.org/search.php?search_id=unanswered), or share personal knowledge of solving a particular problem, or discuss something new on the [forums](https://forums.gentoo.org/).
-   [Join the IRC channels](https://www.gentoo.org/get-involved/irc-channels/). The main IRC channel for [support](https://wiki.gentoo.org/wiki/Support "Support") is [[#gentoo](ircs://irc.libera.chat/#gentoo)] ([[webchat](https://web.libera.chat/#gentoo)]). Anyone with a little knowledge is welcome to help out there.
-   [Subscribe to the mailing lists](https://www.gentoo.org/get-involved/mailing-lists/). The main mailing list for users is [gentoo-user](https://archives.gentoo.org/gentoo-user/).

## [Document useful information]

The [Gentoo Wiki](https://wiki.gentoo.org) is the main documentation repository for Gentoo. All users are encouraged to add content and edit articles, even if just to provide a correction or add a quick tip.

Documentation is an area that is both very useful to the community, and is particularly easy to start out with, making it an excellent place to start contributing.

Please check out the **[help improve Gentoo by getting involved with documentation!](https://wiki.gentoo.org/wiki/Help_improve_Gentoo_by_getting_involved_with_documentation! "Help improve Gentoo by getting involved with documentation!")** page. In addition, there is the [contributor\'s guide](https://wiki.gentoo.org/wiki/Gentoo_Wiki:Contributor%27s_guide "Gentoo Wiki:Contributor's guide") and [extensive documentation on how to edit the wiki](https://wiki.gentoo.org/wiki/Help:Contents "Help:Contents").

Help [translating documentation](https://wiki.gentoo.org/wiki/Help:Translating#For_Anyone:_Becoming_a_translator "Help:Translating") is much appreciated, for readers in languages other than English.

Also check out the page [about the Gentoo wiki](https://wiki.gentoo.org/wiki/Gentoo_Wiki:About "Gentoo Wiki:About"), the [Gentoo wiki project page](https://wiki.gentoo.org/wiki/Project:Wiki "Project:Wiki"), and the [Gentoo wiki FAQ](https://wiki.gentoo.org/wiki/Gentoo_Wiki:FAQ "Gentoo Wiki:FAQ").

## [Report and resolve bugs]

[Gentoo Bugzilla](https://bugs.gentoo.org) is used to track bugs for Gentoo and its packages. Anyone is encouraged to report, confirm and resolve bugs:

-   [Report a bug](https://wiki.gentoo.org/wiki/Bugzilla/Bug_report_guide "Bugzilla/Bug report guide") on the [Gentoo Bugzilla](https://bugs.gentoo.org) whenever an error, flaw, failure or fault is encountered. The [Bugzilla guide](https://wiki.gentoo.org/wiki/Bugzilla/Guide "Bugzilla/Guide") provides more in-depth information for how to provide details about an issue.
-   [Confirm new bugs](https://bugs.gentoo.org/buglist.cgi?chfield=%5BBug%20creation%5D&chfieldfrom=-1D&chfieldto=Now&list_id=2122804&order=bugs.bug_id&query_format=advanced) by reproducing them. This helps consolidate the cause as well as highlight severe bugs.
-   [Offer solutions for open bugs](https://sites.google.com/site/yacoset/Home/how-to-fix-bugs-step-by-step). Patches, ebuilds and links to upstream (or other distribution) bug reports are more than welcome!
-   [Participate in a Bugday](https://wiki.gentoo.org/wiki/Bugday "Bugday"), squashing bugs together with Gentoo developers.

## [Create and maintain packages]

The [Gentoo ebuild repository](https://wiki.gentoo.org/wiki/Ebuild_repository#The_Gentoo_ebuild_repository "Ebuild repository") is the heart of Gentoo; it is important to keep it alive and kicking, that\'s why contributions to it are very valuable. After all [ebuilds](https://wiki.gentoo.org/wiki/Ebuild "Ebuild") are what makes building and [installing](https://wiki.gentoo.org/wiki/Emerge "Emerge") packages possible.

You can either maintain or [write](https://wiki.gentoo.org/wiki/Basic_guide_to_write_Gentoo_Ebuilds "Basic guide to write Gentoo Ebuilds") [ebuilds](https://wiki.gentoo.org/wiki/Ebuild "Ebuild"), one commit at a time. Whether you want to commit occasionally, be a [Proxied Maintainer](https://wiki.gentoo.org/wiki/Project:Proxy_Maintainers/User_Guide "Project:Proxy Maintainers/User Guide") or a [Developer](#Become_a_developer), it all helps!

After the first steps, [learn more with the Development Guide](https://devmanual.gentoo.org/).

The following resources can be helpful when adding a new package to the Gentoo ebuild repository, updating an existing one, or finding new packages to contribute to:

-   [Submit pull requests](https://wiki.gentoo.org/wiki/GitHub_Pull_Requests "GitHub Pull Requests") on [GitHub](https://github.com/gentoo/gentoo/pulls) - see [Gentoo git workflow](https://wiki.gentoo.org/wiki/Gentoo_git_workflow "Gentoo git workflow").
-   [Report version bumps](https://wiki.gentoo.org/wiki/Bugzilla/Bug_report_guide#Report_a_version_bump.3B_a_newer_upstream_release_is_available_since_a_while "Bugzilla/Bug report guide") for existing packages at [Gentoo Bugzilla](https://bugs.gentoo.org).
-   [Submit a new package](https://wiki.gentoo.org/wiki/Submitting_ebuilds "Submitting ebuilds") for the Gentoo tree. To maintain a new package, contact the [proxy maintainers](https://wiki.gentoo.org/wiki/Project:Proxy_Maintainers "Project:Proxy Maintainers").
-   [Proxy-maintain](https://wiki.gentoo.org/wiki/Project:Proxy_Maintainers "Project:Proxy Maintainers") one or more of the [orphaned packages](https://qa-reports.gentoo.org/output/maintainer-needed.html).

\
Anyone who can write ebuilds (or is willing to learn), and wants to help out can join the [GURU](https://wiki.gentoo.org/wiki/Project:GURU "Project:GURU") project, to collaboratively maintain packages with the help of a few seasoned developers.

## [Test Gentoo and the packages]

Gentoo is a very stable distribution, but to get there everything needs to be thoroughly tested, even after things have already been shown to be working. To help out making sure everything is always working perfectly for end users, configure and use the [testing branch](https://wiki.gentoo.org/wiki/Handbook:X86/Portage/Branches#Testing "Handbook:X86/Portage/Branches") and [report issues](#Report_and_resolve_bugs) to the developers!

## [Become a developer]

Become a Gentoo Developer to contribute to the project at the most fundamental level. Developers create Gentoo by contributing code, documentation, infrastructure, and such. Developers start by helping out, participating in the community, and once they find a mentor, take the developer quizzes and get recruited.

See the [Becoming a Gentoo Developer](https://www.gentoo.org/get-involved/become-developer/) article for details on how people get on board with the project!

Developers should join one or more of the many [projects](https://wiki.gentoo.org/wiki/Project:Gentoo/Staffing_Needs "Project:Gentoo/Staffing Needs"), depending on areas of contribution.
