**Error Correcting Code** (**ECC**) is a data encoding mechanism intended to render a data resistant to corruption during handling or transit. This should not be confused with techniques such as [parity](https://en.wikipedia.org/wiki/Parity_bit "wikipedia:Parity bit") bits which merely allow the receiver to detect that an error has occurred. Errors can take many forms: errors in storage and data retrieval commonly called [bit rot](https://en.wikipedia.org/wiki/Data_degradation "wikipedia:Data degradation"), transmission errors over telecommunications networks, or random errors that occur in [RAM](https://en.wikipedia.org/wiki/Random-access_memory "wikipedia:Random-access memory") while handling the data. Such errors can have a variety of causes: cosmic radiation, strong electromagnetic interference, a failing power supply, intentional [fuzzing](https://en.wikipedia.org/wiki/fuzzing "wikipedia:fuzzing"), and even cosmic radiation.

Error Correcting Codes of various kinds are used in software and various computer architecture components. [ECC RAM](https://wiki.gentoo.org/wiki/ECC_RAM "ECC RAM") is likely the best known computer component which uses Error Correcting Codes. ECC RAM refers to a class of RAM that can correct for --- and ideally log --- single-bit errors that would otherwise go undetected in a system with non-ECC RAM which would directly result in data corruption and, in time, inevitably result in system instability.
