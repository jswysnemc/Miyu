The concept of a **meta ebuild**, also referred to as a **meta package**, exists in the Gentoo [ebuild repository](https://wiki.gentoo.org/wiki/Ebuild_repository "Ebuild repository") and other ebuild repositories. These [ebuilds](https://wiki.gentoo.org/wiki/Ebuild "Ebuild") typically include the term \"meta\" as a naming suffix, although doing so is not a requirement of a Gentoo development standard (such as the [EAPI](https://wiki.gentoo.org/wiki/EAPI "EAPI"), [GLEPs](https://wiki.gentoo.org/wiki/Project:GLEP "Project:GLEP"), or the [developer manual](https://wiki.gentoo.org/wiki/Project:Devmanual "Project:Devmanual")). Not all meta-style ebuilds use the same terminology.

A meta ebuild is an ebuild containing a developer-selected, curated software suite that generally relates to a certain *family* of related software packages. Many meta ebuilds do not install any files themselves, but are used to reference the installation requirements for a family of related software packages.
